# 核心概念体系

## 信息净化

信息净化指对一组已有或正在生成的信息进行结构、内容和约束层面的优化，使其更容易被人或 AI 正确理解、继续推理、执行和维护。

它不是简单润色，而是降低信息体系的总熵。

暂定公式：

```text
信息净化 = 保真事实 + 闭合概念 + 明确归属 + 显式关系 + 可执行判断 + 可校验结构
```

## 概念闭合

概念闭合是本轮实践中新增的关键概念。

一个概念如果要进入规则或 SOP，它不仅要有名称，还要能支持下一步判断。

检查问题：

- 它是什么意思？
- 它不是什么意思？
- 谁维护它？
- 什么时候使用它？
- 使用它之后下一步该怎么判断？
- 是否存在相近概念需要消歧？

本轮案例：

- `Execution Mode` 如果只定义为「主控或 Subagent」，无法支持触发、跳过、分支判断；升级后才闭合。
- `统一 todo 形式` 如果不说明 dima 和直接需求都经 `i-todo` 形成，就无法支持派发入口判断。

## 权威来源

权威来源用于避免一个文档承担所有规则。

一个概念可以不在当前文档展开全部细节，但当前文档应说明：

- 当前文档只约定什么。
- 细节由谁维护。
- 运行时应该参考谁。
- 维护时可以参考谁。

本轮案例：

- 仓库协议消费归 `i-verify`。
- 需求类型路由归 `i-todo`。
- 阶段产物契约归对应 stage-bound skill 的设计资料。
- skill 运行时行为归 skill 可执行正文。
