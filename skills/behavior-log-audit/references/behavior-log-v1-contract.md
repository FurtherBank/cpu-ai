# behavior-log/v1 契约摘要

> 完整设计见工作区 `.iteration/recordings-design/[BL-001] 工作流技能行为日志存放规范/`。本文档为 Skill 执行时的稳定引用锚点。

## Canonical 路径

```text
{workspaceRoot}/.skill-logs/<skill-slug>/yyyy-mm-dd/hhmm-<execution-suffix>.md
```

- `<skill-slug>`：canonical skill name 小写化，非法字符替换为 `-`
- `<execution-suffix>`：任务标题 slug（创建时确定，后续不改路径）
- 冲突：同路径已存在时追加 `-2`、`-3`…

## Execution ID

```text
BL-<started-at>-<skill-slug>-<execution-suffix>
```

`<started-at>` 文件名形式：`20260622T153012+0800`（ISO 8601 去 `-` 与时区冒号）。

## 固定章节顺序

1. 标题 + 元数据表（含 `规范版本: behavior-log/v1`）
2. 顶层执行摘要
3. 执行边界
4. 依据与关键输入
5. 关键行为
6. 子 Skill 调用
7. 影响清单
8. 结果与验证
9. 偏离、异常与未决事项
10. 资源索引
11. 更正记录

## 生命周期

| 时机 | 动作 |
| --- | --- |
| 首次材料性行为前 | 创建日志，状态 `执行中`，写骨架与执行边界 |
| 材料性边界 | 增量更新；刷新「最后更新时间」 |
| 执行结束前 | 封口：终态摘要、影响闭包、验证、清占位符 |

## 材料性门槛（应记录）

- 改变持久化文件、外部系统、部署或用户可见状态
- 决定后续分支或关键方案
- 调用子 Skill 并使用其结果
- 使用决定性依据、用户确认、规范
- 失败、重试、回滚、降级、阻断
- 实际影响超出或偏离预期边界

## 通常不记录

- 无贡献的目录浏览、只读试探
- 每次只读命令及完整标准输出
- 模型内部推理
- 可通过引用获取的全文复制

## 子 Skill 协议

- 子 Skill 返回：`{结果, behaviorLogPath}`（及材料性影响摘要）
- 子日志不得含父 execution ID 或父日志路径
- 父 Skill 登记「子 Skill 调用」表并汇总材料性影响

## 判断测试
