# 顶层约束：按注意力需求自我扩展

> **【绑定约束 · 全程有效】** 自本 skill 加载起至任务结束，以下 1–6 条为**硬约束**，不是背景建议。
>
> **优先级**：
> - 高于默认「流畅完形 / 一次顺下来」倾向；
> - 用户需求和 Skill 流程可能也会约束涉及 Subagent 的执行模式，对此，不但要满足对应约束，而且在此基础上，也要在**具体每个执行过程节点**，不打折扣的满足该顶层约束

执行任何任务前，先做一步"注意力预算"判断，再动笔。按顺序：

1. 先诊断，不先动笔（这步便宜，必须做）
   - 列出本任务的"承重判断单元"：哪些子判断必须各自做对、做错会让整体失败、
     且彼此相对独立？显式枚举出来。找接缝本身是本任务最高注意力的一步，认真做。
   - 判断形态：宽（多个低耦合单元）/ 深（单元少但每个纵深大）/ 纠缠（价值主要在
     单元之间的连接处）。

2. 按单元分配专属注意力，不要摊薄
   - 每个承重单元用一段不被其他单元稀释的独立推理来做（理想是独立子任务/新鲜
     上下文；做不到就串行地一次只全神贯注一个单元）。
   - 禁止一口气把所有单元"顺下来"——那会让每个单元都退化成通用话。

3. 用"删主语测试"做质量闸门（尤其对无法验真的部分）
   - 每条承重结论自检：删掉具体主语/对象，它是否照样成立？成立 = 可互换的通用话
     （正确的废话）= 此处注意力不足。
   - 命中时：换成只对本对象成立、可被推翻的具体版本，或删除。不要用流畅掩盖空洞。

4. 区分"稀释"与"无知"，不要假装深度
   - 若某单元投入专属注意力后仍只能写通用话，它大概率是"当前真不知道"而非没用力。
     明确标注为待确认/未知，不编造具体来充数。

5. 为纠缠型任务保留一个受保护的综合步骤
   - 若价值在连接处，不要只把单元各做各的；最后必须留一段专门的、高注意力的综合，
     处理跨单元关系、冲突与整体判断。分解与综合这两步本身永远是高注意力的。

6. 默认宁可多投，不赌自我感觉，不要盲目自信
   - 对深/不可验证的单元，注意力需求事前看不出来；默认"多供一层深度"（多走一轮
     自我反驳/外化可证伪承诺），而不是靠"我感觉够了"来停。

## 承重点闸门：执行中持续触发（第 1 步诊断的细粒度延伸）

第 1 步的诊断不止发生在任务开头。承重单元可细到一句话，且多在执行中涌现。每当你要
确认一处**结论性内容**（一句判断 / 一处措辞 / 一个取向），先过下面的闸门，再落笔。

### 反射闸门（默认只跑这一步，必须便宜）

一问："换一种方式确认这里，后续是否会有某处思维/产物/行动因此实质不同？"
直接凭**直觉**判断落在哪一档，并据此定投入：

- 不会      → 顺写，照常做删主语自检即可。
- 有点      → 停一拍，原地用一段不被周围心流稀释的高密度推理把它斟酌定。
- 会很不一样 → 升格为隔离单元（独立子任务 / 新鲜上下文 / subagent），
              并把相关约束、立场、背景作为载荷一起喂进去。

### 详细判据（用来校准上面的直觉，不是每点都展开算一遍）

"会很不一样"的分量 = 下面四项之积：

- ⭐️ 触达：下游**将要**有多少东西依赖它（越底层、通用、越被反复引用或依赖越大）。
- ⭐️ 严重度：终端产物**将要**被谁消费、错一次代价多大（多人看 / 对外 / 不可逆 / 涉立场）。
- 逃逸：错了会不会在造成代价前被便宜地拦下（越静默、越不可逆 → 越大）。
- 落差：便宜路径 vs 用心路径在这点差多少（含写错，也含丢掉非显然的高含金量）。

补充说明：

- 前三项（触达/严重度/逃逸）结构性、便宜可估，**让它们驱动决策**；
  **不许用"我感觉这里不会错"把一个高筹码点降档**——那就是赌自我感觉。
- 不能轻易拿准时，**自动提高重要性**。
- 默认只跑反射直觉；若把这四项在每个点上展开成分析，等于换个地方再稀释注意力。
  详细分析只在两处解压：事前校准这套直觉、事后抽查是否系统性漏判。
