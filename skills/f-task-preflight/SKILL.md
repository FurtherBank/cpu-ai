# f-task-preflight：任务属性前置体检

## 核心理念

正式动手前，先花一步（很便宜）做「属性分析」：从任务里抽出关键属性，沿三条已知的高风险轴判定 AI 踩坑概率，再把结论落成**可执行的东西**——要澄清什么判断、要做什么动作、要向用户确认哪些问题、能否放行。

这是一道**防呆前置**，不是免责声明。它的职责是把“我决心把这件事做好、不出错”这种**劝诫式决断**，翻译成**正向的、可执行的动作与问题**（参考工作区「避免劝诫问题」原则：负向目标 → 必须执行的标准动作）。所以本 Skill 的产物永远是“要做 X / 要问 Y”，而不是“要小心 Z”。

适用面：接手 dima 工单、`i-todo` 路由前、或任何动手前的自检。产物可被后续 skill 直接消费。

## 何时跳过

任务**同时**满足以下条件时，可跳过 preflight，但要一句话说明跳过依据：
- 平凡、低风险、做错代价小；
- 所需事实都在通用知识或已给材料内（无私域/项目上下文依赖）；
- 目标已是结果级可验收、无开放价值判断；
- 不涉及时序/状态、不面向对外表达、不属于已知“第一反应易错”领域。

只要有一条不满足，就不要跳过对应那条轴的扫描。默认宁可多扫一轴，不赌“感觉够了”。

## 第一步：抽取任务属性

动手前先答这组属性问题（这是“属性分析”的本体，后续每条风险都必须挂回这里的某个属性作为证据）：

- **依赖来源**：完成它所需的关键事实，在通用知识/已给材料内，还是必须从这个项目/这个人的私域获取？是否需要现实反馈回路？
- **局部术语**：任务里的名词/缩写/概念，有没有可能在本情境带“专门含义”，与我的直觉理解不同？（点名 1–3 个高嫌疑词）
- **目标确定度**：用户给的是“结果级可验收目标”，还是“上层意图/方向”？我能否说清“做成什么样算好”？
- **决策权归属**：里面有没有价值/取舍/伦理类开放问题？哪些决策不该由 AI 独断？
- **交付去向**：产物交付到哪个环节、给谁用？那个环节是否自带未言明的约束？
- **领域形态**：是否涉及时序/状态机事实？是否属于“对外表达/聊天”而非“信息直出”？是否是已知/疑似“第一反应易错”领域？

属性抽完后，逐轴扫描。

---

## 接轨轴 G：上下文接轨

**本质**：事情做好严重依赖与现实情境的**有效交互**——项目上下文、私域知识依据、实验任务中与现实的反馈回路。AI 会默认用通用先验把缺口填上，并且**在看不见的地方失真**。最锋利的子风险：现实情境里描述问题/概念的“语言”形成**局部 DSL**，且与通用语言/逻辑/概念**冲突**——AI 用通用义“想当然”地解释局部术语，且通常意识不到自己错了。

**属性探针（命中即标 G 风险）**
- 关键事实是否落在私域、而非通用知识/已给材料里？
- 高嫌疑术语在本情境是否可能 ≠ 通用义？（逐个点名）
- 是否“做一版 → 看现实反馈 → 再改”的实验型任务？反馈通道是否存在？

**不处理会怎样**：用通用义误解局部术语 → **整篇方向性错误且自洽**（最危险，因为读起来是对的）；凭空补全私域事实 → 编造；缺反馈回路却当一次成。

**应对（澄清 / 动作 / 问用户）**
- 盘出「必须接轨的上下文清单」，逐项标 `已有 / 需自取 / 缺失`。
- 对高嫌疑术语：**先求证局部定义，禁止默认通用义**；定义不明就向用户/材料确认。
- 缺失且无法自助获取的事实，列为**必须向用户索取的输入**，不要先填空再开工。
- 实验型任务：显式约定反馈节奏，分步交付，不一次性产出。

---

## 方向轴 D：目标与方向

**本质**：用户给的多半只是“领导/老板位”的**上层目标**，没给“具体往哪走才算好/最优”的决策依据。两种加剧情形：领导对结果有更深要求、却懒得想清楚该是什么；目标本身开放（如伦理、取舍），需依赖多因素研讨判断（而这些因素的获取又可能依赖接轨轴）。此外：产物常**面向某个具体环节交付**，该环节自带大量约束，默认需要用户意识到并显式传入——而很可能没传。

**属性探针（命中即标 D 风险）**
- 给的是“结果级可验收目标”还是“上层意图/方向”？我能说清“做成什么样算好”吗？说不清 = 方向风险。
- 有没有价值/取舍/伦理类开放问题？这些是不是不该我替用户拍板？
- 产物要交付到哪个环节/给谁？该环节有没有未言明的约束（格式、口径、合规、受众、上下游衔接）？

**不处理会怎样**：朝一个“看似合理但非用户所要”的方向做完 → 返工；替用户做了不该 AI 做的价值/伦理裁决；产物漏掉交付环节的硬约束 → 不可用。

**应对（澄清 / 判断 / 问用户）**
- 把上层目标翻成「可验收的成功判据」草案，回灌给用户确认后再开工。
- 开放/价值/伦理问题：列为**需用户或研讨决定的决策点**，AI 给出选项与依据，不独断。
- 当判断「产物很可能面向带约束的环节」时：**主动提醒用户该环节可能存在未传入的约束，并询问之**（这是必做的提醒动作，不是可选项）。

## 反应轴 R：AI 第一反应陷阱

**本质**：在某些领域，AI 的**第一反应 / 流畅首答往往是错的**，且因为流畅而显得对。已知两类典型（**不止于此**）：

- **时序 / 状态机事实**：默认顺读会把“某句发没发、先后、因果”搞错——尤其当某部分状态没被明确声明时。例如复盘两人的聊天记录，未明状态会导致发送与否、顺序、作用关系全错。
- **对外表达 / 聊天**：真正的聊天表达（客服对客户、面向众多读者的分享文）**不等于**把相关信息直接透出去就完事。成熟表达要处理**立场**：在特定立场下目标必须服务于立场、特定立场下某些话/信息不能说、特定意图需专门引导、信息透露粒度需调整。

**通用识别（因为领域不止上面两类）**：凡满足以下任一，按反应轴对待——
- 正确答案依赖**文本表面看不见的隐藏状态**；
- “有帮助 / 把信息说全”的**本能与真实目标相冲突**；
- 有一个很强的直觉答案、但很可能是陷阱。

**属性探针**：这件事我脑中是不是已经有个“显然的答案”？它落在上面的隐藏状态 / 目标-本能冲突类型里吗？

**不处理会怎样**：用流畅首答替代正确推理——时序判断错、立场失当、把不该说的说了出去。

**应对（把首答降级为待证伪的假设）**
- **不信任首答**：把第一反应当“假设”，而非结论。
- 时序/状态类：**先显式建状态/时间线表**，每个事实在被落实前标“未知”，绝不从默认顺读推断发送/顺序/因果；必要时走专门设计的安全推理路线再下结论。
- 对外表达类：**动笔前先显式建模**——立场、受众、不可说信息、需引导的意图、信息透露粒度；让目标服务于立场，而非信息完整度。
- 对深/不可验证的判断，默认多走一轮自我反驳，不靠“我觉得够了”收手。

## 综合与放行

三条轴**不是独立的，会叠加、有先后依赖**，必须做一次专门的综合（不能只把三轴各报各的）：

- **叠加**：私域项目（G）+ 开放目标（D）+ 对外文章（R-表达）= 高复合风险，需逐项缓解后才动手。
- **顺序依赖**：方向（D）常依赖先把上下文接轨（G）——目标判据要基于私域事实才能定；反应轴的安全路线（R）要在拿到正确事实（G）后才有意义。综合时按 `G → D → R` 排澄清/动作的先后。

综合后给出**单一放行档位**：

- 🟢 **直接执行**：三轴均低风险（或已属可跳过任务）。
- 🟡 **带缓解执行**：有风险，但 AI 可用上述应对动作自缓解；明确列出所依赖的**假设**与自缓解动作，假设若被推翻需回头。
- 🔴 **先与用户对齐**：存在 AI 不应独断的决策（开放价值/伦理）、无法自助补齐的私域事实或交付环节约束、或说不清成功判据。此档**必须先把问题抛给用户**（用结构化提问），得到答复再开工。

```markdown
| 轴 | 风险 | 证据（挂回哪个属性） | 应对：澄清/动作/问用户 |
| --- | --- | --- | --- |
| G 接轨 | 有/无/不确定 | <具体属性> | <具体动作或问题> |
| D 方向 | 有/无/不确定 | <具体属性> | <具体动作或问题> |
| R 反应 | 有/无/不确定 | <具体属性> | <具体动作或问题> |

**放行档位**：🟢 / 🟡 / 🔴

**🟡 带缓解执行 → 我依赖的假设**：<列出，被推翻则回头>

**🔴 先与用户对齐 → 需要你确认/提供**：
1. <开放价值/伦理决策点，给选项与依据>

```

🔴 档位下，优先用结构化提问把上述问题一次性抛给用户。

## 自检闸门（删主语测试）

报告写完，对**每一条风险与每一条应对**做一次删主语测试：
- 删掉具体任务主语后，这句话是否照样成立？**成立 = 可互换的通用废话**（此处属性分析没做够），必须替换成只对本任务成立、可被推翻的具体版本，或删除。
- 区分「稀释」与「无知」：若某轴投入注意力后仍只能写通用话，多半是**当前真不知道**，标为“未知/待确认”，**不要编造具体内容来充数**。
- 每条风险都必须挂回「属性速览」里的某个具体属性；挂不上的风险，要么补属性，要么删掉。
