# 推导认知工作流

## 绑定产物

把**当前根节点**表示成一张从其输入认知状态到输出认知状态的有向图。图中的直接子节点必须是认知状态转换，边必须说明上下游语义如何被保持、细化、增加或退回。

一次调用默认只封口一层：

```text
当前根节点 L0 → 直接子节点 L1
```

不要在同一次调用中继续展开 `L1 → L2`。即使已经想到子节点内部可能包含什么，也只记录待注入的现实信息、展开触发条件和稳定节点路径。后续再次调用本 Skill，选定某个 `L1` 作为新的当前根节点，再根据届时的现实情境展开它的下一层。

部门、角色、文档、会议、模板、活动名和结果名不能直接充当节点。它们只能作为转换的载体、证据或消费接口。

本 Skill 默认推导认知工作流，不代替用户执行原任务。当前节点尚未展开到可执行粒度时，不要凭抽象主干直接执行其内部工作。

## 必读材料

开始推导前完整读取：

- [topology-derivation.md](references/topology-derivation.md)：怎样从当前根节点长出一层主干与分支。
- [state-and-edge-contracts.md](references/state-and-edge-contracts.md)：怎样记录当前根、直接子节点、边和跨节点义务。
- [validation-and-output.md](references/validation-and-output.md)：怎样校验当前层、停止展开并交接后续递归。

## 核心对象

**当前根节点**：本次调用唯一负责展开的任务或既有节点。它有稳定路径、上游输入边界、预期输出边界和下游消费者。

**层级路径**：节点在累计图中的稳定地址，例如 `root/N4X`。后续调用沿路径扩展，不因改名或换载体丢失来源和关系。

**认知状态**：在某一时点已经成立、可被后续依赖的信息系统。至少区分事实、推断、假设、决定与未知，并显式记录关键概念、关系、来源、决定权和消费者。

**认知转换节点**：把不足以下游消费的输入状态，经由明确认知动作，转化成更可用的输出状态。节点优先写成“把 A 转化为 B”或能唯一指向该转换的动宾短语。

**转换边契约**：规定上游语义进入下游时，什么必须守恒，什么允许下游原生增加，什么不能被静默改写，以及何时回开上游。

**跨节点义务**：不能由一个节点独立兑现、必须沿多个状态或边保持的性质，例如方向、可追溯、一致性或决定权。

**展开状态**：节点标为 `未展开`、`本层已展开`、`证据失效待重开` 或 `已达可执行粒度`。本层已展开不等于内部已经可执行。

## 调用模式

### 初始化任务根

没有既有图时，把用户的具体任务作为 `root`。本次只推导完成该任务所需的第一层认知工作流，达到此前“产品功能设计主干”那样的层次即可。

### 展开既有节点

用户提供既有图、节点路径或明确指向某个环节时，只展开该节点。继承：

- 上层目标、主要取舍和不可牺牲项。
- 父节点输入/输出边界及兄弟节点关系。
- 上游已经守恒到此的概念、决定、假设与未知。
- 下游消费者期待的输入语义。
- 已存在的跨节点义务、证据与回开条件。

再注入本次新增的现实事实、执行反馈、领域约束和决定。不要从任务根重新生成整张图，也不要顺便展开兄弟节点。

### 审计或修复既有层

用户要求检查既有图时，只攻击指定层的节点、边与边界。若发现父层定义错误，标明回开位置；不要借审计无授权地重写整棵树。

## 单层推导流程

### 冻结当前根契约

先写明：

- 当前根路径、它解决的现实对象及本次展开边界。
- 上游输入状态、下游消费者与预期输出状态。
- 上层目标、取舍、不可牺牲项和决定权。
- 本次新增现实信息、外部依赖、未知和明确范围外。

目标方向不够时，不要凭通用最佳实践替用户决定。可逆且不改变本层拓扑的内容可标假设继续；会改变本层主干或不可牺牲项的内容先校准。

### 建立当前根的两端状态

分别描述当前根开始前已经成立什么，以及它完成后下游必须能直接依赖什么。如果下游仍需补做本节点应负责的核心判断，输出边界仍不闭合。

不要先列阶段。直接子节点只能从当前根两端之间不可跳过的状态差长出来。

### 独立生成本层候选

对当前根做三次互不替代的推导：

- **反向前提链**：从当前根输出边界回推必要前态。
- **正向可达链**：从当前根输入状态前推合法认知增量。
- **失败反演链**：从“当前根形式完成但下游仍失败”反推缺失转换、断边或静默决定。

复杂任务可分别交给不同无答案上下文执行者。候选只允许到直接子节点粒度，不展开候选内部子图。

### 封口本层拓扑

判断候选之间是等价、层级、依赖、并行、汇合、条件分流、反馈或冲突。只保留删除后会破坏当前根输出边界、或迫使下游补核心判断的承重节点。

为直接子节点分配稳定路径，并检查它们组合完成后是否自然产生当前根的输出状态。若某个候选其实属于子节点内部，把它挂入该节点的“后续展开线索”，不提升到本层。

### 建立本层心智索引

按 [state-and-edge-contracts.md](references/state-and-edge-contracts.md) 为每个直接子节点填写简明索引卡，只确认本层所需的输入边界、认知增量、输出边界、依赖、证据门与回开方向。

不要填写每个子节点的完整内部操作步骤，也不要生成孙节点。需要进一步回答“这个节点内部具体怎样做”的问题，进入后续展开队列。

### 显式化本层关系

只为承重边记录语义守恒、下游原生增量、禁止静默决定和回开条件。把事实规律、领域标准、风险和质量要求附着到当前层节点、边或跨节点义务；无法附着的内容要么揭示本层漏项，要么留给节点后续展开。

通用框架只能用于发问和攻击遗漏，不能直接决定当前层拓扑。

### 攻击当前层并停止

执行 [validation-and-output.md](references/validation-and-output.md) 的单层校验。当前根的一层拓扑连续、直接子节点边界稳定、承重关系闭合、后续展开队列清楚时，停止本次调用。

不得因为子节点仍未达到执行粒度而继续递归。只声明“当前层暂定闭合”，列出尚未展开节点、下一次调用需要的现实信息和失效重开条件。

## 后续调用交接

每个待展开节点必须能直接成为下一次调用输入，至少交接：

- 稳定路径和展开状态。
- 从父层继承的输入/输出边界。
- 与兄弟节点、承重边和跨节点义务的关系。
- 本层已确认的事实、决定、假设、未知与证据。
- 下一层推导前应取得的现实材料或反馈。
- 触发重开父层而不是继续下钻的条件。

新增现实发现若推翻父层边界，先回开父层；不要在错误父节点下继续精细展开。

## Subagent 派发约束

当使用 Subagent 时，一次只派发一个本层候选透镜、承重边或层级攻击，不把多个待确认单元混成“综合想想”。派发中写明当前根契约、允许来源、禁止假定、深度边界、返回格式和写入路径。

实验隔离使用 fresh context，只传 Skill 与任务基准，不传答案键、前序样本或主控结论。主控先做单点关系判断，再整合本层图。

## 禁止退化

- 不以常见生命周期、部门分工或行业模板开局。
- 不把活动名、角色名或交付物名直接当作认知节点。
- 不用更长清单替代状态、动作、关系与证据。
- 不把本层已展开写成整个任务已经可执行或绝对完整。
- 不在一次默认调用中展开多个层级或顺带展开兄弟节点。
- 不在缺少现实情境时预先填满子节点内部细节。
- 不让下游借“细化”之名重新决定上游已封口语义。
