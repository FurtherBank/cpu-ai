# 状态、节点与边契约

## 认知状态的最小结构

每个承重状态按需包含：

- **对象与概念**：讨论的实体、术语及边界。
- **关系与动力**：依赖、因果、顺序、约束、冲突、反馈。
- **认识层级**：事实、推断、假设、决定、未知分别标记。
- **来源与权威**：内容来自哪里，谁有权确认或推翻。
- **适用条件**：情境、时间、角色、范围和例外。
- **消费者与用途**：谁将依赖它做什么判断或行动。
- **生命周期**：何时形成、怎样验证、何时失效或重开。

不是每个状态都需要一份大表，但缺失项若会改变后续路线，必须显式化。

## 节点契约

每个认知转换节点使用以下形状：

```markdown
### <把输入状态转化为输出状态的动宾名称>

- 目标增量：完成后新增什么可依赖能力。
- 输入状态：开始前必须成立什么；事实/假设/未知如何分层。
- 核心认知动作：观察、区分、建模、生成、映射、权衡、选择、模拟、验证、修正或封口什么。
- 操作顺序与分支：哪些动作有真实依赖，何时并行或回退。
- 守恒约束：转换中不能丢失或偷换什么。
- 输出状态：形成哪些概念、关系、决定、边界和剩余未知。
- 完成证据：哪种观察、追溯、反例或下游试用证明状态可消费。
- 下游消费者：谁用它继续做什么；不应再替本节点补什么判断。
- 失败路由：什么失败回到本节点内部，什么失败必须重开上游。
- 载体映射：可用哪些文档、图、代码、会议或数据记录状态与证据。
```

### 核心认知动作的粒度

动作要能指导注意力怎样发生变化。仅写“分析、设计、处理、保障质量”通常太空；需要说明分析什么关系、由什么约束生成什么、用什么现实切片验证什么耦合。

一个节点若包含多个会让后续走向明显不同的确认点，把它展开成子图或条件分支。一个叶节点只有在执行者不必现场发明新的领域判断、价值取舍或完成标准时，才算可执行。

## 转换边契约

上下游不是“把文档交过去”，而是对语义做受约束的变换。每条承重边填写：

```markdown
#### <上游状态> → <下游状态>

- 来源语义：下游实际消费上游的哪些概念、关系、规则、决定和未知。
- 转换操作：选择、组合、拆分、投影、翻译、细化或约束什么。
- 必须守恒：变换后仍必须为真的意义、目标和边界。
- 下游原生增量：为解决下游自己的问题，允许新增哪些判断。
- 禁止静默决定：哪些变化看似是细化，实则重定义了上游语义或越过决定权。
- 可追溯关系：如何从下游元素回指来源，从上游义务追到所有下游落点。
- 回开条件：不可实现、互相矛盾、证据失败或来源过期时，回到哪个节点由谁重决。
```

不要假设映射是一对一。一个上游概念可能投影到多个下游结构，多个上游关系也可能共同决定一个下游结构。完备性至少同时检查：

- **正向覆盖**：每项承重上游语义都有必要的下游落点或明确的不适用理由。
- **反向有据**：每个承重下游决定都能回到上游需求、下游原生问题或外部约束。
- **交叉一致**：不同投影对同一来源语义没有互相冲突。

## 跨节点义务

跨节点义务不能另列一份无人负责的“非功能清单”。对每项义务写明：

- 来源和适用条件。
- 首次进入图的位置。
- 哪些节点生成、细化、验证或消费它。
- 沿哪些边必须守恒。
- 最终证据和失败回开位置。

若一项义务在多个节点各自有局部表现，用一条贯穿关系把它们连接起来，不要重复成互不相干的条目。

## 载体的正确位置

文档模板、表格、图、代码、会议和检查表只承担三种作用：

- 记录已经形成的认知状态。
- 为认知动作提供外部操作界面。
- 向下游呈现完成证据或可消费接口。

如果去掉某种载体，认知转换仍可用别的方式完成，则载体不是节点本体。反过来，如果填满载体仍不能证明状态变化，不能判定完成。
