# 当前根、心智索引与边契约

## 当前根契约

每次调用先固定唯一当前根：

```markdown
- 当前根路径：
- 展开模式：初始化 / 展开既有节点 / 审计修复
- 从父层继承的输入状态：
- 本节点承诺的输出状态：
- 下游消费者与用途：
- 上层目标、取舍与不可牺牲项：
- 本次新增现实事实或反馈：
- 事实 / 推断 / 假设 / 决定 / 未知：
- 本次只负责：当前根 → 直接子节点
```

展开既有节点时，不重新解释上层目标。新增信息若与父层冲突，先标记父层重开，不在失效边界下继续下钻。

## 当前层心智索引卡

默认只给每个直接子节点填写一张简明索引卡：

```markdown
### <稳定路径> <把输入状态转化为输出状态的动宾名称>

- 本层作用：它为父节点增加什么不可替代的认知状态。
- 输入边界：从父层或兄弟节点接收什么；哪些仍是假设或未知。
- 核心认知动作：观察、区分、建模、生成、映射、权衡、选择、模拟、验证或修正什么。
- 输出边界：形成什么可被下一节点依赖的概念、关系、决定或证据。
- 本层关系：前置、并行、汇合、条件分流或反馈到哪里。
- 放行信号：怎样知道输出边界足以交给相邻节点，不要求内部已经完全执行。
- 回开方向：哪类失败说明本节点、上游或父层定义有误。
- 后续展开：内部仍有哪些承重问题；下一次调用需注入什么现实信息。
- 展开状态：未展开 / 本层已展开 / 证据失效待重开 / 已达可执行粒度。
```

动作名称要说明注意力怎样变化。仅写“分析、设计、处理、保障质量”通常太空；写清分析什么关系、由什么约束生成什么、用什么现实切片验证什么。

如果一个索引卡内部仍藏有多个会改变走向的确认点，把它标为“后续展开”，不要在本次调用中生成孙节点。

## 本层转换边契约

上下游不是交文档，而是受约束地变换语义。只对影响走向的承重边填写：

```markdown
#### <上游路径> → <下游路径>

- 来源语义：下游实际消费哪些概念、关系、决定、假设和未知。
- 必须守恒：变换后仍必须为真的目标、意义和边界。
- 下游原生增量：下游为解决自己的问题允许新增什么判断。
- 禁止静默决定：哪些变化实质重定义上游语义或越过决定权。
- 回开条件：不可实现、互相矛盾、证据失败或来源过期时回到哪里。
```

必要时补充多对多追溯：

- **正向覆盖**：每项承重上游语义都有必要落点或不适用理由。
- **反向有据**：每项承重下游决定都有上游、下游原生问题或外部约束来源。
- **交叉一致**：同一来源语义的不同投影不互相冲突。

## 本层跨节点义务

跨节点义务不能成为无人负责的附加清单。当前层只记录：

- 来源与适用条件。
- 首次进入的直接子节点。
- 在当前层经过哪些节点与边。
- 本层必须守恒的意义。
- 本层证据门和失败回开位置。
- 向后续展开节点传递的义务。

不要在本次调用中预先设计该义务在所有后代节点中的具体实现。

## 载体的正确位置

文档、表格、图、代码、会议和检查表只承担三种作用：

- 记录已经形成的认知状态。
- 为认知动作提供外部操作界面。
- 向下游呈现证据或可消费接口。

填满载体不能证明状态转换完成；更换载体也不改变节点身份。
