# 校验与输出

## 叶节点停止条件

只有同时满足以下条件，才停止递归展开：

- 输入状态可获得，且未知不被伪装成事实。
- 核心动作能被执行，不依赖“凭经验把它做好”之类隐藏步骤。
- 动作内部不再藏有会改变后续路线的未分配判断。
- 输出是可被下游直接依赖的认知状态，不只是一个载体。
- 有完成证据、消费者与失败路由。
- 执行者不必重新决定上层目标、领域语义或不可牺牲项。

不能继续可靠展开但又不满足条件时，标为阻塞未知，不用常识补齐。

## 全图攻击

### 结构攻击

- 删除每个节点，观察谁被迫补做其核心判断。
- 交换相邻节点，验证依赖是否真实。
- 找只有活动没有认知增量的空转节点。
- 找只有标题连接、没有类型连续性的断边。
- 检查条件分支是否被错误压成唯一顺序。
- 检查反馈回路是否说明了回开原因与决定权。

### 语义攻击

- 从终点逐项追到来源，检查概念、关系、目标和边界是否守恒。
- 从每个来源义务向前追到全部落点，检查是否遗漏或被单点化。
- 搜索下游“合理补充”中实际发生的上游重定义。
- 检查事实、推断、假设、决定和未知是否混层。
- 检查同一概念在不同节点是否发生无声明漂移。

### 执行攻击

- 让一个不了解推导过程的执行者仅凭节点契约描述下一动作、所需输入和完成证据。
- 让下游消费者试用输出，记录它还必须猜、补、重建或重新决定什么。
- 用一个主路径外的现实切片攻击边界、例外、失败、恢复和回滚。
- 反问“所有框都打勾但任务仍失败”可能怎样发生，并定位到图上的缺口。

### 方向攻击

- 图的终点是否服务原始消费者和场景，而不是以“产物已生成”结束。
- 局部优化是否可能牺牲上层目标、主要取舍或不可牺牲项。
- 目标仍含糊时，图是否把猜测标成假设并保留校准门。

## 完备性声明

不要用“覆盖了很多类别”证明完整。暂定完备只表示：

- 在已知任务边界与来源下，起点到终点存在连续的合法转换路径。
- 每个终态义务都能追到生成、守恒和验证位置。
- 每个节点与边都有依据、消费者、证据和失败路由。
- 已知强反例没有制造无人负责的缺口。
- 未知、外部依赖与失效重开条件被显式保留。

新增现实事实、目标变化、消费者变化、下游不可实现或验证失败时，重新打开相关节点与边。

## 固定交付结构

```markdown
# <任务>的认知工作流

## 暂定结论与适用边界

## 任务契约
- 起点：
- 终点与消费者：
- 上层目标与取舍：
- 不可牺牲项：
- 范围外：
- 假设与未知：

## 主干图
<Mermaid，节点写认知转换或输出状态，显式画分支、汇合、门与回改边>

## 主干为什么成立
<每个承重节点的任务来源、删除后果、与相邻节点的关系>

## 节点子图与契约
<按认知依赖展开，不按文档目录展开>

## 承重边契约
<守恒、下游原生增量、禁止静默决定、追溯、回开>

## 跨节点义务
<来源、贯穿关系、证据、失败位置>

## 载体与产物映射
<载体怎样记录状态、支持动作或提供证据；不把载体当节点>

## 完备性攻击结果
<删除、换序、断边、消费者试用、强反例>

## 未知与失效重开条件
```

输出先给主干与关系，再给节点细节。细节必须能回挂到节点、边或跨节点义务；不能回挂的零散最佳实践不要进入“完备工作流”。
