# 单层校验与递归交接

## 单次展开停止条件

同时满足以下条件就停止本次调用：

- 当前根的输入与输出边界已经明确，路线级未知没有伪装成事实。
- 直接子节点组合后能自然产生当前根输出状态。
- 每个直接子节点都有不可替代的认知增量、稳定路径和相邻关系。
- 承重边说明了守恒、允许新增、禁止静默决定和回开。
- 当前层跨节点义务有进入点、守恒路径和证据门。
- 每个未展开子节点都记录下一次调用所需的现实信息与展开问题。
- 图中没有生成直接子节点的内部子图或孙节点。

这表示“当前层暂定闭合”，不表示所有子节点已经可执行，更不表示整个任务绝对完整。

## 当前层攻击

### 结构攻击

- 删除每个直接子节点，观察父节点输出是否仍成立，或谁被迫补做其核心判断。
- 交换相邻节点，验证依赖是否真实；无影响时检查伪顺序。
- 找只有活动名称、没有认知增量的空转节点。
- 找只有标题连接、没有输入输出连续性的断边。
- 检查分叉、汇合、条件门和反馈是否被错误压成单链。
- 检查某个候选是否其实属于子节点内部，被错误提升到当前层。

### 语义攻击

- 从当前根输出边界逐项回到直接子节点与来源。
- 从当前根输入义务向前追到当前层所有必要落点。
- 搜索下游合理细化中发生的上游静默重定义。
- 检查事实、推断、假设、决定和未知是否混层。
- 检查同一概念在当前层是否无声明漂移。

### 深度攻击

- 搜索是否已经给某个直接子节点写出完整内部步骤或孙节点。
- 搜索是否为了显得完整，提前填入尚无现实来源的领域细节。
- 搜索是否把兄弟节点的内部问题顺带展开。
- 搜索是否把“内部尚待展开”误判为当前层不完整，从而无限下钻。

### 方向与消费攻击

- 当前根输出是否服务它的真实下游消费者，而不是以载体已生成结束。
- 局部节点是否可能牺牲上层目标、主要取舍或不可牺牲项。
- 下游是否仍需重建本层本应完成的主干判断。
- 新现实是否已经推翻父层边界；若是，先回开父层。

## 分层完整性声明

不要用覆盖很多类别证明完整。单次调用只允许声明：

- 在当前根边界与已知来源下，输入到输出存在一层连续的合法转换拓扑。
- 当前层义务能追到直接子节点、承重边与证据门。
- 当前层强反例没有制造无人负责的主干缺口。
- 子节点内部未知、待注入现实信息和后续展开路径已经显式保留。

整个任务只有在后续多次调用把必要节点逐层展开到可执行粒度，并且累计图通过来源、语义、证据和消费者校验后，才能声明“当前版本可执行闭合”。

## 固定交付结构

```markdown
# <当前根>的一层认知工作流

## 当前层结论与边界
<本次只展开哪一个根、到哪一层、不能据此声称什么>

## 当前根契约
- 稳定路径：
- 上游输入状态：
- 下游输出状态与消费者：
- 上层目标、取舍与不可牺牲项：
- 新增现实信息：
- 假设、未知与范围外：

## 一层主干图
<Mermaid，只画当前根的直接子节点、分支、汇合、门与回改>

## 本层心智索引
<每个直接子节点的本层作用、输入、认知动作、输出、关系、放行信号、回开和后续展开>

## 承重边与跨节点义务
<只写会改变当前层走向或语义的关系>

## 当前层校验结果
<删除、换序、断边、深度越界、消费者偷工和强反例>

## 后续展开队列
<稳定路径、优先触发条件、所需现实材料、父层重开条件>

## 当前未知与失效重开条件
```

默认不要输出三条候选推导的全部草稿、每个节点的完整执行契约、所有可能载体或后代细节。只保留足以解释本层主干为何成立、并能支持下一次调用的高承重信息。
