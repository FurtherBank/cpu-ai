# 主干拓扑推导

## 任务来源，不是现成目录

主干必须同时受以下来源约束：

| 来源 | 提供什么 | 不能直接提供什么 |
| --- | --- | --- |
| 上层目标与方向 | 为什么做、冲突时优先什么 | 具体怎样做 |
| 现实对象与当前状态 | 起点事实、因果、能力和限制 | 理想终态 |
| 最终消费者 | 结果要支持的行动与判断 | 上游如何形成结果 |
| 领域规律与外部规则 | 不可违反的事实、标准、风险 | 任务特有拓扑 |
| 失败样本与近敌 | 容易出现的断点、伪完成 | 唯一正确顺序 |
| 下游反馈 | 上游状态是否可消费、是否可实现 | 静默重写上游语义的权力 |

通用方法论与同类案例只用于生成候选问题或攻击图。候选若无法连接到上述任务来源，不得因为“通常如此”进入主干。

## 从状态差长出节点

对每一处起点—终点差异，依次问：

1. 终点中的哪个可用状态当前还不存在？
2. 哪个认知动作能合法地产生它？
3. 该动作开始前必须依赖哪些已成立状态？
4. 这些依赖当前是否存在；若不存在，还需要哪个转换？
5. 这个状态怎样被观察、反驳或由下游试用？

得到的是依赖图，不预设是直线：

- 多个前提共同支持一个状态时使用汇合。
- 同一前提可独立推进多个状态时使用分叉。
- 不同情境需要不同动作时使用条件分流。
- 下游证据会否定上游假设时使用反馈边。
- 只有在某状态达到证据关口后才允许继续时使用门。

## 三种独立透镜

### 反向前提链

从最终消费者的可行动状态逐层回推必要前提。每一层都写“若没有它，下游必须补做什么判断或会发生什么失败”。停止于当前已可靠成立的状态。

这条链擅长发现结果闭包，但可能低估现实迁移成本。

### 正向可达链

从当前事实出发，只允许增加已有来源支持的认知状态。每次写明输入、操作和新获得的下游能力。不能跨过未知、冲突或决定权边界。

这条链擅长保证可执行，但可能停在局部可达而非真正终态。

### 失败反演链

至少攻击这些伪完成：

- 有交付物但没有形成可依赖判断。
- 局部都合理但上下游语义断裂。
- 下游为继续工作静默补了上游决定。
- 做了活动但认知状态没有变化。
- 主路径可用，例外、恢复或边界一触即溃。
- 达成局部指标却背离上层方向或不可牺牲项。

从失败机理反推缺失节点、断边、证据关口或跨节点义务。

## 主干整合规则

对候选先判断关系，再决定结构：

- **等价**：输入、核心动作和输出实质相同，合并别名。
- **层级**：一个候选是另一个转换内部不可跳过的子状态，放入子图。
- **依赖**：B 的合法输入必须使用 A 的输出，建立 A → B。
- **并行互补**：两者独立生成不同前提，之后汇合。
- **条件分流**：现实状态不同导致合法动作不同，保留条件边。
- **反馈**：后置证据能否定前态，建立带原因的回开边。
- **冲突**：两种确认方式会让后续走向不同，保留冲突并请求决定权来源。

不要按篇幅或视觉整齐强制 MECE。认知状态可能被多节点共同约束；追求的是依赖闭合、语义不重不漏和可执行，不是目录形式上的互斥。

## 主干封口测试

- **端点测试**：起点与终点是否分别落到现实和消费者，而不是任意生命周期边界。
- **删除测试**：删除节点后，终点是否仍能成立且无人补做它的核心判断；若能，节点可能不承重。
- **空转测试**：执行节点动作后，输出状态是否真的增加了可依赖信息或能力。
- **换序测试**：交换相邻节点会不会违反输入前提；不会则检查伪顺序。
- **桥接测试**：每条边的输出类型是否真能成为下游输入；名字顺口不算连续。
- **替代路径测试**：是否存在同样合法的路径；有则表达成条件或候选，不伪装成唯一链。
- **消费者偷工测试**：最终消费者是否仍需重建概念、补决定、猜边界或发明验收口径。
