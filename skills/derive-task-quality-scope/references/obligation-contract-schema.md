# 质量义务契约 Schema

## 必填字段

| 字段 | 说明 |
| --- | --- |
| Obligation ID | 稳定唯一，如 `QO-01` |
| 名称 | 短且可区分，不用“关注 X” |
| 来源 | 候选 ID、来源 ID、现实模型锚点 |
| 触发条件 | 哪种对象、状态或事件使义务生效 |
| 义务主张 | 必须使什么现实状态成立 |
| 证据要求 | 什么证据有资格支持三态判断 |
| 维持期限 | 成立到何时、绑定哪个版本/环境 |
| 依赖 | 前置义务、关系、权利或来源 |
| 失效/重开 | 哪种变化自动令义务重新待办 |
| 状态判据 | 分别写明什么现实状态与证据组合构成 `OK`、`NOT_OK`、`INDETERMINATE` |
| 当前状态 | `OK / NOT_OK / INDETERMINATE` |
| 未知与阻断 | 缺什么、阻断什么动作、向谁取得 |

## 原子性测试

尝试构造义务内部两个主张一真一假：

- 能构造：拆成两个义务或提升为显式联合义务。
- 不能构造且共同变化：可以保留。

逐个检查由“和、以及、或、/、与”、逗号枚举或集合名隐藏的子主张。合同、渠道、地区、用途、能力维度、控制动作等只要可以独立真/假，就必须：

- 一实例一义务；或
- 建立义务族，并让每个实例拥有独立 ID、证据、三态和重开条件。

不得仅给聚合行一个“取最差状态”的总状态来掩盖未建模子主张；聚合只能放入闭包或联合义务。

## 义务准入

以下内容不能直接成为义务：

- 单个质量属性词。
- “已执行某动作”。
- “已签认、已通知、已检查、已批准”等程序事实，但没有写出程序必须使哪个现实状态成立。
- 没有证据资格的愿望。
- “检查未知风险”等不可判真万能项。
- 没有现实承载点或进入理由的通用原则。

## 三态规则

- `OK`：触发义务已由当前有效证据兑现。
- `NOT_OK`：存在反证或硬条件失败。
- `INDETERMINATE`：缺证据、来源冲突、超界、无权裁决或评价方法无效。

不得通过降低措辞强度把 `INDETERMINATE` 改成 `OK`。

每项必须单独写出三条状态判据，不能只引用本节通则。存在“满足义务或获得例外”时，例外也必须成为独立、可授权、可失效的义务，并明确不得接受任务契约中的硬护栏或不可接受损失；否则原义务保持 `NOT_OK` 或 `INDETERMINATE`。
