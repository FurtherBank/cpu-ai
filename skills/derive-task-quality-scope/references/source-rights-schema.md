# 来源与权利地图 Schema

## 来源记录

| 字段 | 要求 |
| --- | --- |
| Source ID | 稳定唯一，如 `SRC-01` |
| 来源对象 | 文件、系统、接口、人员角色、标准、运行数据或现实反馈 |
| 支撑主张 | 具体主张 ID，不写“提供背景” |
| 权威范围 | 能确认什么，不能确认什么 |
| 事实层级 | 实证 / 推断线索 / 未验证陈述 |
| 版本与时间 | 文件版本、系统版本、查询时段、获取日期 |
| 访问与权限 | 只读/写入、敏感级别、有权持有人 |
| 替代路径 | 受托查询、现场核验、签认摘要或其它方式 |
| 失效触发 | 什么变化后必须重新读取或确认 |

## 权利记录

至少逐项判断：

- 事实确认权：谁能确认事实状态。
- 语义裁决权：谁能解释业务规则或领域含义。
- 方案决策权：谁选择行动或替代方案。
- 执行权：谁可改变现实状态。
- 停止/回退权：谁能叫停、隔离、回滚或降级。
- 披露权：谁决定信息能否对外表达。
- 残余风险接受权：谁有权接受未消除风险。

## 完成检查

- 每个关键主张都有首选来源或具名缺口。
- 每个来源反向链接其支撑主张。
- 参与者、专家、材料作者和有权裁决者没有混同。
- 动态来源记录日期、版本和重查触发器。
- 来源冲突保留为冲突组，没有被多数意见抹平。
- 敏感材料只记录稳定引用和完成判断所需最小内容。
