# 质量范围对抗测试

## 核心攻击

寻找：

> 所有现有义务和证据都显示完成，但整体任务仍然失败。

## 失败族

- 边界与受影响者：遗漏主体、外部性、上下游或真实使用情境。
- 状态与接口：局部正确但组合、顺序、共享资源或传播失败；在途工作、延迟回调、持久化产物或历史副作用越过状态改变点。
- 时间与版本：绿灯来自不同快照，后续变化推翻历史完成。
- 权利与治理：无权角色确认事实、披露、执行、停止或风险接受。
- 测量与推理：指标测错构念、证据失真、证据为真却推不出主张。
- 反馈与监测：无告警来自探测器失效，长期目标已漂移。
- 极端与异常：规模、边界输入、低频严重事件、不可逆恢复。

## Defeater 分型

- 反驳主张：直接构造整体失败。
- 削弱证据：证据版本、样本、来源或观察机制不可信。
- 切断推理：即使证据为真，也不足以支持整体主张。

## 发现后的最小修订

| 发现 | 回退位置 |
| --- | --- |
| 任务目标、边界或风险政策错误 | 任务契约 |
| 现实机制、主体或状态遗漏 | 任务现实模型 |
| 必要义务完全缺失 | 多源候选义务 |
| 义务定义过弱或不可判定 | 义务契约化 |
| 局部正确但组合失败 | 组合闭包 |
| 证据无效、无独立性或关口不阻断 | 证据与关口 |
| 已有义务只是未被执行 | 原任务执行控制层，不修改范围 |

修订后重跑所有受影响校验与独立测试。
