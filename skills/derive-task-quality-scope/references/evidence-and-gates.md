# 证据与关口

## 证据层级

| 证据类型 | 能支持什么 | 不能自动支持什么 |
| --- | --- | --- |
| 分析/形式推导 | 在模型和前提内满足某条件 | 规格是否代表真实目标 |
| 检查 | 静态事实、结构、配置或文档符合性 | 动态行为和真实使用效果 |
| 演示 | 特定操作路径可执行 | 边界、规模、长期稳定性 |
| 测试 | 已定义用例/环境中的行为 | 未建模场景和分布漂移 |
| 真实使用反馈 | 观察窗口内的实际影响 | 不可观测、低频、未来或超界失败 |
| 独立评价 | 降低自证偏误 | 若共享模型/数据，仍可能共享盲区 |

## 证据记录字段

- Evidence ID、支撑义务/关系 ID。
- 对象、方法、来源、执行者和评价者。
- 系统/制品/规则版本、环境、样本和时间窗。
- 预期、实际、判定和原始证据位置。
- 指标构念、分母、阈值依据、偏差和不确定性。
- 证明域、泛化边界和有效期。
- 与其它证据共享的假设、数据、评分器或失败模式。
- 什么事件使证据失效。

## 关口规则

每个关口必须写：

- 允许动作和禁止动作。
- 必须 `OK` 的义务与关系。
- 任一 `NOT_OK` 的处置。
- `INDETERMINATE` 的阻断、责任来源和补证动作。
- 谁有权作出 go/no-go 或残余风险接受。
- 版本/时间变化后的重开规则。

## 关键定理

证据结论强度不得超过证明域。形式证明、覆盖率、通过率、零事故或多个一致评分都不能跨越其模型、样本、观察和评价条件。
