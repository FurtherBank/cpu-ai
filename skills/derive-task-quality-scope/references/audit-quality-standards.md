# derive-task-quality-scope：行为日志可比对质量标准

## 结构完整性

- [ ] 日志位于 `.skill-logs/derive-task-quality-scope/yyyy-mm-dd/hhmm-*.md`
- [ ] 元数据含 `规范版本: behavior-log/v1`、执行 ID、Skill、状态、时间、工作区、执行目标
- [ ] 11 个固定章节齐全且顺序正确
- [ ] 封口后无“执行中，待收敛”占位符

## 阶段覆盖（Audit Hook）

| 阶段 | 日志须更新的章节 | 完成判据 |
| --- | --- | --- |
| 冻结任务契约 | 执行边界、依据与关键输入、关键行为、资源索引 | 目标、边界、初始未知、承重判断和契约路径可追溯 |
| 建立来源与权利地图 | 依据与关键输入、关键行为、偏离/异常、资源索引 | 决定性来源、权限受阻、替代取证和权利判断有记录 |
| 恢复任务现实模型 | 关键行为、子 Skill 调用、偏离/异常、资源索引 | 任务形态、事实冲突、模型版本和分域交接可追溯 |
| 生成多源候选义务 | 关键行为、子 Skill 调用、偏离/异常 | 每个生成源输入输出、淘汰和去重决定有记录 |
| 契约化质量义务 | 关键行为、子 Skill 调用、影响清单 | 定义、拆分/合并、删改、状态与单点封口有记录 |
| 建立组合闭包 | 关键行为、影响清单、偏离/异常 | 关系、不变量、权衡、完成层级和未闭合关系有记录 |
| 设计证据与裁决关口 | 关键行为、结果与验证、偏离/异常 | 证据选择、有效性、替代证据、关口和阻断未知有记录 |
| 独立挑战并封口 | 子 Skill 调用、影响清单、结果与验证、资源索引、更正记录 | 测试、失败分类、修订、最终裁决、残余风险和重开条件闭合 |

## 子 Skill 链

- [ ] 每个显式 Skill 子调用在「子 Skill 调用」表登记
- [ ] 子日志链接可访问且 workspace-relative
- [ ] 纯 Task 的派发、返回路径和 `auditSummary` 在父日志可查
- [ ] 子任务材料性影响已汇总进父「影响清单」

## 影响闭包

- [ ] 影响清单覆盖本次质量范围包全部创建、修改、重开和封口
- [ ] 明确记录未修改原任务制品或外部系统，除非用户另行授权
- [ ] 顶层摘要覆盖整个执行链的主要判断、产物和风险

## 与交付物对齐

- [ ] 日志资源索引链接 `quality-scope-package/` 和所有测试报告
- [ ] 日志只记录过程与路径，不复制质量范围正文
- [ ] `quality-scope.md` 的最终裁决、证明域、有效期与日志「结果与验证」一致
- [ ] `INDETERMINATE` 阻断项和残余风险在日志与交付物间无矛盾
