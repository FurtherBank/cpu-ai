## 核心定义

**任务模型**由任务类型、具体输入、整体目标、责任边界、环境、时间窗、风险政策、利益相关者和可观测条件组成。任何一项变化都可能使已有质量范围失效。

**质量范围**不是属性词集合，而是：

```text
质量义务集合
+ 义务之间的关系与全局不变量
+ 实例化、取证、裁决和失效重开的规则
+ 适用性门槛
```

**质量义务**是可单独记账的待兑现主张。局部不等于独立；一个义务可以检查多个对象的联合状态。

**整体 OK**表示：在声明的边界、环境和时间窗内，任务目标成立，且不可接受损失与残余风险满足有权角色确认的政策。

## 完整性的条件表达

完整性是质量范围相对于任务模型的关系属性，不是列表自身属性。

正向判据：

```text
所有触发义务均 OK
+ 关系约束与全局不变量成立
+ 无阻断 UNKNOWN
⇒ 整体 OK
```

反向判据：

```text
每一种证明域内的整体失败
⇒ 至少命中一个未完成义务、失效关系或超界条件
```

要让这两个判据可执行，还需要：

- 任务语义固定，变化时重开。
- 条件义务能正确触发。
- 跨项耦合、时序、共享状态和反馈已进入闭包。
- `OK` 有有效证据，`UNKNOWN` 不默认通过。
- 后续变化会传播失效。
- 义务集合存在共同可达状态，避免真空充分。

## 多源正向生成

不要寻找更大的单轴 MECE 清单。分别使用以下生成源，再保留来源去重：

| 生成源 | 主要问题 | 产生的候选 |
| --- | --- | --- |
| 边界、利益相关者与价值 | 为谁、为何、谁受影响、谁有权 | 价值义务、外部性、权利、承诺和环境条件 |
| 成功机制、功能与因果 | 整体成功由哪些转换共同产生 | 必要状态、功能、接口、资源、控制和因果前提 |
| 生命周期、状态与时间 | 质量在何时出现、维持或失效 | 状态转移、不可逆点、交接、运行、演化和退出 |
| 约束、损失与风险 | 哪些状态或后果不能发生 | 硬护栏、停止条件、恢复、残余风险和授权 |
| 标准、规律与历史事实 | 已积累的领域知识提醒什么 | 领域义务、常见测量、已知失败与参考模型 |

质量属性只作为“如何才算好”的选择性投影，不做所有轴的全量笛卡尔积。候选只有能说明缺失后会失去什么价值、成功机制或损失控制时才进入。

## 近敌边界

| 近敌 | 能贡献什么 | 为什么不等于质量范围 |
| --- | --- | --- |
| 质量属性列表 | 提供候选词汇 | 没有触发、证据、失效和整体关系 |
| 流程清单 | 固化动作顺序 | 做过动作不等于目标状态成立 |
| 验收清单 | 提供局部观察点 | 只能覆盖已列样例或最终快照 |
| 风险清单 | 暴露已知坏结局 | 已知风险不证明失败空间已覆盖 |
| 知识目录 | 指示应查阅的材料 | 知道什么不等于必须使什么成立 |

## Trial and error 的位置

Trial and error 是模型外残余未知的升级通道，不是从零发现基本质量范围的主要方法。

现实失败出现后先分类：

| 失败类型 | 修改对象 |
| --- | --- |
| 已有义务存在但执行者未遵守 | 执行控制层 |
| 义务执行了但定义过弱 | 义务契约 |
| 局部完成但组合失败 | 关系或全局不变量 |
| 证据显示通过但测错对象 | 证据/测量模型 |
| 失败机制完全不在模型内 | 任务现实模型与候选生成 |

## 权威参考锚点

- [NASA Stakeholder Expectations Definition](https://www.nasa.gov/reference/4-1-stakeholder-expectations-definition/)：从利益相关者、使用方式和运行场景定义期望。
- [NASA Logical Decomposition](https://www.nasa.gov/reference/4-3-logical-decomposition/)：从功能、性能、行为和时间关系推导要求。
- [ISO/IEC/IEEE 15288:2023](https://www.iso.org/standard/81702.html)：覆盖构思、开发、生产、使用、支持和退役的生命周期过程。
- [ISO/IEC 25010:2023](https://www.iso.org/standard/78176.html)：产品质量参考模型；用于需求、测量和评价，不等于具体任务的完整范围。
- [ISO/IEC 25030:2019](https://www.iso.org/standard/72116.html)：质量需求的引出、定义、使用和治理。
- [ISO/IEC 25020:2019](https://www.iso.org/standard/72117.html)：质量测量、可靠性与有效性。
- [SEI Quality Attribute Workshops](https://www.sei.cmu.edu/library/quality-attribute-workshop-collection/)：用利益相关者场景发现和细化质量要求。
- [MIT STPA Handbook](https://psas.scripts.mit.edu/home/books-and-handbooks/)：用控制结构主动分析组件均正常时仍可能发生的系统损失。
- [NIST AI RMF Core](https://airc.nist.gov/airmf-resources/airmf/5-sec-core/)：区分 Govern、Map、Measure、Manage，强调持续、情境化的风险管理。

这些来源支持方法部件；“多源生成 + 义务契约 + 组合闭包 + 独立挑战”是本 Skill 对它们的综合推断，不是任何单一标准直接给出的通用算法。
