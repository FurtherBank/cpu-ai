# 建立来源与权利地图执行指导

## 输入信息

**必须入参**：sealed `task-contract.md`、工作区和可访问的私域/外部入口。

**上下文参考**：[来源与权利 Schema](../references/source-rights-schema.md)、项目 `AGENTS.md` 与适用领域官方资料。

## 目标要求

生成 `source-rights-map.md`，使每个关键主张能够定位权威来源或具名缺口，并区分事实、语义、行动和风险裁决权。

完成标准：

- 每个关键主张有首选/替代来源、版本/时间、权威范围、权限和失效触发。
- 每个来源反向说明支撑的主张。
- 事实确认、语义裁决、方案决策、执行、停止、披露和风险接受权分开。
- 敏感来源仅使用最小必要访问。
- 冲突来源保留冲突组和裁决需求。

## 工作依据

- 从“删掉这条信息会不会诱发幻觉”反推硬依赖来源。
- 项目/私域事实优先从真实材料和运行状态取得；动态外部事实按需浏览官方一手来源。
- 参与、熟悉、撰写文档或最先回复都不自动等于有权确认。
- 无权限时不绕过控制：使用受托查询、现场核验或签认摘要，并写明证明边界。
- 大型独立来源域可委派只读 Subagent；原始需求、主张清单、读取边界和返回 schema 必须完整传入。

## 产出格式

```markdown
# 来源与权利地图

## 关键主张
## 来源表
## 权利角色表
## 来源冲突
## 缺失来源与替代取证
## 动态重查条件
```

将决定性读取、权限受阻、替代取证和权利判断增量写入行为日志。
