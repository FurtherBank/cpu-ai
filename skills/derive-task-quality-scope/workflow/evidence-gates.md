# 设计证据与裁决关口执行指导

## 输入信息

**必须入参**：sealed `quality-obligations.md`、`closure-model.md`、`source-rights-map.md`。

**上下文参考**：[证据与关口](../references/evidence-and-gates.md)、适用测试规范、测量模型和运行资料。

## 目标要求

生成 `evidence-gates.md`，使每个义务和关系都有证明域明确的证据与可执行下游关口。

完成标准：

- 每项义务/关系有证据对象、方法、来源、版本、环境、责任人、通过/未知/失效条件。
- 指标写明构念、分母、样本、阈值依据、偏差和泛化边界。
- 证据之间是否共享模型、数据、评分器和失败模式可见。
- 阻断未知连接禁止动作、责任来源和重开步骤。
- verification、validation、运行反馈和独立评价不互相冒充。

## 工作依据

- 优先选择直接支持义务状态的证据；代理关系本身也必须取证。
- 多证据按失败模式独立性判断，不按数量计票。
- 定义义务的执行者不得被当作独立评价者。
- 真实环境不可安全测试时，组合受控演练、影子/离线、人工版本化检查、独立对账和有权风险接受；公开外推边界。
- 证据不可得且无可接受替代时保留未知，不删除义务。

## 产出格式

```markdown
# 证据与裁决关口

## 证据计划
## 测量有效性
## 证据独立性
## 关口矩阵
## 替代证据与外推边界
## 证据失效与重开
```

完成后运行 V4 和 T3；失败按关系、证据或义务缺口回退。
