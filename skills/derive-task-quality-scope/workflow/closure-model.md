# 建立组合闭包执行指导

## 输入信息

**必须入参**：sealed `quality-obligations.md`、`task-reality-model.md`、`task-contract.md`。

**上下文参考**：[组合闭包方法](../references/closure-methodology.md)、[注意力预算接入](../references/attention-integration.md)。

## 目标要求

生成 `closure-model.md`，解释局部义务和关系如何在同一相关状态中支持整体裁决。

完成标准：

- 依赖、冲突、顺序、共享状态、权衡、证据耦合和失效传播显式存在。
- 全局不变量、硬护栏、联合义务、完成层级和决策关口齐全。
- 已知局部全绿/整体失败反例被命中。
- 义务集合有共同可达状态，不制造真空充分。
- 质量冲突保留多维证据并交给有权角色，不使用伪总分。

## 工作依据

- 从现实模型的接口、状态、资源、传播和反馈建立关系，不对所有义务做全量交叉。
- 对承重关系构造“两项都真但整体失败”和“后续变化推翻历史完成”反例。
- 区分制品、集成、真实用途、转交/运行和长期适用，只保留当前任务适用层级。
- 发现缺少局部义务时回候选生成；发现义务不可判真时回契约化。
- 整合受前文锚定或达到注意力门槛时，委派只读取义务封口摘要、现实关系和整体主张的整合 Subagent。

## 产出格式

```markdown
# 组合闭包模型

## 关系图
## 全局不变量与硬护栏
## 联合义务
## 权衡、敏感点与决策权
## 完成层级
## 失效传播
## 局部全绿/整体失败反例
## 整体裁决规则
```

关系判断、反例、权衡和未闭合关系进入行为日志。
