# 恢复任务现实模型执行指导

## 输入信息

**必须入参**：sealed `task-contract.md`、`source-rights-map.md`、已取得事实与冲突登记。

**上下文参考**：[任务形态路由](../references/task-shape-routes.md)、[质量范围方法论](../references/quality-scope-methodology.md)、项目/组织/运行资料。

## 目标要求

生成 `task-reality-model.md`，使整体结果的现实生成机制、破坏位置、不可逆点和反馈路径可被定位。

完成标准：

- 利益相关者/对象边界、输入输出、成功机制、状态/时间、接口/资源、控制/责任、生命周期和反馈被逐项判定适用性。
- 对状态改变、退役、迁移或异步任务，显式判定在途工作、延迟回调、持久化产物/句柄、缓存与历史副作用是否会越过切换点；不得以“新请求归零”代替生命周期结束。
- 每个关键事实有来源 ID；冲突、低频依赖和不可观测路径显式存在。
- 任务形态及组合路由明确。
- 没有以理想流程、行业惯例或“未观察到”填补现实事实。

边界：只恢复现实，不设计方案。

## 工作依据

- 从最终效果反向追踪承载点、输入、转换、状态和反馈。
- 根据任务形态选择入口，多个形态可并存。
- 行为痕迹、运行状态、真实制品和有权材料优先于概括性访谈。
- 两种模型解释会导致不同义务时，分别单点封口，不顺手选一个。
- 多系统域达到注意力门槛时一域一 Subagent，全部封口前只维护依赖图。

## 产出格式

```markdown
# 任务现实模型

## 任务形态
## 对象与利益相关者边界
## 成功机制与必要状态
## 状态、时间与生命周期
## 接口、资源与外部依赖
## 控制、责任与反馈
## 不可逆点
## 事实冲突与模型盲区
```

完成后运行 V2 和 T1；失败按来源缺口或模型缺口回退。
