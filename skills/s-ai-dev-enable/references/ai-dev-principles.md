# AI 可研发项目的七条能力维度

这是判断一个项目是否具备"AI 可无缝托管研发"状态的基准。  
每条能力对应一个可验收的状态，赋能建设以这七条为最终验收标准。

---

## 能力一：可读性

**定义**：AI Agent 进入项目后，无需人类辅助即可理解项目的架构、约定和工作方式。

**必备资产**：`AGENTS.md`（含技术架构、目录说明、模块边界约定）

**验收标准**：
- Agent 能正确回答"新增一个 API endpoint 应放在哪个目录？"
- Agent 能正确回答"运行测试的命令是什么？"
- Agent 能识别哪些文件是"约定文件"（如 Next.js 的 `page.tsx`），哪些是业务文件

**常见缺陷**：AGENTS.md 只有泛泛描述（"遵循项目规范"），无可执行命令；目录结构未说明各目录用途。

---

## 能力二：可执行性

**定义**：AI Agent 能在本地和 CI 环境中完整运行项目的开发工作流。

**必备资产**：完整的工具链命令清单（test/lint/build/start）

**验收标准**：
- 所有命令可在 clone 后直接运行，无需额外查文档
- CI pipeline 能在无人工干预下完整运行

**常见缺陷**：命令中有 placeholder 未替换（`<your-port>`）；依赖本地配置文件但未提供 `.env.example`。

---

## 能力三：可约束性

**定义**：AI 编写的代码在风格、架构和安全性上有明确约束，且约束可被机器执行（非仅靠 AI 自律）。

**必备资产**：`.cursor/rules/` 下的规则文件集合 + linter 配置

**验收标准**：
- 提交不符合规范的代码，linter 能自动检测并报错
- cursor rules 与 linter 无循环冲突（AI 改了格式，linter 不会又改回去）

**常见缺陷**：cursor rules 用口号性描述（"写高质量代码"）而非可检查约束；规则与 linter 配置冲突。

---

## 能力四：可感知性（SDK/发布型项目必须）

**定义**：AI 能感知公共 API 表面积的变化，自动识别 breaking change。

**必备资产**：机器可读的 API 基线文件（如 `etc/sdk.api.md`，由 api-extractor 生成）

**验收标准**：
- 修改公共 API 签名后，`npx api-extractor run --local` 能检测到差异
- CI 中 api-extractor 检查能阻断未标注为 major 的 breaking change PR

**常见缺陷**：仅靠文字约束"不要引入 breaking change"，AI 在上下文大时会遗漏；无 CI 防护。

---

## 能力五：可操作性

**定义**：AI 对外部平台（云服务、CI 系统、监控平台等）的操作能力有清晰边界，"能做什么"和"不能做什么"都明确定义。

**必备资产**：
- 平台资产清单（每个外部依赖的 AI 可操作方式）
- 可用的 Toolkit wrapper（中/重度依赖场景）

**验收标准**：
- AI 能通过 CLI/Toolkit 查询各类状态（部署状态、告警、日志）
- 每个外部工具有明确的"可自动/需人工"分类

**常见缺陷**：AI 被允许"想象"自己能操作云控制台；无 toolkit 情况下要求 AI 直接调用复杂 API。

---

## 能力六：可阻断性

**定义**：不可逆操作（生产部署、数据库迁移、发布）有物理机制阻止 AI 自动执行，即使 AI 在上下文膨胀时"忘记"了 AGENTS.md 的文字约束。

**必备资产**：物理闸门机制（CI environment protection / workflow_dispatch / token 验证）

**验收标准**：
- 每个 AGENTS.md 红区操作对应至少一个物理闸门
- 模拟 AI 尝试执行红区操作，物理闸门能成功阻断

**常见缺陷**：只有 AGENTS.md 写"禁止"，无任何物理约束；CI 没有 environment protection。

> ⚠️ **关键认知**：文字约束在 AI 上下文足够大时会被遗忘，物理闸门不会。

---

## 能力七：可验证性

**定义**：赋能完成后，可以通过端到端的 AI 研发循环验证赋能效果，而非仅通过人工审查文档。

**必备资产**：端到端验证通过（正向路径 + 错误路径）

**验收标准**：
- AI 仅读 AGENTS.md 即可独立完成一个完整研发循环（需求→编码→测试→PR）
- linter 拦截和物理闸门均通过模拟验证确认生效

**常见缺陷**：赋能以"文档写完"为终点，未实际验证 AI 能运行；只测正向路径，不测错误路径。
