# Step 8 - 生产发布执行指导

> **路径根**：发布记录写入 **工作区根目录** `{workspaceRoot}/.iteration/`，与 `i-verify/SKILL.md` 约定一致。

## 输入信息

**必须入参**：
- `🚀 仓库生产发布指引`：从 AGENTS.md 提取的完整章节内容（硬依赖）
- `生产发布要求`：从变更计划表提取的发布要求（灰度比例、Feature Flag、观察期等，硬依赖）
- `pre_release_verify` 状态=`completed`（必须确认）

## 目标要求

**任务**：按变更计划表的发布要求，安全执行生产发布，留有发布记录

**目标**：生产发布完成，监控无告警，发布记录存在

**特性要求**：
- 发布前必须确认预发验证通过（二次检查 `{workspaceRoot}/.iteration/verify-state.json`）
- 灰度发布/Feature Flag 必须按计划表要求配置，不得擅自全量
- 生产 DB 迁移（若有）必须等待人工输入确认指令，严禁自动执行
- 发布后必须按指引持续观察监控，不得在观察期结束前宣告完成

## 工作依据

**执行流程**：

**1. 前置确认**

```
检查 `{workspaceRoot}/.iteration/verify-state.json` 中 pre_release_verify == completed
若非 completed：立即阻断，输出警告，不执行任何发布操作
```

**2. 读取发布要求**

从变更计划表「生产发布要求」提取：
- 是否需要 Feature Flag（若需要，先配置再发布）
- 灰度比例（如 10%，则第一次只发布 10%）
- 观察期（如 24h）
- 是否有生产 DB 迁移

**3. 预发布配置（若需要）**

若变更计划表要求 Feature Flag：
- 按 🚀 指引找到 Flag 配置入口
- 确认 Flag 已创建且默认为关闭状态
- 按灰度比例配置 Flag（如 10% 流量）

若有生产 DB 迁移：
```
输出：
  ⚠️ 生产 DB 迁移需要人工确认

  迁移内容：{从变更计划表提取的迁移摘要}
  迁移脚本：{脚本路径}
  影响：{表名/操作类型/是否不可逆}

  请确认后输入：CONFIRM MIGRATE PRODUCTION
  （输入其他内容或超时 → 不执行，进入 BLOCKED 状态）

等待人工确认...
```

**4. 执行发布**

按 🚀 指引执行生产发布命令。记录命令和输出。

**5. 发布后验证**

按 🚀 指引说明的验证方式（如健康检查接口、监控面板）确认新版本已运行。

**6. 观察期监控**

若发布要求有观察期（如 24h）：
- 标注观察期开始时间
- 按指引检查关键监控指标（错误率、响应时间、业务指标等）
- 若出现告警：**立即阻断，按 🚀 指引执行回滚**，状态置为 HALTED，升级人工

若无特定观察期：按指引完成发布后验证即可。

**7. 写入发布记录**

```
{workspaceRoot}/.iteration/recordings-publish/production-{YYYYMMDD-HHmmss}.md
```

格式见下方产出模板。

**方法论**：
- 发布前最后一次检查：预发验证报告结论是 ✅ 全部通过（不是人工跳过）
- 灰度期间遇到任何可疑指标（即使还未告警）：主动暂停，宁可保守
- 生产回滚比「再观察一下」成本更低——不确定时优先回滚
- 观察期内不得宣告完成，即使看起来一切正常
