# 整合阶段回归校验清单

> 完备层：收录历史翻车点对应的回归防线，供整合步骤 5 逐条核对。这里的多数条目可从产物模型的职责句与收录判据推出，仍单列是因为它们违反的是执行 agent 的默认倾向（穷举、附加相关内容、强行收束），推导内化不可靠。
> 新增防线一律追加到本清单并附一行触发场景，不回写 SKILL 正文；若发现某条已被判据结构性堵死、可以删除，须用户确认。

## 核心上下文.md

- [ ] 正文不是大纲、事实流水账、问答台账或纯结论清单；每个模块实际聚合出定义、边界、事实、观点、关系、应用、待确认。（触发场景：agent 把索引条目改写成「看起来完整但没有内容承接」的大纲）
- [ ] 未闭环命题保持待讨论状态，没有为文档完整感强行收束成结论。（触发场景：模型默认倾向于给出完整结论姿态）
- [ ] 没有维护台账混入正文：候选筛选表、旧新映射、事实承诺表、专项交接、升级候选表都在净化日志或交付说明。（触发场景：审计表被顺手贴在正文末尾）
- [ ] 正文没有内嵌主题边界章节、来源清单或「相关主题」清单；来源只留 `Sxx` 锚点。（触发场景：边界与来源在正文重复维护后与专门文件漂移）
- [ ] 排除材料、相邻主题内容、维护建议没有被写成既定正文结论。（触发场景：筛选阶段的候选被污染进正文）
- [ ] 医疗、法律、财务、职业终局等高风险内容保持方法论或待验证判断形态，未写成普遍建议。

## 主题边界.md 与结构归属

- [ ] 主题边界只用本主题自身表述，未借其他主题解释边界。
- [ ] 每个关键候选都有结构动作判断（新主题 / 新模块 / 归入已有模块 / 拆分归属 / 历史案例 / 升级候选），没有按标题或大方向相似归类。（触发场景：新对话被草率并入宽泛命名的旧主题）
- [ ] 归入既有主题的候选都写了脉络接点；写不出接点的没有被标为「纳入」。
- [ ] 持续对话检查过轮次漂移：后续轮次换了讨论对象的，未整段沿用前半段归属。
- [ ] 派生 / 桥接主题未被强行合并进最相似的单一上游主题；「相关但未引用」的主题没有被写成依赖或边界。
- [ ] 多因叠加问题单列了成因栈，独立因素未被误写成某一上游主题的派生问题。
- [ ] 既有主题的升级诉求区分了「必要边界升级 / 泛化风险 / 真实结构升级」，没有顺手扩大主题。
- [ ] 用户声明完结的主题只被作为历史案例或验证来源引用，没有写入新策略正文。

## 索引.md

- [ ] 每行来源通过收录判据自查：删掉它会让正文某个最终内容失去出处。
- [ ] 无逐轮 `001.md/002.md` 展开；无关联、相邻、排除、暂缓、背景、「未来可能参考」材料；无级联依赖。（触发场景：agent 默认倾向于附上「尽量全」的资料清单）
- [ ] 无详细摘要、推理链、关系分析、行动建议等正文内容。
- [ ] 用户原有精心维护的链接被合并重排而非删除（除非明确替换）。

## 概念文件与权威结构

- [ ] 全文评价轴概念已单独成文，其权威位置、非范围、关系约束进入配套表格；仅属单一模块的定义未被拆成独立文件。

## 过程与日志

- [ ] 双数据源（ChatGPT / Sider）进同一份筛选记录，主题结构未按数据源拆成两套。
- [ ] 纯案例只有在提炼出模型 / 边界 / 判断规则并实际进入正文后才出现在索引，否则仅留筛选记录。
- [ ] 日志落盘档位（轻量 / 标准 / 审计）已判断；未落盘时关键偏离、覆盖缺口、升级候选、剩余风险已进入交付说明。
- [ ] `public/chats/` 原始资产未被复制、改写或移动。
