# 阶段一：任务基本性质确认

## 输入信息

**必须入参**：

- 用户原始需求。
- 待净化信息集合。
- 用户明确给出的不可改变约束、目标读者、输出要求。

**上下文参考**：

- [信息净化方法](../references/info-purify-method.md)

## 目标要求

**任务**：确认本次任务到底是在做内容整理、信息结构重构、生成过程控制，还是已经进入设计/规则升级。

**完成标准**：

- 写出任务身份说明。
- 写出目标消费路径：谁消费、消费后做什么、偏好的输出形态。
- 写出主要目标：清单化整理、结构重构、规则审查、知识库重建、执行 SOP 化等。
- 当主要目标是结构重构时，明确“最终产物必须能替换或重组原结构”。
- 写出不可改变约束。
- 写出输入盲区清单。
- 对未知信息类型标记“未知类型”，不套模板。

**边界要求**：

- 不阅读后直接润色。
- 不替用户裁决目标读者冲突。
- 不把升级问题当作正文优化处理。
- 不把“保全来源”和“说明引用”设为主要目标；它们是后续防失真约束。

## 工作依据

先问“这批信息净化后给谁用”，再问“怎么写得更好”。如果消费路径缺失，后续所有结构选择都会失焦。

遇到目标不明时，生成临时候选消费路径并标为待确认，例如“阅读理解”“照做执行”“评审决策”“排错判断”“后续生成”。

如果用户要求“重新梳理完整信息结构”“重建知识体系”“把材料整理成可继续使用的体系”，默认主要目标是结构重构，而不是内容清单。

```markdown
...

...
```
