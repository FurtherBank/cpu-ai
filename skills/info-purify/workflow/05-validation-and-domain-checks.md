# 阶段五：事实概念权威关系与领域风险核查

## 输入信息

**必须入参**：

- 目标结构草案。
- 旧新映射表。
- 信息块台账。
- 信息类型识别结果。
- 消费路径说明。
- 已有领域正式规范（如有）。

## 目标要求

**任务**：在目标结构和追溯映射稳定后，核查事实一致性、状态边界、核心概念、权威来源、显式关系和具体信息类型的专用风险。

**完成标准**：

- 强事实和强规则都有来源。
- 事实、推断、假设、历史背景、候选方案、待确认、废弃项已分层。
- 核心概念有定义、边界、非范围、生产方、消费方、下一步判断。
- 权威来源说明运行时参考谁、维护时参考谁。
- 关系表覆盖生产、消费、依赖、冲突、前后置、约束关系。
- 需要升级才能优化的问题已记录为升级候选。
- 信息类型已识别；未知类型已标注。
- 已知类型应用对应风险检查项。
- 未知类型先识别消费路径和专用约束，再生成临时检查项。
- 临时检查项标明依据和置信度，不作为领域裁决。
- 领域依据不足时输出补充信息请求。
- 明确区分“Skill 已处理的通用问题”和“使用者需要继续判断的 case 专项问题”。
- 对 case 专项问题输出审查建议、判断注意点和待确认问题。

**边界要求**：

- 不裁决同级来源之间的事实冲突。
- 不把开放问题写成既定规则。
- 不把历史概念写成当前有效规则。
- 不硬套已有类型模板。
- 不把临时检查项写成权威规则。
- 不让领域检查改变已确认事实。
- 不把专项审查交接清单写成已经完成的领域裁决。

如果没有领域正式规范，不视为入参缺失；本阶段的职责就是基于消费路径生成临时专用检查项，并明确其非权威状态。

## 工作依据

先核查事实，再核查概念。事实不稳定时，概念闭合会把错误前提固化。

判断升级候选的简式规则：

```text
如果修改会改变事实承诺、执行行为、上下游契约、责任归属或设计取舍，
就登记升级候选。
```

常见类型检查：

- 规则/规范：当前规则、历史背景、开放问题、设计候选是否隔离。
- SOP：触发、跳过、输入、输出、完成判定、异常路径、执行主体是否明确。
- 架构设计：事实、约束、候选方案、历史背景、模块关系、概念闭合是否明确。
- 会议/聊天记录：正式决定、待决项、假设、少数派意见、行动项字段是否明确。
- Schema/契约：字段语义、必填/可选、生产方、消费方、兼容性是否明确。
- 知识库文章体系：文章事实和概念、跨文章重复/冲突/断链、归属不清可由 Skill 审查；栏目策略、用户任务覆盖、文章生命周期和修改优先级应进入专项审查交接。
- 未知类型：先识别消费路径和输出偏好，再生成临时检查项。
