# 阶段六：产物生成与主控校验

## 输入信息

**必须入参**：

- 阶段一至五全部定稿产物。
- [净化信息包模板](../templates/purified-info-package.md)

## 目标要求

**任务**：生成完整净化信息包，并执行主控结构校验。

**完成标准**：

- 净化后正文可供目标读者直接消费。
- 配套台账可追溯、可复核、可维护。
- 升级候选清单完整。
- 专项审查交接清单完整，且没有伪装成已裁决结论。
- 剩余风险和补充信息请求明确。
- 结构校验清单逐项完成。

**边界要求**：

- 不在最终正文中隐藏低置信、待确认或升级候选。
- 不为了简洁删除来源锚点。
- 不把校验失败项留给下游自行发现。

## 工作依据

净化正文服务消费路径，配套台账服务复核和维护。轻量任务可以压缩台账，但强事实、强规则、待确认和升级候选四类信息必须保留追溯方式。

## 主控校验清单

- [ ] 原始信息已保全或说明无法保全的原因。
- [ ] 净化正文中的强事实和强规则能回溯来源。
- [ ] 事实、推断、假设、待确认、历史背景、候选方案没有混写。
- [ ] 核心概念具备定义、边界、非范围、生产方、消费方、下一步判断。
- [ ] 权威来源和责任边界明确。
- [ ] 关系已经显式化，不只是一组并列段落。
- [ ] 领域特定检查项已应用或已生成临时检查项。
- [ ] Skill 已处理的通用问题和使用者需要继续判断的 case 专项问题已分开记录。
- [ ] 需要升级才能优化的问题已进入升级候选清单。
- [ ] 目标消费路径下，读者能直接使用结果。
- [ ] 剩余风险和补充信息请求已记录。

## 产出格式

使用 [净化信息包模板](../templates/purified-info-package.md)。
