# 阶段六：产物生成与主控校验

## 输入信息

**必须入参**：

- 阶段一至五全部定稿产物。
- [净化信息包模板](../templates/purified-info-package.md)

## 目标要求

**任务**：生成完整净化信息包，并按“产物本体 / 净化日志记录”两层结构执行主控校验。

**完成标准**：

- 产物本体是面向目标读者的最终正文，可独立抽取、发布、复制或替换原结构。
- 产物本体不依赖净化日志才能被理解和使用。
- 产物本体是可直接消费的新结构，不只是信息清单、分析报告或执行记录。
- 阶段一至五的诊断、映射、台账、风险、校验等内容进入净化日志记录，不混入产物本体。
- 净化日志记录能说明结构被怎样改造，并支撑追溯、复核和维护。
- 升级候选记录完整，不伪装成产物本体中的已裁决结论。
- 专项审查交接记录完整，且与 Skill 已处理的通用问题分开。
- 剩余风险和补充信息请求明确。
- 产物本体校验和日志记录校验均逐项完成。

**边界要求**：

- 不把“生成了完整信息包”误判为“最终正文已经可用”。
- 不把结构诊断、旧新映射、台账、校验清单写进产物本体，除非目标读者的消费动作本身就是审计这些记录。
- 不在产物本体中隐藏低置信、待确认或升级候选；需要目标读者知道的风险应以正文可消费的形式表达。
- 不为了让产物本体简洁而删除强事实、强规则、待确认和升级候选的追溯方式；追溯可以放在日志记录中。
- 不把校验失败项留给下游自行发现。

## 工作依据

阶段六要先回答“真正交给目标读者使用的是什么”，再回答“本 Skill 如何证明它没有失真”。

产物本体服务消费路径。它是最终正文，也是这次净化的主要交付。净化日志记录服务审查路径，包含任务基本性质、原结构诊断、目标结构判断、旧新映射、信息块台账、事实概念权威关系核查、专项审查交接、升级候选、结构校验和剩余风险。

轻量任务可以压缩日志记录，但不能取消产物本体和日志记录的边界。强事实、强规则、待确认和升级候选四类信息必须在产物本体或日志记录中保留可追溯方式；其中目标读者必须知道的风险要进入产物本体。

## 生成步骤

1. 确认产物本体形态：根据消费路径确定最终正文应是文章、SOP、知识库结构、规范、表格、问答、决策材料或其他结构化输出。
2. 生成产物本体：只写目标读者需要直接使用的内容；必要的来源锚点、低置信提示和待确认标记要用正文可理解的方式表达。
3. 隔离净化日志记录：把阶段一至五的过程性产物填入模板 B 区，不把日志记录倒灌进最终正文。
4. 校准产物本体与日志记录：确认正文中的强事实、强规则、核心概念、关系和风险能在日志记录中找到依据。
5. 执行主控校验：分别检查产物本体可消费性、日志记录可追溯性，以及二者之间的一致性。
6. 标记剩余风险：把无法由净化阶段解决的问题写入专项审查交接记录、升级候选记录或剩余风险记录。

## 主控校验清单

### 产物本体校验

- [ ] 产物本体可单独抽取，且不依赖净化日志才能被目标读者消费。
- [ ] 产物本体是可直接消费的新结构，不只是信息清单、分析报告或执行记录。
- [ ] 产物本体符合阶段一确认的目标读者、消费动作和输出范围。
- [ ] 产物本体对原结构做了实质性重排、合并、拆分、上提、下沉或隔离。
- [ ] 产物本体中的事实、推断、假设、待确认、历史背景、候选方案没有混写。
- [ ] 目标读者必须知道的低置信、待确认、冲突或风险已在产物本体中以可消费形式表达。

### 净化日志记录校验

- [ ] 已形成原结构诊断和目标结构判断，并记录在净化日志中。
- [ ] 已提供旧结构到新结构的映射，并说明每类关键原始信息的去向。
- [ ] 原始信息已保全或说明无法保全的原因。
- [ ] 产物本体中的强事实和强规则能回溯来源。
- [ ] 核心概念具备定义、边界、非范围、生产方、消费方、下一步判断。
- [ ] 权威来源和责任边界明确。
- [ ] 关系已经显式化，不只是一组并列段落。
- [ ] 领域特定检查项已应用或已生成临时检查项。
- [ ] Skill 已处理的通用问题和使用者需要继续判断的 case 专项问题已分开记录。
- [ ] 需要升级才能优化的问题已进入升级候选记录。
- [ ] 剩余风险和补充信息请求已记录。

### 一致性校验

- [ ] 产物本体没有引用日志中不存在的来源、概念、规则或结论。
- [ ] 净化日志记录没有把未进入产物本体的候选方案伪装成最终正文结论。
- [ ] 产物本体的结构改造结果能被旧新映射和结构改造策略解释。
- [ ] 若日志记录被压缩，压缩理由和不可压缩的追溯项已保留。

## 产出格式

使用 [净化信息包模板](../templates/purified-info-package.md)。模板 A 区是产物本体，模板 B 区是净化日志记录。
