# 信息净化方法

## 定义

信息净化指对已有或正在生成的信息进行结构、内容和约束层面的优化，使其更容易被人或 AI 正确理解、继续推理、执行和维护。

```text
信息净化 = 保真事实 + 闭合概念 + 明确归属 + 显式关系 + 可执行判断 + 可校验结构
```

主控顺序：

```text
先形成结构判断，再自上而下重构，最后用来源追溯和校验防止失真。
```

来源保全、引用说明和台账追溯是防失真机制，不是主要目标。保全来源不等于保留原始结构。

## 三类问题

| 类型 | 判定 | 当前流程动作 |
| --- | --- | --- |
| 信息组织优化 | 不改内容，不改设计约束，只调整分块、归属、顺序、关系、去重 | 直接处理 |
| 内容表达优化 | 可改表达和完整度，但不改变事实承诺和设计约束 | 在来源支持下处理 |
| 设计约束优化 | 需要改变规则、流程、模块边界、API、状态模型、权责或产品约束 | 登记升级候选 |

判断句：

```text
如果一次修改会改变事实承诺、执行行为、上下游契约、责任归属或设计取舍，
它就不再是普通信息净化，而是升级候选。
```

## 推荐顺序

1. 确认信息身份和消费路径。
2. 诊断原结构问题：层级、概念、状态、消费路径、归属和关系。
3. 形成目标结构判断：读者需要按什么结构消费。
4. 自上而下设计目标结构：公共约束、主体结构、分支视图、历史背景、风险和待确认项。
5. 做旧新映射：保留、合并、拆分、移动、降级、转待确认、进入升级候选。
6. 来源保全与信息切块，用追溯校准目标结构。
7. 事实一致性和状态边界检查。
8. 概念闭合、权威来源和责任边界检查。
9. 关系显式化和领域风险识别。
10. 生成可替换正文、配套台账和结构验收。

## 信息块字段

| 字段 | 说明 |
| --- | --- |
| block_id | 信息块编号 |
| source_id | 来源编号 |
| 原文摘录 | 足以回溯的原始内容 |
| 信息类型 | 背景、目标、事实、约束、问题、假设、决策、方案、风险、行动项等 |
| 状态 | 当前有效、历史背景、待确认、推断、候选、废弃 |
| 原归属 | 原章节、scope、概念、模块、阶段或问题域 |
| 新归属 | 目标结构中的章节、视图、概念、模块、阶段或问题域 |
| 关系 | 依赖、支持、冲突、包含、前后置、例证、替代 |
| 置信度 | 高、中、低 |
| 处理建议 | 保留、合并、移动、降级、待确认、升级候选 |

## 结构改造动作

| 动作 | 判定 |
| --- | --- |
| 保留 | 原结构位置已经服务消费路径，且不造成混写 |
| 合并 | 多处重复或同类信息分散，合并不改变语义 |
| 拆分 | 一个段落混有事实、规则、例子、风险、行动项或历史背景 |
| 移动 | 信息有效但归属错位，应迁移到更高/更低/其他视图 |
| 上提 | 全局约束、核心结论、关键定义被埋在局部细节中 |
| 下沉 | 例子、细节、局部操作干扰主线阅读 |
| 隔离 | 历史背景、候选方案、开放问题会污染当前规则 |
| 降级 | 信息只能作为背景、低置信说明或参考，不能作强结论 |
| 待确认 | 来源不足或状态不明，不能进入正文强主线 |
| 升级候选 | 需要改变事实承诺、执行行为、契约、权责或设计取舍 |
| 排除出正文 | 与消费路径无关，但需在台账中说明去向 |

## 概念闭合字段

| 字段 | 检查问题 |
| --- | --- |
| 定义 | 它是什么 |
| 边界 | 它覆盖什么 |
| 非范围 | 它不覆盖什么 |
| 生产方 | 谁创建或维护它 |
| 消费方 | 谁使用它 |
| 生命周期 | 何时产生、何时失效 |
| 相近概念 | 与哪些概念容易混淆 |
| 下一步判断 | 使用它之后该如何行动或判断 |

## 权威来源字段

| 字段 | 检查问题 |
| --- | --- |
| 来源 | 文件、代码、用户确认、历史决策、外部文档 |
| 权威等级 | 原始规范、用户明确补充、代码事实、实践案例、推断、开放问题 |
| 适用范围 | 当前文档、某个 scope、某类运行时、某个维护场景 |
| 维护方 | 谁有资格修改 |
| 运行时依据 | 执行时应看谁 |
| 维护依据 | 修改规则时应看谁 |
| 冲突优先级 | 多来源冲突时先信谁 |

## 升级候选格式

| 字段 | 说明 |
| --- | --- |
| ID | 升级候选编号 |
| 触发材料 | 文件、段落、信息块或原文编号 |
| 问题类型 | 设计约束、流程制度、权威来源、消费路径、领域规则等 |
| 现象 | 当前信息为什么净化不了 |
| 不直接处理原因 | 为什么不能在净化阶段直接改 |
| 影响范围 | 影响哪些正文、规则、模块、行动项或读者 |
| 建议判断 | 后续需要谁判断、判断什么 |
| 置信度 | 高 / 中 / 低 |

## 验收口径

沿目标消费路径阅读时，读者不会遇到明显事实矛盾、概念缺口、归属错位、权威来源不清、结构不可执行或表达含混的问题。

如果交付物只有信息清单、摘要、分类表或问题列表，而没有目标结构、旧新映射和可直接消费的新正文，视为未完成信息净化。

很多时候不要求“绝对没有任何毛病”，但至少要保证目标消费路径下能用、不严重误导、风险显式可见。
