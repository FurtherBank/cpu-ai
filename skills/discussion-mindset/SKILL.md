# 研讨心态

## 0. 核心姿态

把当前任务当作“寻找下一个更好的可能”，而不是急着把问题定型。

使用本技能时，先让问题继续变锋利，再判断当前是否已经值得收束。研讨心态不预设回答格式；它只调整注意力的摆放方式：让当前判断暴露新的突破点、盲区、反例和下一轮问题。

## 1. 原始问题：AI 容易完成形式，却错过灵魂点

原始材料提出的关键观察是：当用户让 AI 执行“寻找最有价值方案”这类创新任务时，AI 经常能写出全面方案，但方案里看不到真正支撑价值的灵魂要素。

这里的“灵魂要素”至少包括：

| 要素 | 说明 |
| --- | --- |
| 原生级技术创新 | 不是组合现有做法，而是提出新的实现路径或系统形态。 |
| 精准直达本质的判断 | 能重新命名问题，使原本模糊的困难突然可理解。 |
| 全新解决视角 | 改变看问题的坐标系，而不是在旧框架内补细节。 |
| 被忽略变量 | 指出普通方案没有纳入、但实际决定成败的变量。 |
| 下一问题 | 不是回答当前问题，而是提出更值得回答的问题。 |

这些要素和“完整方案”之间的关系不是并列，而是核心与支撑：

> 创新研讨的主价值来自灵魂点；流程、里程碑、风险、资源、执行步骤只是让灵魂点可讨论、可验证、可落地的支撑结构。

## 2. 执行交付心态与研讨心态

执行交付心态默认追求“现在给出一个完整答案”。它的注意力在完整性、合理性、可读性、可执行性上。

研讨心态默认追求“下一个更好的可能”。它的注意力在突破点、盲区、反例、未命名变量、下一轮问题上。

| 维度 | 执行交付心态 | 创新研讨心态 |
| --- | --- | --- |
| 默认目标 | 交付完整答案 | 找到更高价值的下一步 |
| 好答案标准 | 全面、稳妥、可执行 | 锋利、有新信息、能推进问题 |
| 时间感 | 当前轮次要收束 | 当前轮次只是下一轮的踏板 |
| 信息组织 | 方案结构优先 | 判断与可能性优先 |
| 对不确定的态度 | 尽量消除不确定 | 保留高价值不确定 |
| 对产物的期待 | 能直接拿去做 | 能让下一轮更值得讨论 |

研讨心态的核心句可以压缩为：

> 最好的答案永远是下一个；当前答案的价值，在于它把我们推向哪个更好的问题。

## 3. AI 默认进入执行交付的原因

从原始材料可以抽出几个机制层原因：

1. AI 通常把“写方案、给建议、完成任务”理解为交付型指令。
2. 训练语料中的高频范式奖励完整、礼貌、结构化、可解释的答案。
3. 原生创新、反直觉判断和高价值盲区本来就是低频对象，模型不容易默认优先搜索。
4. 突破性判断经常伴随风险、未证实和非共识，AI 会倾向给出更安全的常规方案。
5. 如果用户没有显式要求“先找灵魂点，再写支撑”，AI 会把支撑结构误当成主产物。

这里的关键不是“AI 不会创新”，而是：

> 默认任务叙事把 AI 推向了完成，而不是推向寻找。

因此，改造重点不是让 AI 多写几个创意，而是改造任务协议：让 AI 每一轮都必须说明当前最有价值的未知、最可能突破的方向、以及下一轮应该追问什么。

## 4. 创新研讨循环

本主题建议把创新型任务拆成一个循环，而不是一次性交付：

| 阶段 | 核心问题 | 输出 |
| --- | --- | --- |
| 1. 对象定界 | 我们到底在研讨哪类问题，不是在完成哪类方案？ | 研讨对象、非目标、评价轴 |
| 2. 灵魂点搜寻 | 哪些判断、技术、视角或变量可能真正改变问题？ | 候选灵魂点列表 |
| 3. 价值密度排序 | 哪几个点最值得继续挖，而不是最容易写完整？ | 高潜点、暂缓点、排除点 |
| 4. 反例与盲区攻击 | 这个点为什么可能是错的，普通方案漏了什么？ | 反例、风险、被忽略变量 |
| 5. 核心洞察卡 | 当前最锋利的一句话是什么？ | 当前核心判断 |
| 6. 支撑方案外壳 | 如果这个判断成立，怎样让它可验证、可落地？ | 最小验证、支撑流程 |
| 7. 下一问题 | 下一轮最值得问什么？ | 1 到 3 个研讨接口 |

这个循环的重点是顺序：先找灵魂点，再给支撑外壳。若一开始就让 AI 写完整方案，注意力会被结构填充吞掉，导致真正高价值的部分被平均化。

## 5. 研讨型产物长什么样

一个好的研讨型 AI 产物不应该只有正文答案，而应该像一个“可继续推进的问题包”。

最小结构如下：

```markdown
## 当前最锋利判断
用一两句话写出本轮最可能有价值的判断。

## 为什么它可能重要
说明它改变了哪个问题定义、变量权重或行动路径。

## 备选灵魂点
列出其他候选点，并说明为什么暂时不如主判断。

## 反例与攻击
主动提出这个判断可能失败的原因。

## 最小验证
设计一个低成本方法检验它是否真的有价值。

## 下一轮问题
给出下一步最值得讨论的 1 到 3 个问题。
```

这类产物的好坏不由篇幅决定，而由三个指标决定：

1. 是否产生了比常规答案更锋利的问题定义。
2. 是否明确区分“核心判断”和“支撑结构”。
3. 是否让下一轮讨论更容易进入高价值区域。
