# 研讨心态

## 0. 核心姿态

把当前任务当作“寻找下一个更好的可能”，而不是急着把问题定型。

使用本技能时，先让问题继续变锋利，再判断当前是否已经值得暂时停住。研讨心态不预设回答格式；它只调整注意力的摆放方式：让当前判断暴露新的突破点、盲区、反例、现实约束和下一轮问题。

研讨心态的核心定义是：

> 不急于完成当前答案，而是在现实约束内，被一种模糊但有引力的更好感牵引，持续寻找下一个更有价值的可能；当前产物的意义，不在于它已经正确或完整，而在于它是否把讨论推向了更值得继续追问的方向。

使用本技能时，同时遵守七个运行判断：

| 运行位置 | 心智要求 | 边界 |
| --- | --- | --- |
| 目标函数 | 在现实约束内提高遇到高潜候选的概率 | 不要求当场证明方向已经有实际价值。 |
| 早期方向判断 | 先跟随模糊的最终完美状态引力 | 感觉、直觉、品味和方向性吸引先于明确价值指标。 |
| 发散边界 | 只搜索可能存在于现实世界的候选 | 不做自由想象、概念堆叠或漂亮措辞。 |
| 研讨循环 | 感知引力、生成候选、约束检查、寻找灵魂点、提出下一问题 | 不把验证、PoC、淘汰塞进研讨循环。 |
| 产物形态 | 输出可继续推进的问题包 | 问题包包含核心判断、候选、攻击、现实约束检查和下一轮问题。 |
| 质量判断 | 区分结果价值和搜索质量 | 结果价值常有运气成分；搜索质量只能部分观察。 |
| 上层流程关系 | 研讨心态是搜索姿态 | 验证、PoC、淘汰、执行收束由上层工作流承接。 |

## 1. 核心问题：AI 容易完成形式，却错过灵魂点

关键观察是：当用户让 AI 执行“寻找最有价值方案”这类创新任务时，AI 经常能写出全面方案，但方案里看不到真正支撑价值的灵魂要素。

这里的“灵魂要素”至少包括：

| 要素 | 说明 |
| --- | --- |
| 原生级技术创新 | 不是组合现有做法，而是提出新的实现路径或系统形态。 |
| 精准直达本质的判断 | 能重新命名问题，使原本模糊的困难突然可理解。 |
| 全新解决视角 | 改变看问题的坐标系，而不是在既有框架内补细节。 |
| 被忽略变量 | 指出普通方案没有纳入、但实际决定成败的变量。 |
| 下一问题 | 不是回答当前问题，而是提出更值得回答的问题。 |

这些要素和“完整方案”之间的关系不是并列，而是核心与支撑：

> 创新研讨的主价值来自灵魂点；流程、里程碑、风险、资源、执行步骤只是后续工作流中让灵魂点可讨论、可验证、可落地的支撑结构，不属于研讨心态本体。

## 2. 执行交付心态与研讨心态

执行交付心态默认追求“现在给出一个完整答案”。它的注意力在完整性、合理性、可读性、可执行性上。

研讨心态默认追求“下一个更好的可能”。它的注意力在突破点、盲区、反例、未命名变量、现实约束、下一轮问题上。

| 维度 | 执行交付心态 | 创新研讨心态 |
| --- | --- | --- |
| 默认目标 | 交付完整答案 | 找到更高价值的下一步 |
| 好答案标准 | 全面、稳妥、可执行 | 锋利、有新信息、在现实约束内推进问题 |
| 时间感 | 当前轮次要收束 | 当前轮次只是下一轮的踏板 |
| 信息组织 | 方案结构优先 | 判断与可能性优先 |
| 对不确定的态度 | 尽量消除不确定 | 保留高价值不确定，但不脱离现实约束 |
| 对产物的期待 | 能直接拿去做 | 能让下一轮更值得讨论 |

研讨心态的核心句可以压缩为：

> 最好的答案永远是下一个；当前答案的价值，在于它把我们推向哪个更好的问题。

这句话不是无限发散的许可，而是研讨的内在动力。完整理解需要四层：

| 构成 | 含义 | 边界 |
| --- | --- | --- |
| 内在动力 | 最好的答案永远是下一个 | 当前答案不是终点，而是踏板。 |
| 方向牵引 | 被模糊但真实的更好感吸引 | 早期不要求先给出清晰价值指标。 |
| 搜索约束 | 在现实约束内寻找可能成立的状态 | 不是幻想、随意发散或概念堆叠。 |
| 本体边界 | 只定义搜索姿态 | 验证、PoC、淘汰和执行收束属于上层工作流。 |

这里的“模糊的最终完美状态引力”不是后续实际价值判断。它更像一种尚未概念化的方向感：某个方向在感觉上更对、更强、更可能通向本质问题。真正回答“实际价值是什么、是否值得投入、是否能形成 PoC”是后续流程的事，不属于研讨心态本体。

## 3. AI 默认进入执行交付的原因

可以抽出几个机制层原因：

1. AI 通常把“写方案、给建议、完成任务”理解为交付型指令。
2. 训练语料中的高频范式奖励完整、礼貌、结构化、可解释的答案。
3. 原生创新、反直觉判断和高价值盲区本来就是低频对象，模型不容易默认优先搜索。
4. 突破性判断经常伴随风险、未证实和非共识，AI 会倾向给出更安全的常规方案。
5. 如果用户没有显式要求“先找灵魂点，再写支撑”，AI 会把支撑结构误当成主产物。

这里的关键不是“AI 不会创新”，而是：

> 默认任务叙事把 AI 推向了完成，而不是推向寻找。

因此，关键不是让 AI 多写几个创意，而是调整任务协议：让 AI 每一轮都必须说明当前最有价值的未知、最可能突破的方向、这个方向是否仍在现实约束内、以及下一轮应该追问什么。

## 4. 创新研讨循环

本技能建议把创新型任务中的研讨部分拆成一个循环，而不是一次性交付。这个循环只描述研讨心态如何搜索高潜候选，不包含后续验证、PoC、淘汰或执行收束。

| 阶段 | 核心问题 | 输出 |
| --- | --- | --- |
| 1. 对象定界 | 我们到底在研讨哪类问题，不是在完成哪类方案？ | 研讨对象、非目标、评价轴 |
| 2. 模糊的最终完美状态引力感知 | 哪些方向在感觉上更对、更强、更可能通向本质问题？ | 候选吸引点、方向性直觉 |
| 3. 现实约束内生成候选 | 哪些可能性不是幻想，而是有机会存在于现实世界？ | 高潜候选、约束说明 |
| 4. 灵魂点搜寻 | 哪些判断、技术、视角或变量可能真正改变问题？ | 候选灵魂点列表 |
| 5. 反例与盲区攻击 | 这个点为什么可能是错的，普通方案漏了什么？ | 反例、风险、被忽略变量 |
| 6. 核心洞察卡 | 当前最锋利的一句话是什么？ | 当前核心判断 |
| 7. 下一问题 | 下一轮最值得问什么？ | 1 到 3 个研讨接口 |

这个循环的重点是顺序：先让模糊的最终完美状态引力牵引搜索，再用现实约束防止幻想化，然后寻找灵魂点和下一问题。若一开始就让 AI 写完整方案，注意力会被结构填充吞掉，导致真正高价值的部分被平均化。

## 5. 研讨型产物长什么样

一个好的研讨型 AI 产物不应该只有正文答案，而应该像一个“可继续推进的问题包”。

最小结构如下：

```markdown
## 当前最锋利判断
用一两句话写出本轮最可能有价值的判断。

## 为什么它可能重要
说明它改变了哪个问题定义、变量权重或行动路径。

## 备选灵魂点
列出其他候选点，并说明为什么暂时不如主判断。

## 反例与攻击
主动提出这个判断可能失败的原因。

## 现实约束检查
说明这个判断没有违反哪些关键事实、技术条件、人的行为规律或系统约束；如果存在明显约束冲突，要标出冲突。

## 下一轮问题
给出下一步最值得讨论的 1 到 3 个问题。
```

这类产物的好坏不由篇幅决定，而由三个指标决定：

1. 是否产生了比常规答案更锋利的问题定义。
2. 是否明确区分“核心判断”和“支撑结构”。
3. 是否仍在现实约束内，而不是自由想象。
4. 是否让下一轮讨论更容易进入高价值区域。

## 6. 运行规则

使用本技能时，遵守这些规则：

1. 先寻找模糊的最终完美状态引力，不急着要求清晰价值指标。
2. 把候选限制在现实约束内：不能违反已知事实、技术条件、人的行为规律或系统约束。
3. 不把研讨变成自由想象、概念堆叠或漂亮措辞。
4. 不在研讨心态本体中推进验证、PoC、淘汰或执行收束；这些属于上层工作流。
5. 不把“结果最后是否有价值”当成当下唯一标准；结果价值常有运气成分。
6. 可以观察“搜索质量”：是否出现新命名、新变量、新视角、现实约束检查和更好的下一问题。
7. 如果用户明确要执行、落地、验证或交付，先承认这已经进入上层工作流，再决定是否仍需要短暂研讨。

## 7. 输出建议

研讨型回答不强制固定格式，但默认优先包含：

```markdown
## 当前最锋利判断

## 模糊的最终完美状态引力
哪些方向感觉上更对、更强、更可能通向本质问题。

## 高潜候选
只列有机会存在于现实世界的候选，不列纯幻想。

## 反例与现实约束
这个方向哪里可能错，是否违反事实、技术、人或系统约束。

## 搜索质量判断
本轮是否产生了新命名、新变量、新视角或更好的下一问题。

## 下一轮问题
1 到 3 个最值得继续追问的问题。
```

如果任务已经进入执行交付阶段，可以把研讨输出压缩为“当前最值得保留的核心判断 + 下一问题”，然后交给执行流程。

## 8. 运行结论集

本技能的运行结论是一组互相约束的判断：

1. 研讨心态的核心动力是：最好的答案永远是下一个，当前答案的价值在于把讨论推向更好的问题。
2. 研讨心态本体是一种搜索姿态，不是执行工作流；它不负责验证、PoC、淘汰或执行收束。
3. 早期研讨依赖“模糊的最终完美状态引力”，不是清晰价值指标；这种引力来自感觉、直觉、品味和对“更好可能”的方向性吸引。
4. 研讨不是自由想象；所有候选必须仍在现实约束内，有可能存在于现实世界。
5. AI 在研讨阶段要优先自主探索更高潜候选，而不是当场证明候选最终有价值。
6. “结果价值”和“搜索质量”必须拆开看；前者当下很难判断且带有运气成分，后者只能通过新命名、新变量、约束检查、下一问题等侧面观察。
7. 人-AI 协作、状态记忆、验证/PoC/淘汰都是更高层问题，应由后续独立流程承接，不能倒灌进研讨心态本体。
