# Workflow Skill 生成模板

创建 workflow 类型 Skill 时，按此模板组织目录和主文件。主文件负责调度，细节下沉到 `workflow/`、`references/`、`templates/`。

使用这个模板时，我先写清这个 Workflow 为什么不能只靠临场发挥完成，再写执行契约。执行者心智叙事负责解释设计意图，Input / Goal / Guidance / Execution Mode 等规范化契约表述负责保证 Agent 能稳定照做。

## 目录结构

```text
${skill-name}/
├── SKILL.md                    # 主文件：流程骨架 + 调度逻辑
├── workflow/                   # 主控执行步骤的详细指导
│   ├── ${step_1_name}.md
│   ├── ${step_2_name}.md
│   └── ...
├── references/                 # 方法论、规范、外部参考
│   ├── ${reference_name}.md
│   └── ...
└── templates/                  # 产出物模板（如有）
    └── ${output_template}.md
```

## SKILL.md 主文件模板

```markdown
---
name: ${skill-name}
description: |
  WHAT：${做什么}
  WHEN：${何时启用}
---

# ${Skill 标题}

${一段话描述整体目标和适用场景}

${执行者心智叙事：说明我面对什么输入、为什么不能直接进入执行或生成、我会先保全/推演/拆解/校验什么，以及这个 Workflow 主要防止哪类误判或失控。}

第一步必须使用 `TodoWrite` 按以下阶段创建待办项：每个阶段一个待办，名称和顺序与本文档一致。

行文模式参照 [references/workflow-writing-styles.md](references/workflow-writing-styles.md)：需要建立阶段判断力、风险感知和承接关系的位置，按执行者心智叙事写；需要固定输入、目标、完成判定、返回格式和执行边界的位置，按规范化契约表述写。

### 阶段 N-1：校验

**执行模式**：主控执行

校验清单：
- [ ] ${校验项 1}
- [ ] ${校验项 2}
- [ ] ...

校验全部通过后进入测试阶段。任何校验失败，回退到对应步骤修复。

### 阶段 N：测试

**执行模式**：Subagent 委派（必须）

**Subagent 配置**：
- `subagent_type`: `generalPurpose`
- `readonly`: ${根据测试类型决定}

**派发 Prompt**：

\`\`\`markdown
请按照以下测试用例实际执行验证。每个用例必须返回执行操作、预期结果、实际结果、结论和证据。

## 测试用例

${测试用例列表}

## 执行要求

你是本次交付的独立质量关卡。请按验证标准实际执行测试，并用证据支撑结论。

要求：
- 每个测试用例必须实际执行，不允许只基于代码阅读推断结果。
- 发现问题时，必须给出复现步骤、实际现象和可能涉及的位置。
- 如果所有用例都通过，请额外检查一个最可能出问题的边界场景。
- 输出先列失败项和风险，再列通过项；没有失败项时，也必须说明证据为何足以支撑通过结论。

## 场景化补充要求

${按需从 references/pua-testing.md 的「通用测试片段 / 功能测试片段 / 回归测试片段」中选择并填入；涉及已有功能时必须包含回归测试片段。}

## 报告格式

请按 [references/pua-testing.md](references/pua-testing.md) 中的测试报告标准格式输出。
\`\`\`

如果测试未全部通过，回退修复后重新测试，直到全部通过。

## 注意事项

1. 严格遵守阶段顺序，禁止跳过任何阶段。
2. ${其他注意事项}
```
