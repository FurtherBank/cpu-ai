# 工作流架构设计方法论 (Workflow Architecture Methodology)

**核心设计哲学：将工作流设计为一条带有“强制 Todo 锁”的组装流水线**
如果说单步环节（Step）是“如何做对一件事”，那么工作流设计就是“如何控制全局复杂度、控制上下文衰减，并确保最终交付物不偏离目标”。

我们将复杂工作流视为一个**带有多个检查点（Checkpoints）的状态机流水线**。推导工作流结构时，需要遵循以下核心原则：

---

## 一、 终局思维与阶段划分 (Phase Division)

**目标**：通过“上下文隔离”控制认知负荷，防止 AI 在长程任务中因信息过载而产生幻觉。

### 1. 划分准则：上下文的“折旧”与“污染”
- **寻找自然断点**：识别任务中认知负荷的切换点（如：从“务虚”的设计到“务实”的编码）。
- **隔离冗余细节**：如果阶段 A 的中间尝试过程（如报错修复、废弃方案）对阶段 B 的执行没有直接帮助，则必须拆分为独立的子工作流。
- **定义阶段契约**：每个子工作流必须有明确的 **输入（硬依赖）** 和 **输出（阶段性交付物）**。下一阶段只能消费上一阶段的“精炼产出”，而非全量对话历史。

---

## 二、 风险左移与防御性编排 (Risk Left-Shifting)

**目标**：在错误放大之前将其拦截，通过“制衡机制”打破 AI 的证实偏差。

### 1. 强制插入检查点 (Checkpoints)
在每个关键交付节点后，必须强制插入校验或测试环节，绝不允许“带病交付”。

- **校验环节 (Validation) —— 静态自检**：
  - **定位**：低成本、快速的形态检查。
  - **执行**：允许**主控 Agent** 通过读取文件、正则匹配、AST 检查等方式自行执行。
  - **目标**：确保产出物符合 Schema、规范和逻辑自洽。

- **测试环节 (Testing) —— 动态验证**：
  - **定位**：高成本、真实环境的功能验证。
  - **执行**：**必须委派给 Subagent**（如 `shell` 或 `browser-use`）。
  - **核心逻辑**：利用 Subagent 的“上下文隔离”来吸收执行噪音（如海量编译日志），利用其“客观性”打破主控 Agent 的“证实偏差”幻觉。

---

## 三、 主控 Agent 的角色重塑 (Orchestration vs. Execution)

**目标**：将主控 Agent 从“全干工程师”转变为“车间主任与状态维护者”。

### 1. 职责边界
- **主控 Agent (车间主任)**：负责把控进度、维护核心状态（DoD、修改路径）、组装并派发上下文。**严禁主控 Agent 亲自执行高污染、高风险的原子操作（如写代码、跑大型测试）**。
- **Subagent (专业技工)**：负责在隔离的上下文中执行具体任务，并返回精炼后的结果。

### 2. 强制 Todo 映射 (Todo Mapping)
在 Skill 的执行指令中，必须强制要求主控 Agent 使用 `TodoWrite` 工具将预定义的工作流阶段映射为待办事项。这能防止 AI 在执行过程中跳步或自我发挥。

---

## 四、 契约化派发与归档 (Contract & Archiving)

**目标**：确保工序间的信息传递不失真，并实现决策的可追溯性。

### 1. 模板化派发 (Template-Driven Prompting)
在唤起 Subagent 时，主控 Agent 必须遵循 Skill 中预设的 **Prompt 模板**。
- **要求**：必须将前序阶段的产出（如【完成标准】）原封不动地填入模板占位符，确保 Subagent 接收到的指令是未经主控“二次加工”的原始契约。

### 2. 终局归档 (Traceability)
工作流的最后一个环节必须是**“输出功能记录”**。
- **逻辑**：将本次任务的核心决策（DoD）、修改路径、测试结论、遗留风险持久化到项目的 `docs/` 目录下。这不仅是为了人类 Review，更是为了给未来的 AI 留下“软上下文”，解决会话易失性问题。

---

## 五、 设计推导 SOP (Step-by-Step)

1. **定义终局**：明确最终要交付的物理形态（文件、服务、文档）。
2. **拆分阶段**：根据“上下文污染率”划分子工作流，定义每个阶段的输入输出契约。
3. **植入防御**：在每个阶段末尾插入校验（主控）或测试（Subagent）环节。
4. **细化环节**：对每个子工作流内的具体 Step，应用 `task-step-schema.md` 推导四要素。
5. **定义模板**：编写主控派发给 Subagent 的标准 Prompt 模板。
6. **设计归档**：定义最终输出的记录文件格式与存放路径。
