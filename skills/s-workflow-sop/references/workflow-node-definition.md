## 工作流构成定义
这里在概念上明确工作流的形式化要素构成：
- **工作流**：可以由若干**子工作流**和**环节**组成。  
其中**子工作流**也是工作流，因此可以继续递归包含**子工作流**和**环节**。
  - 每个工作流的执行报告，必须要落位到一个报告文件中
- **环节**


## 工作流环节要素定义

- 输入信息：包括执行这个任务的必须入参，以及必要上下文背景信息的补充(以索引形式，agent 实际使用时获取，或者完整内容形式补充。优先考虑索引形式，让 AI 消费时读取，这样满足渐进式消费原则)
- 目标要求：需要刻意阐述一次：这个任务 要做什么，以及要达成的目标是什么
  - 特性要求：对于交付成果中特定属性或性质的明确要求。如要通过 xxx 检验，满足 xxx 性质
- 工作依据：我完成这个任务的依据内容。可能包括：
  - 流程规范依据（如需要）：按照特定流程/标准操作完成任务
    注意，这部分是可选的，对于偏执行的工作流，大概率会有这部分，如果是让 agent 执行一些创造性工作，则很可能没有。
    举例：执行特定的命令完成操作、输出遵从特定格式模板、要求按照指定 SOP 手册完成操作
  - 方法论依据：完成这个任务的方法论/经验/遇到 xx 情况如何处理
    举例：理论指导、经验直接说明等
- 是否通过 subagent 执行：取决于任务性质，是否强依赖执行过程上下文。
  - 如果强依赖执行过程上下文，不要走 subagent，但也不需要一开始把所有流程的 prompt 信息交给 agent。
    而是根据渐进式披露的原则，声明流程名称列表，每一项直接给出流程具体指导(包括上面那些信息)的索引
