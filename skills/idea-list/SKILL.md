# idea-list Skill

## 使用场景

当用户通过自然语言描述一个想法、灵感或进展时，本 Skill 负责将该输入拆解并同步到 `.cursor/rules/idea-list.mdc` 文件中，完成灵感列表的增删改查。

常见触发场景：
- "记录一个灵感：做一个博客摘要工具"
- "把 idea-list skill 标记为进行中"
- "查看当前有哪些灵感"
- "更新「博客摘要工具」的状态为完成"

## idea-list.mdc 文件位置

文件路径：`/Users/a1111/Desktop/ai-workspace/.cursor/rules/idea-list.mdc`

该文件是 Cursor alwaysApply rule，包含灵感列表的消费策略和条目内容。

## 灵感条目模板

每个灵感条目使用以下 Markdown 格式，追加在文件末尾的 `<!-- 在下方按模板追加灵感条目 -->` 注释之后：

```markdown
## [灵感名称]

**状态**：待探索
**来源/背景**：这个灵感从哪个思考/场景中来，解决什么问题或抓住什么势头

**阶段**：
- **阶段 1：{名称}**
  - 目标：
  - 判断（何时算完成）：
  - 依赖：
- **阶段 2：{名称}**
  - 目标：
  - 判断：
  - 依赖：

**备注**：其他注意事项、参考资料、来源等
```

状态枚举（仅使用以下四种）：`待探索` `进行中` `搁置` `完成`

## 各类输入的处理方式

### 新增灵感

识别关键词：记录、新增、添加、想法、灵感、做一个、开发……

处理步骤：
1. 从用户输入中提取灵感名称（简洁明确，5~20 字）
2. 从用户输入中提取背景描述（如有）
3. 合理推断阶段拆解（若用户未指定，默认写 1~2 个探索阶段）
4. 读取 `idea-list.mdc` 当前内容
5. 在文件末尾（`-->` 注释块之后）追加新条目
6. 写回文件

### 更新状态

识别关键词：标记为、改为、更新状态、完成了、搁置……

处理步骤：
1. 从用户输入中提取灵感名称（模糊匹配标题）
2. 从用户输入中提取目标状态
3. 读取 `idea-list.mdc`，找到对应的 `## [灵感名称]` 段落
4. 将该段落中 `**状态**：xxx` 替换为新状态
5. 写回文件

### 查询列表

识别关键词：查看、列出、有哪些、当前……

处理步骤：
1. 读取 `idea-list.mdc`
2. 提取所有 `## [灵感名称]` 及其 `**状态**` 字段
3. 以列表形式输出给用户

### 模糊/综合输入

若用户输入包含多个意图（如"把 xxx 标记为完成，并记录新灵感 yyy"），依次执行对应的处理步骤。

## 默认行为

若输入意图不明确，默认将其作为**新增灵感**处理：
- 将输入本身作为灵感背景
- 从输入中提取最简洁的主题作为灵感名称
- 填充基础模板后追加到列表末尾
