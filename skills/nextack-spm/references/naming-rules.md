# SPM 命名规则参考（PC Web SPM）

本文档定义 **PC Web 中台** 场景下 `spm="模块名-按钮名"` 等中文命名的约定与反模式。实现埋点时以本文件与同目录上一级的 **`SKILL.md`** 为准。

## 核心格式

```
spm="模块名-按钮名"
```

模块名与按钮名用 **英文连字符 `-`** 分隔；两侧均为 **简短中文** 业务语义，避免 HTML 标签名或无意义占位。

## 模块名（spmc）

- 同一视觉区块 / 业务组件内 **共用同一模块名**，便于模块粒度聚合。
- 用 **业务语义**，不用技术名（避免 `div区域`、`container1`）。
- 长度建议 **2～6 个汉字**。

| 代码语境 | 模块名示例 |
|----------|------------|
| 顶区运营位 | `头图`、`banner模块` |
| 交易相关 | `购买区域`、`交易区域` |
| 列表区块 | `任务列表` |
| 顶栏侧栏 | `导航栏` |
| 弹窗 | `确认弹窗`、`弹层` |
| 表格行卡片 | `产品卡片` |
| 筛选区 | `筛选栏`、`搜索栏` |
| 中台常见「实例/告警/文件」页 | `实例详情`、`告警规则`、`文件列表` |

## 按钮名（spmd）

- 用 **动作或导航结果** 描述点击。
- 有可见文案时 **优先贴近文案**（可略缩短）。
- 图标按钮写动作，如 `关闭`、`显隐切换`。

| 元素 | 按钮名示例 |
|------|------------|
| 立即购买 | `立即购买`、`买入` |
| 查看详情 | `查看详情` |
| 关闭 | `关闭` |
| 确定/取消 | `确定`、`取消` |
| 删除行 | `删除` |
| 外链协议 | `用户协议`、`隐私政策` |

## 组合示例

```jsx
<div spm="banner模块-立即领取" onClick={handleClaim}>立即领取</div>
<Button spm="购买区域-买入" onClick={handleBuy}>买入</Button>
<a spm="底部信息-用户协议" href="/agreement" target="_blank">用户协议</a>
```

## 分屏（列表 / 循环）

在 `.map` / `forEach` / `v-for` 等 **重复项** 上：

```
spm="任务列表-任务项_N"
spm="产品卡片_N"
spm="列表_${idx}-操作按钮_${idx}"
```

- `_N`：工具申请后常回填为 `` `..._${index}` ``，需在代码中改为 **实际循环索引变量**（见 `SKILL.md`「开发验证工作流」中的「分屏回填修正」）。

## Tracert API 埋点（非 DOM）

程序化上报使用 `Tracert.click` / `Tracert.expo`；申请编码后替换为平台返回的 CD，并保留注释或描述便于检索。

## 反模式

| 不推荐 | 推荐 | 原因 |
|--------|------|------|
| `spm="div-点击"` | `spm="头图-立即领取"` | 非业务语义 |
| `spm="aaa-bbb"` | 有业务含义的词组 | 无法分析 |
| 同区块多个模块名 | 同区块统一模块名 | 聚合口径分裂 |
| 过长描述 | 简短动宾结构 | 可读与平台展示 |
