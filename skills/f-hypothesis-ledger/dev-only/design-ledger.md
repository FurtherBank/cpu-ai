# 设计账：机制追溯、裁剪记录与有效性假设

本文件不供运行期 agent 消费，供维护者修订 skill 时使用。SKILL.md 的每条条款都应能在此反查机制依据；修订条款前先核对其机制出处是否仍成立。

## 机制追溯

上游机制（前置研讨封口结论）：

- M1 判断型假设参与定义自己的验证标准（评价框架内生化），其目标是下游思维行动链质量而非命题真值。
- M2 「照做后没感觉出问题」四维零载荷：无比较项、阈下不敏感、标尺内生、无归因。
- M3 迷宫机制 = 可辨识性坍塌（内部自由度增速 > 独立外部约束增速）+ 归因路由权在利益方 + 无补丁记账；识别判据必须轨迹性，快照判据误杀孕育期。
- M4 判断合格持有形态 = 结论与失效条件同体存储；出处提取时间窗在形成时刻；裸语句四种静默失效（越界复用、僵尸判断、传递洗白、冲突不可仲裁）。
- M5 统一原则「事前+外生」：评价要素时间上前移到污染前、来源上外移到判断网络外。
- M6 单主体闭环不可自救：机制效力依赖至少一个主体控制之外的环节。
- M7 边界反例：已校准直觉、承诺构成型判断、元层被吞不可完全防御。

|---|---|---|
| 三本追加式账 + commit 时间戳权威 | M5、M6 | 事后改写、再合理化；时间差与 git 历史是纯文本协议能借到的外生性 |

## 裁剪账

| 被裁内容 | 失守的面 | 取舍依据 |
|---|---|---|
| 全部机制原理阐释 | 无（追溯保留在本文件） | agent 不需被说服，只需被正确触发 |
| 分歧预测对账作为独立条款 | 工作档判断无对手比较，反事实盲区只在地基档被补 | 全档对手对账维护成本不可持续；地基档承重最大 |
| 外部角色（第三方 agent 运行期评审） | 运行期无独立裁决者，只有事后账面审计 | 初版按单 agent 挂载裁掉；后经注意力/激励分析复审，以「结算形态」部分恢复——不是运行期评审理论内容，而是按节拍从产出流检出锚点事件并记账，理论评价仍只属账面审计 |
| 评估感真值检测 | agent 否认产生评估感即绕过该触发器 | 内部状态原则上不可检测；周期审计兜底，接受降级 |

| 元层完全防御 | 一切条款可被上移一层的动机性推理吞噬 | 已证不可达上界；对策止于「吞噬须留可检测代价」 |
| 反馈稀缺领域的专门降级模式 | 纯思辨零反馈域中斜率门控与现实绑定退化，孕育期与迷宫可能长期不可区分 | 无可靠机制可写；标注为已知边界而非假装覆盖 |

## 有效性假设（本 skill 有效——只要以下仍成立）

- 宿主环境有 git 与可落盘、追加式的文件载体；无此载体时协议退化为劝诫，应弃用而非硬跑。
- 产出侧配合条目 ID 引用规则；账外复用在产出不带 ID 时不可见，这是档位系统的单点依赖。
- 研究域存在至少偶发的可落盘现实反馈事件；长期零反馈域中孕育期门控与现实绑定失效。
- 台账字段被如实填写；写入端造假只能靠事后验收断言的分布异常捕获，无运行期防御。
- 目标 agent 会在触发锚点处自我观测到「我正在改假设/复用结论/归因外部」；锚点识别本身失灵不在协议覆盖内。

## 注意力与激励复审（2026-07-18 第二轮）

对「留档协议破坏研究注意力流」质疑做了隔离对抗分析（控方论证＋中性成本模型，独立 subagent）。裁决：均匀征税指控不成立——协议无任何以纯思考为触发的条款，税集中在采纳/复用/异常节点（漏斗式）；但四项指控成立并已处置：

- 异常登记逆向激励（自报制下诚实比沉默贵）→ 结算形态（锚点外化为文本事件，检出权移出利益方）。残留：从未落笔的异常仍不可见，列为首要未解风险，实跑观测异常登记率。
- 零反馈域审计螺旋 → 边界节显式弃用条款（原仅在本文件记载）。
- 评估感与工作层剪枝评估边界模糊 → 正文豁免分支取舍类评估。
- 「新假设产生」可被读成每个念头 → 正文钉死为写入产出为准。

控方两处夸大已驳回未采纳：链内组合推演逐步征税（实际中间判断走默认档零 commit）；合法概念重构被枪毙（泡利式拯救假设自带独立新预测，走预注册正门即合规）。结算形态新增失效面：结算方检出精度（漏报回到自报制、误报造摩擦）、结算延迟给失效命中留一个周期的污染窗口——均已由验收断言「检出完备性」「弹回闭环」「覆盖连续性」兜底，实跑校准 S。

### 结算形态落正文后的复验（同日第三轮）

结算形态写入正文（缺省化）后，重跑对抗评估＋零上下文执行审查。对抗方裁决：打断类伤害基本不成立（检测外移＋周期内补写豁免），残余为结算税（吞吐档）——「这一点」的设计目标达成。新暴露的激励漏洞与执行分歧一并修入正文：

- 安静账面必然告警（健康低异常研究 <5 异常 → 连续不可判定 → 告警）→ 新增审计结论「窗口未满」：账面正常推进而异常不足窗口时不进告警连击、也不算「正常」。
- 默认档跨语境复用对结算方不可见（载荷只含 active 条目）→ 新增「无来源前提」检出项：未经本段推导又无 ID 的前提性断言按裸语句条款弹回，静默复用文本可检。
- 异常相位法律危险化（诚实记录异常比沉默贵）→ 新增纯异常条目：核对未命中且不修改假设时低成本留痕；纯异常无补丁反而使补丁增速指标不命中，激励反转到位。
- 草稿豁免 vs 思考落盘矛盾（影子工作区）→ 显式草稿区条款：草稿区不触发分型，其语句被结论区作前提时按新判断处理。
- 执行分歧修补：载荷加协议文本；失效命中直转退出的告警条目规则；工作档补丁与审计覆盖不分档；字数口径排除账目；不适用判定须现实绑定实际执行过。

接受未修的残余：空洞失效条件可句法通过（效力上限声明覆盖，实跑回填）；评估感语句可被委婉语规避（周期审计兜底＋结算方检语义不检措辞，部分缓解）；孕育期 N 减半对合法慢兑现命题的压力（反自我续期是有意设计，实跑观测误杀率）；结算税本身（协议目标以质量换吞吐，方向正确）。

## 挂载升级：两态化与 discussion-mindset 耦合（2026-07-19）

用户决定把挂载判定升级为规则级心智（desc 自触发＋被心态类 skill 必然引用），协议本体不外迁。落法：正文新增首节「挂载判定」——加载即武装态（唯一义务＝标注纪律，「资格凭证」句给其事后兑现的机制牙齿：建账时无标注者按裸语句处置），三条可数绊线切建账态（写下跨语境复用的那一刻／动手修改已落盘假设的那一刻／无人在环累计落盘达 S 字）。绊线检出的是动作是否发生（文本层事实），不是重要性自评，与第一原则不冲突且正文括号句显式声明。存量判断默认档封顶补办：堵死「拖到绊线前顺手确认关键判断再入场」的时序套利——无论何时入场只能从默认档起步，复用仍须现场预注册。淘汰的备选：绊线用「预计超过节拍」（含预估，违反可数原则，改为事后累计字数）；「想修改假设」（念头不可数，改为动手修改已落盘文本）；存量完整补办（造预注册豁免通道）；存量按补写全废（武装态失去意义）。
