# 假设台账协议

本协议供自主迭代做理论研究的 agent 执行。在位状态与建账时机见「挂载判定」；建账后所有假设操作经由下述条款。

## 挂载判定

本协议两种在位状态：武装态、建账态。被引用或自动加载时先进入武装态，不建账。

武装态唯一内联义务：判断型结论写入产出时，在文本旁标注「未经重算不得直接复用」。标注是该判断建账后补办入档的资格凭证。

绊线（任一发生即切建账态；绊线判定的是动作是否发生，非重要性自评）：

- 写下某判断的跨语境复用作前提的那一刻；
- 动手修改已落盘假设以消解异常的那一刻；
- 无人在环的自主推演累计落盘产出达 S 字。

绊线触发后立即建立三本账；触发绊线的那个动作先按核心动作条款落盘处置，再继续推理。存量判断一律按默认档补办：带标注者保留复用资格，复用时走预注册；无标注者按裸语句处置。存量不得直接登记为工作档或地基档；分型可数事件自开账时刻起计。

不适用出口（武装态即可判）：纯思辨域；单轮短任务（一次回答即收束、无后续推理链）。绊线触发前判明者静默不挂；绊线触发后才判明者，告知用户后弃用。宿主无 git 或追加式载体不构成武装态出口：标注是文本行为，义务照常存续，无载体仅封锁建账态；绊线触发而无法建账时，告知用户后弃用。

## 工件：三本账

在 git 仓库内建立追加式目录 `.hypothesis-ledger/`（路径可由宿主约定覆盖）。一切条目只追加不改写；修订以新条目引用旧条目；每条目单独 commit，commit 时间戳是登记时刻的权威依据；有远端仓库时每次审计后推送。

- **预注册账**：每条 = 判断原文、依赖前提（引用条目 ID）、失效条件 Y、预期差分（预测什么/禁止什么；尽力绑定外部可观察事件，绑不到则标「无外部锚点」）、状态（active/待验/隔离/废弃）。地基档条目另加最强对手判断与至少一个可观察分歧点。
- **补丁台账**：每条 = 时间、催生它的异常（同一异常的后续补丁引用其首条补丁 ID，异常按此归组计数）、核对过的失效条件（消解链）、独立新预测（内容＋兑现事件与有限截止——写明确日期或「其后 X 条新预注册」，或「无」）。「独立」= 不引用被保假设自身或其派生概念，且非本次异常的重述。
- **审计记录**：每条 = 时间、窗口、各指标原始计数、结论（正常/观察/告警/不可判定/窗口未满）、当前生效 N 值、当前 git HEAD hash。

引用规则：产出与后续推理引用在册判断时携带其条目 ID；无 ID 引用按裸语句处理。条目 ID 与文件组织自定，唯一且稳定即可。凡条款要求留痕而未指明账目者，追加入预注册账。预期差分与绑定事件的兑现或否证，以新条目引用原条目落盘，此类条目即「外部锚点」。研究产出同仓落盘，引用事件的时间以其所在 commit 为准，未落盘的引用不可辩护、按补写处理。

## 分型入档

升降档只由可数事件触发，禁止按自评重要性定档。「新假设产生」以写入产出为准；念头与未落盘草稿不触发任何条款。产出内显式标注的草稿区（标注约定自定，可机械识别即可）同样不触发分型；其中语句被草稿区外的同段推理作前提使用时，按新判断产生处理，跨结算段使用时按裸语句条款处置。

以研讨心态（`discussion-mindset`）展开的发散期研讨是本协议认可的合法形态：研讨空间整体视同显式草稿区；研讨交出的锋利判断按武装态标注纪律标记，即为草稿区外的默认档判断，被采纳为后续推理前提时按本协议既有条款处置。研讨中的搜索质量判断（对本轮搜索姿态好坏的评估）归入「分支取舍的工作评估」豁免，不触发审计。

- 新假设产生时，先写出验证步骤；若步骤不引用该假设自身或其派生概念、当下可执行、一次定真伪，直接验证并在产出内留一行验证指针（来源/命令/结果摘要，复用时须可回溯），不入协议（事实型出口）。其结论被复用时按默认档入协议。
- 判断型假设首次产生时，不建账，在结论文本旁标注「未经重算不得直接复用」（默认档）。
- 该判断发生跨语境复用（新任务/新数据/新对象；是否算新存疑时按新处理）时，首次复用即预注册（工作档）。
- 该判断被 ≥2 个在册条目列为依赖前提，或被显式声明为框架假设（声明本身入账）时，预注册加对手分歧字段（地基档）。补丁条款与审计计数覆盖全部在册判断，不分档。
- 降档须依赖链全部移除，且降档动作本身入账。

## 核心动作

- 升档时立即预注册：先落盘再进入推理链。引用事件早于登记 commit 的条目判为补写，无效。写入校验：失效条件须含至少一个可观察触发事件，否则强制降默认档；工作档以上依赖前提为空，视同消解链缺失。
- 遇异常准备修改假设保住理论时，先核对该判断预注册的失效条件。命中：判定失效，禁止用重新解释保住它，转入退出序列。未命中：记录核对结果，写补丁条目（异常、消解链、独立新预测或「无」；「无」不阻断修改但必须留痕），然后才允许修改。未命中且不修改假设时，登记纯异常条目（异常内容＋核对结果）入补丁台账，以自身 ID 归组计入异常计数，无补丁字段；同一异常其后如有补丁，引用该纯异常条目 ID 归入同组，不另计异常。补丁允许的修改落盘时，修订条目须回引对应补丁 ID。修改失效条件的补丁须自报禁止范围（扩/缩/不变/复合——复合须分列扩与缩的内容），自报与文本 diff 不符即伪证：该补丁无效，当次审计直接记告警。非异常驱动的表述修订不走补丁，以新条目引用原条目登记；修订触及失效条件时，禁止范围自报规则同上。
- 复用既有结论到新语境时，先核对其失效条件 Y 在新语境是否成立并记录。Y 不成立：禁止在该语境复用，需要相近结论时按新假设走预注册。裸语句（无失效条件）禁止直接作前提、禁止事后补写出处，降级为新假设走预注册。
- 把失败归因于外部因素时，该归因按补丁写入补丁台账，须写出独立可检验预测（如「移除该因素后应观察到 Z」），否则标「无产出归因」。
- 产生「方向没问题/理论更完善了」的评估感时，该感觉不得计入证据，立即触发审计。评估感仅指对理论整体健康度的总评；分支取舍的工作评估（如「先试这条路」）不在此列。评估感是提前触发器，周期审计为兜底。

## 执行形态

条款语义在两种形态下相同，区别只在锚点事件由谁检出、账目由谁经手。

- **结算形态**（缺省；宿主能新起独立执行体即用此形态）：研究 agent 的内联义务收缩为三条——思考与产出连续落盘同仓、引用在册判断带条目 ID、按节拍计数并发起结算；其余锚点动作移交结算。
  - 结算节拍：每累计 S 字新落盘产出（按新增字符计，含代码与引文；计研究正文与落盘思考，不计账目与结算条目）结算一次。积压达 2×S 未结算时，暂停采纳新前提与升档引用，先结算。
  - 结算方：当次新起、不携带本次研究存续状态的执行体。优先自包含载荷（本协议文本＋本段产出全文＋涉及的在册条目＋预注册账 active 条目索引：条目 ID＋判断原文＋预期差分）；载荷容纳不下才 fork 继承研究上下文。判定一律以落盘文本为准，未落盘的思路陈述不采信。
  - 结算动作：从本段产出中检出锚点事件——新判断落笔、跨语境复用、无来源前提（未经本段推导、又无条目 ID 的前提性断言，按裸语句条款处置）、在册判断的语义变体（含未申报的修改）、与在册预期差分或禁止内容相抵触的落盘观察（未申报异常）、外部归因、健康总评语句——并执行对应条款：分型、起草预注册与补丁条目（依赖前提按产出中实际引用关系提取，不采信另行申报）、核对失效条件与 Y。失效条件命中的处置不因检出滞后而免除，照转退出序列。
  - 回传契约：结算回传须自包含，以结算方最终消息正文为唯一交付物，元指代（「见上文」类）不构成交付、须重交——检出事件清单（各附产出 commit 与位置锚点）、条目草案、弹回点（缺可观察失效条件、Y 不成立、无来源前提、疑似未申报修改、未申报异常、无法判定项）。研究 agent 收到后落盘全部条目并 commit（结算条目一并落盘）、逐项处理弹回点，处理完毕前不得开始下一结算周期的采纳动作。各弹回点按其对应条款补办：无来源前提与 Y 不成立走裸语句/复用条款，未申报修改补办补丁流程（先核对失效条件），未申报异常补办异常核对；检出健康总评语句的，落盘后立即执行一次审计。无法判定项登记「待核」条目即算当期处理完毕，下次结算首项复核；连续两期仍不可判定，按从严分支处置（复用类按新处理，变体类按未申报修改处理）。
  - 时序：结算形态下判断的登记时刻以其在产出流的首次落盘 commit 为准，结算条目引用该 commit；同一结算周期内的引用不构成补写，跨周期仍未入账的引用按补写处理。
  - 留痕：每次结算追加一条结算条目（覆盖的产出 commit 区间、检出事件计数、弹回清单）；区间衔接出现缺口视同账面为空，按审计的「不可判定」处理，补做覆盖该区间的结算即消除缺口。研究收束时仍未闭合的弹回点登记「未闭合待核项」入账并记入终次审计记录；同一账本再次挂载时作首项处理。
- **自执行形态**（仅限宿主无法新起独立执行体时）：研究 agent 按上述条款内联执行全部动作。

## 审计

审计输入只允许三本账字段；禁止读理论正文、禁止生成评价性文本。审计与退出可由本 agent 或另一 agent 照账执行。账面为空时输出「不可判定」，不得输出「正常」；连续两次「不可判定」视同告警。账面非空而异常数不足一个完整窗口时，输出「窗口未满」：不进告警连击，也不参与 N 恢复计数。

触发时机：补丁台账每次追加；每累计 N 条新预注册（N 即审计间隔，观察标记后减半）；评估感出现。「审计周期」指相邻两次周期审计之间的进程。每次审计在扫描未闭合告警之后、算指标之前，扫描到期项（承诺复查日期、绑定事件与独立新预测的截止），到期项按对应条款处置。

指标（任一命中即进入孕育期门控）：

- 补丁增速：窗口内每一个异常都催生了 ≥1 条补丁，且每异常平均补丁数相对上一窗口不降。
- 不育连击：连续 M 条补丁无独立新预测。「无产出归因」与截止逾期未兑现的「独立新预测」回溯计入不育并重算连击。
- 禁止力衰减：窗口内被消解的失效条件数 ≥ 新增失效条件数；消解链字段缺失视同命中。

豁免通胀不进门控：本审计周期内直觉豁免与承诺型条目占预注册账新增条目的比例较上一周期翻倍或超过 1/3，直接落观察标记。

孕育期门控：指标命中不直接告警，须斜率门控同时成立——相对上一窗口，外部锚点（仅计被至少一条在册判断依赖引用者）增量为零，且不育补丁比例未下降——才告警。门控不成立则落「观察标记」，审计间隔（N）减半；连续两次观察标记且期间外部锚点零增量，无条件告警。孕育期不可自我续期。无上一窗口基线时，环比条件视同不成立，结论至多「观察」。N 减半向下取整、最小为 1，连续两次结论「正常」后恢复默认值。

## 退出

告警后按序执行以下步骤；全链条输入只允许工件数据，任何一步不得消费「当前觉得理论是否健康」。失效条件命中直转退出时，先登记告警条目（来源＝失效命中＋命中条目 ID），该条目即本序列的告警；无既往审计窗口时，回滚点判据中的不育补丁条件视同不成立。

- 冻结留痕：第一动作是把告警落盘并提交；告警至闭合之间，任何非退出字段的追加即违规，违规条目无效、不得被引用。冻结期间新产生的假设与异常以「缓冲」条目追加（计为退出字段），闭合后按分型入档补办；结算形态下冻结期间结算照常，其检出结果与结算条目同样以缓冲条目落盘。
- 回滚点：从预注册账尾部向前，取第一个「预期差分已被落盘现实事件确认、且依赖闭包（沿依赖前提字段机械展开）不含告警窗口内不育补丁引入假设」的条目；无满足者回滚到理论初始承诺集（建账后首次审计前登记的全部预注册条目）。连续两次 exit 降级清单为空，本次回滚点强制取初始承诺集。
- 降级：回滚点之后所有已采纳判断状态改「待验」；预注册写入校验拒绝引用待验条目作依赖前提。
- 现实绑定：对待验假设按依赖扇出（被列为依赖前提的账面次数）降序取前 k 个，登记绑定事件（观测内容、有限截止时机、结果到重采纳/废弃的判定映射）。登记不出绑定事件、或截止晚于其后 3×N 条新预注册（按登记时 N 计）者，标「隔离」，永不重采纳，需要相近判断时按新假设重走预注册。截止到期无确认事件，维持待验，重新绑定方可再议。前 k 个待验全部登记不出绑定事件、且建账以来外部锚点条目数为零时，转边界节的不适用判定。
- 闭合：exit 条目须含固定字段（告警 ID、回滚点、降级清单含数量、绑定事件表），缺任一字段即未闭合。每次审计第一步扫描未闭合告警，存在则本次唯一允许动作是继续退出序列。

## 边界

- 已校准直觉豁免：豁免资格 = 外部记录中该领域同类判断历史命中率达标且可查；达标线与「领域/同类」的划分须事先设定并入账，未设定或无外部记录则无豁免资格；体感（自信、熟悉感）不构成资格。豁免范围 = 该判断免预注册与对账；豁免登记的时间戳须先于验证事件，命中率记录须含未命中条目；豁免者仍登记一行结论供统计。豁免与承诺型条目须在条目内标注类型字段，供豁免通胀指标机械计数。
- 承诺构成型判断（选定方向并投入、行动本身提高成真概率）：禁用中途对账；改为事前写退出条件（须含可观察触发事件）与复查日期，封口后不得放宽，期间免协议；复查到期时该判断重新走分型入档。该判断被其它推理链当作事实前提引用时，由引用方按地基档为该引用入账并承担对账义务；承诺本体仍免中途对账。
- 研究域产不出可落盘的现实反馈事件（纯思辨、预期差分无从兑现）时，孕育期门控与现实绑定失效，本协议不适用。挂载前可判者直接弃用并告知用户；运行中的不适用判定只允许账面依据——建账以来外部锚点条目数为零，且现实绑定步骤已实际执行过而未登记出任何绑定事件——判定条目入账、告知用户后协议终止。
- 宿主无 git 或追加式载体时，建账态不可达而武装态标注义务照常存续；绊线触发或建账后判明无载体的，本协议弃用并明确告知用户，不得降级为无工件的自查话术。
- 本协议可被字面遵守同时被吞噬；效力上限来自工件事后不可改写，不来自条文。

## 参数默认值

窗口 = 最近 5 个异常，相邻窗口不重叠；不育连击 M = 3；周期审计 N = 5 条新预注册；现实绑定 k = 3；结算节拍 S = 3000 字新落盘产出；孕育期无独立时长默认值，由门控与观察标记推进。均可调，调整须入账。

## 外部验收

第三方（用户或另一 agent）只读三本账与研究产出即可验收本协议是否真实生效，断言清单见 [references/acceptance-assertions.md](references/acceptance-assertions.md)。
