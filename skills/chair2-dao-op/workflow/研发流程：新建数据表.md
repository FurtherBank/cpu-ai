# 研发流程：新建数据表

## 研发流程：新建数据表

### Step 1：在 `entity/` 目录创建 Entity 文件

**文件路径规则**：`app/module/${moduleName}/entity/{EntityName}.ts`

- `{EntityName}` 使用大驼峰，如 `OpenapiOrder`
- 文件名与类名保持一致

**完整模板（复制后按需修改）**：

```typescript
// app/module/${moduleName}/entity/OpenapiOrder.ts
import { Column, ColumnType, Index, IndexType, Table } from 'chair/tegg/dal';

@Table({
  name: 'openapi_order', // 数据库表名（snake_case）
  comment: '订单表', // 表注释，必填，便于数据库管理
})
// 唯一索引：type 指定 IndexType.UNIQUE
@Index({
  name: 'uk_order_no', // 索引名：唯一索引以 uk_ 开头
  type: IndexType.UNIQUE,
  keys: ['orderNo'], // 填属性名（camelCase），框架自动转 snake_case
})
// 普通索引：type 不填或填 IndexType.INDEX
@Index({
  name: 'idx_user_id', // 普通索引以 idx_ 开头
  keys: ['userId'],
})
// 联合索引：keys 填多个属性名
@Index({
  name: 'idx_user_status',
  keys: ['userId', 'status'],
})
export class OpenapiOrder {
  // ─── 主键（固定写法，不要改动）───────────────────────────────
  @Column(
    {
      type: ColumnType.BIGINT,
      unsigned: true,
    },
    {
      primaryKey: true,
      autoIncrement: true,
      comment: '主键',
    },
  )
  id: string; // 主键虽然是 BIGINT，TypeScript 类型用 string 防止 JS number 溢出

  // ─── 业务字段（VARCHAR）────────────────────────────────────
  @Column(
    {
      type: ColumnType.VARCHAR,
      length: 64, // VARCHAR 必须指定 length
    },
    {
      comment: '订单号',
      default: "''", // 字符串默认空字符串，注意外层单引号内层双引号的写法
    },
  )
  orderNo: string;

  // ─── 业务字段（INT）────────────────────────────────────────
  @Column(
    {
      type: ColumnType.INT,
      unsigned: true, // 非负数加 unsigned
    },
    {
      comment: '金额（分）',
      default: '0',
    },
  )
  amount: number;

  // ─── 业务字段（TINYINT，用于状态枚举）────────────────────────
  @Column(
    {
      type: ColumnType.TINYINT,
    },
    {
      comment: '状态: 0-待支付, 1-已支付, 2-已取消',
      default: '0',
    },
  )
  status: number;

  // ─── 业务字段（TEXT，存 JSON）──────────────────────────────
  @Column(
    {
      type: ColumnType.TEXT,
    },
    {
      comment: '扩展信息，JSON 格式',
      canNull: true, // 可为 NULL，TypeScript 属性加 ?
    },
  )
  extra?: string;

  // ─── 业务字段（可空 VARCHAR）───────────────────────────────
  @Column(
    {
      type: ColumnType.VARCHAR,
      length: 128,
    },
    {
      comment: '备注',
      canNull: true,
    },
  )
  remark?: string;

  // ─── 时间字段（固定写法，三个时间字段保持一致）────────────────
  @Column(
    {
      type: ColumnType.TIMESTAMP,
    },
    {
      default: 'CURRENT_TIMESTAMP',
      comment: '创建时间',
    },
  )
  gmtCreate: Date;

  @Column(
    {
      type: ColumnType.TIMESTAMP,
      autoUpdate: true, // 等价于 ON UPDATE CURRENT_TIMESTAMP
    },
    {
      default: 'CURRENT_TIMESTAMP',
      comment: '更新时间',
    },
  )
  gmtModified: Date;

  @Column(
    {
      type: ColumnType.TIMESTAMP,
    },
    {
      comment: '删除时间（软删除用）',
      canNull: true,
    },
  )
  gmtDeleted?: Date;
}
```

**@Column 类型速查表**：

| MySQL 类型         | ColumnType 写法        | 额外参数                   | TS 类型            |
| ------------------ | ---------------------- | -------------------------- | ------------------ |
| VARCHAR(n)         | `ColumnType.VARCHAR`   | `length: n`（必填）        | `string`           |
| TEXT               | `ColumnType.TEXT`      | 无                         | `string`           |
| BIGINT UNSIGNED    | `ColumnType.BIGINT`    | `unsigned: true`           | `string`（防溢出） |
| INT / INT UNSIGNED | `ColumnType.INT`       | `unsigned: true`（可选）   | `number`           |
| TINYINT            | `ColumnType.TINYINT`   | 无                         | `number`           |
| TIMESTAMP          | `ColumnType.TIMESTAMP` | `autoUpdate: true`（可选） | `Date`             |
| BOOLEAN            | `ColumnType.BOOL`      | 无                         | `boolean`          |

**@Column 第二参数常用字段**：

| 参数            | 类型      | 说明                                       |
| --------------- | --------- | ------------------------------------------ |
| `comment`       | `string`  | 列注释，必填                               |
| `default`       | `string`  | 默认值，字符串字面量填 `"''"` 数字填 `'0'` |
| `canNull`       | `boolean` | 是否允许 NULL，默认 `false`（NOT NULL）    |
| `primaryKey`    | `boolean` | 是否主键                                   |
| `autoIncrement` | `boolean` | 是否自增                                   |

- `dal/dao/OpenapiOrderDAO.ts`：**可修改**，添加自定义查询方法


输出类似：

```sql
CREATE TABLE IF NOT EXISTS openapi_order (
  id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
  order_no VARCHAR(64) NOT NULL DEFAULT '' COMMENT '订单号',
  amount INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '金额（分）',
  status TINYINT NOT NULL DEFAULT 0 COMMENT '状态: 0-待支付, 1-已支付, 2-已取消',
  extra TEXT NULL COMMENT '扩展信息，JSON 格式',
  remark VARCHAR(128) NULL COMMENT '备注',
  gmt_create TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  gmt_modified TIMESTAMP ON UPDATE CURRENT_TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '更新时间',
  gmt_deleted TIMESTAMP NULL COMMENT '删除时间（软删除用）',
  UNIQUE KEY uk_order_no (order_no),
  KEY idx_user_id (user_id),
  KEY idx_user_status (user_id,status)
) COMMENT='订单表';
```

## 参考资料
### 定义表结构

在 module 中，使用 `@Table` 装饰器定义数据库表，使用 `@Column` 装饰器定义列，使用 `@Index` 装饰器定义索引。

文件路径没有限制，通常为了方便管理和维护，可以考虑将表结构定义都放到 module 下的 `table` 目录中。

```typescript
import { Column, ColumnType, Index, IndexType, Table } from 'chair/tegg/dal';

// 定义一个表
@Table({
  comment: '通知记录，防止通知冲突',
  // dataSourceName: 'firstDS', // 如果有多个数据源时，需要使用 dataSourceName 指定 module.yml 配置中的数据源
})
// 定义一个 UNIQUE 类型索引
@Index({
  type: IndexType.UNIQUE,
  keys: [ 'notifyKey' ],
})
// 未指定 type 时，默认定义为 Normal 类型索引
@Index({
  keys: [ 'gmtCreate' ],
})
export class NotifyRecord {
  // 主键
  @Column({
    type: ColumnType.BIGINT,
  }, {
    primaryKey: true,
    comment: '主键',
    autoIncrement: true,
  })
  id: string;

  @Column({
    type: ColumnType.TIMESTAMP,
  }, {
    default: 'now()',
    comment: '创建时间',
  })
  gmtCreate: Date;

  @Column({
    type: ColumnType.TIMESTAMP,
    autoUpdate: true,
  }, {
    default: 'now()',
    comment: '修改时间',
  })
  gmtModified: Date;

  @Column({
    type: ColumnType.VARCHAR,
    length: 256,
  }, {
    comment: '通知 key',
  })
  notifyKey: string;
}

```
### 最佳实践

#### 主键

自增长主键请务必设置为 BIGINT，避免 js number 溢出。

```typescript
  @Column({
    type: ColumnType.BIGINT,
  }, {
    // 声明 primaryKey 为 true
    primaryKey: true,
    comment: '主键',
    // 声明为自增
    autoIncrement: true,
  })
  id: string;
```
#### gmtCreate/gmtModified

`gmtCreate` 以及 `gmtModified` 推荐按照下列方式定义。

```typescript
  @Column({
    type: ColumnType.TIMESTAMP,
  }, {
    default: 'now()',
    comment: '创建时间',
  })
  gmtCreate: Date;

  @Column({
    type: ColumnType.TIMESTAMP,
    autoUpdate: true,
  }, {
    default: 'now()',
    comment: '修改时间',
  })
  gmtModified: Date;
```
### 装饰器参数详情

#### @Table

`@Table(param?: TableParams)` 定义表

```typescript
export interface TableParams {
  // 数据库表名
  name?: string;
  // 数据源名称，如果 module 只有一个 dataSource 可不填，有多个时必须使用此字段进行指定
  dataSourceName?: string;
  // 表注释
  comment?: string;
  autoExtendSize?: number;
  autoIncrement?: number;
  avgRowLength?: number;
  characterSet?: string;
  collate?: string;
  compression?: CompressionType;
  encryption?: boolean;
  engine?: string;
  engineAttribute?: string;
  insertMethod?: InsertMethod;
  keyBlockSize?: number;
  maxRows?: number;
  minRows?: number;
  rowFormat?: RowFormat;
  secondaryEngineAttribute?: string;
}
```
#### @Index

`@Index(param?: IndexParams)` 定义索引

```typescript
export interface IndexParams {
  // 索引的列
  keys: string[];
  // 索引名称，如果未指定会用 列名拼接
  // 如 keys 为 [column1, column2 ]
  // 普通索引为 idx_column1_column2
  // 唯一索引为 uk_column1_column2
  name?: string;
  // type 不填，默认为 Normal 类型索引
  type?: IndexType,
  storeType?: IndexStoreType;
  comment?: string;
  engineAttribute?: string;
  secondaryEngineAttribute?: string;
  parser?: string;
}
```
#### @Column

`@Column(type: ColumnTypeParams, parmas?: ColumnParams)` 定义列

```typescript
export interface ColumnParams {
  // 列名，默认转换规则 userName 至 user_name
  name?: string;
  // 默认值
  default?: string;
  // 是否可空，默认为 false
  canNull?: boolean;
  comment?: string;
  visible?: boolean;
  autoIncrement?: boolean;
  uniqueKey?: boolean;
  primaryKey?: boolean;
  collate?: string;
  columnFormat?: ColumnFormat;
  engineAttribute?: string;
  secondaryEngineAttribute?: string;
}
```

```typescript
  })
```
