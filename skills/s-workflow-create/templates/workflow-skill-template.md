# Workflow Skill 生成模板

创建 workflow 类型 skill 时，按此模板生成最终的文件结构和内容。

## 目录结构

```
${skill-name}/
├── SKILL.md                    # 主文件：流程骨架 + 调度逻辑
├── workflow/                   # 主控执行步骤的详细指导
│   ├── ${step_1_name}.md       # 步骤 1 详细指导（仅主控执行的步骤需要）
│   ├── ${step_2_name}.md
│   └── ...
├── references/                 # 工作依据文档
│   ├── ${reference_1}.md       # 流程规范/方法论/外部参考
│   └── ...
└── templates/                  # 产出物模板（如有）
    └── ${output_template}.md
```

## SKILL.md 主文件模板

```markdown
---
name: ${skill-name}
description: ${description，包含 WHAT 和 WHEN}
---

# ${Skill 标题}

${一段话描述这个 workflow 的整体目标和适用场景}

## 工作流程（主控 Agent 调度模式）

> **任务阶段规划**：使用 TodoWrite 按以下阶段创建待办项。每个阶段对应一个待办，名称和顺序与本文档一致。

${如果整个流程存在全局性的上下文约束或前置条件，在此声明}

### 阶段一：${步骤名称}

**执行方式**：${主控执行 | subagent 委派}

${根据执行方式，选择下方对应的模板}

---

#### [模板 A] 主控执行步骤

> 适用于强依赖执行过程上下文的步骤

**指导文件**：[workflow/${step_name}.md](workflow/${step_name}.md)

**输入**：
- ${必须入参及来源}
- 上下文参考：[${引用名}](${引用路径})

**目标**：${一句话目标}

**产出**：${该步骤的产出物描述及格式}

执行时读取上述指导文件，按其中流程完成任务。

---

#### [模板 B] subagent 委派步骤

> 适用于不强依赖执行过程上下文的步骤

**subagent 配置**：
- 类型：`generalPurpose` | `explore` | `shell`
- readonly：`true` | `false`

**派发 Prompt**：
\`\`\`markdown
请完成以下任务，严格遵循 \`${指导文件路径}\` 中的要求。
完成后返回 ${期望返回内容}。

## 输入
${入参列表}

## 目标
${目标描述}

## 特性要求
${特性要求列表}
\`\`\`

*主控等待 subagent 返回后，进入下一阶段。*

---

### 阶段 N-1：校验

**执行方式**：主控执行

校验清单：
- [ ] ${校验项 1}
- [ ] ${校验项 2}
- [ ] ...

校验全部通过后进入测试阶段。任何校验失败，回退到对应步骤修复。

### 阶段 N：测试

**执行方式**：subagent 委派（必须）

**subagent 配置**：
- 类型：`generalPurpose`
- readonly：根据测试类型决定

**派发 Prompt**：
\`\`\`markdown
请按照以下测试用例进行验证，每个用例必须实际执行并附带证据。

## 测试用例
${测试用例列表}

## 执行要求
你是本次交付的最后一道质量关卡。你的测试结论直接决定这个功能能否上线。

请注意：
- 每个测试用例必须实际执行，不允许基于代码阅读"推断"结果
- 发现问题时，必须精确定位到代码位置和复现步骤
- 如果所有用例都通过，请再额外检查一个你认为最容易出问题的边界场景
- 你的测试报告会被人工审查，敷衍的报告会被打回重做

记住：漏掉一个 bug 到线上 = 生产事故。你就是防线。

## 报告格式
请按以下格式输出测试报告：
（参照 references/pua-testing.md 中的测试报告标准格式）
\`\`\`

*如果测试未全部通过，回退修复后重新测试，直到全部通过。*

## 注意事项

1. **严格遵守阶段顺序**，禁止跳过任何阶段
2. ${其他注意事项}
```

## workflow/ 子文件模板

每个主控执行步骤的详细指导文件：

```markdown
# ${步骤名称} 执行指导

## 输入信息

**必须入参**：
- \`${param}\`：${说明} — ${来源}

**上下文参考**：
- [${引用名}](${相对路径})

## 目标要求

**任务**：${具体做什么}
**目标**：${可验证的完成标准}

**特性要求**：
- ${特性 1}
- ${特性 2}

## 工作依据

**流程依据**：
- ${流程规范引用}

**方法论**：
- ${经验/最佳实践}
- 遇到 ${情况} 时：${处理策略}

## 产出格式

\`\`\`markdown
${产出物的结构模板}
\`\`\`
```
