# 架构分析详细工作流

此文件供 Agent 和 Subagent 参考，以确保分析深度一致。

## 步骤拆解

### 仓库获取与元信息 (主 Agent)
- 记录提交哈希、语言分布、代码规模。
- 确定 `focus_features`（如有）。

### 深度分析 (Subagent)
*接下来的分析流程，每个流程通过单独的 Subagent 执行，并将结果结构化反馈给主 Agent：*

#### 1. 入口与对外接口
- 定位 CLI、API 或 Web 服务启动器。
- 记录签名、职责及调用示例。

#### 2. 模块依赖与分层
- 识别：接口层、业务层、领域层、基础设施层。
- 绘制 Mermaid `graph TD`。

#### 3. 功能链路追踪
- 追踪"请求 -> 处理 -> 输出"的完整路径。
- 生成 `sequenceDiagram`。
- 标注关键代码位置。

#### 4. 核心模块专项分析
- **算法/数据结构**: 识别自定义算法及其复杂度。
- **状态机**: 绘制 `stateDiagram`。
- **并发/异步**: 分析锁、Channel、Promise 处理。
- **存储/持久化**: 记录 ORM/DAO 实现。

#### 5. AI Agent 专项 (如有)
- 提取 System Prompt、Task Prompt。
- 绘制 Agent 工作流图 (`flowchart TD`)。
- 记录工具调用 (Function Calling) 定义。

#### 6. 数据流与控制流
- 标注正常与异常控制路径。

## 最终报告模板 (`code_analysis.md`)
1. **概览**: 包含项目基本元信息。
2. **架构图**: Mermaid 模块图。
3. **主链路**: 时序图 + 关键代码讲解。
4. **专项解析**: 算法、并发、AI Agent 等。
5. **参考资料**: 汇总上述收集的仓库内外资料链接。
