# 字幕 → 中文文章：格式与忠实度规则

本文件与 `ai-workspace/.cursor/commands/srt-transform.md` **语义对齐**，作为 Skill 内嵌依据；若工作区未挂载该命令文件，以本文为准。

## 忠实度

- **完整保留**字幕中的信息：论点、条件、转折、数字、人名、否定与限制语均不得因「润色」而消失或扭曲。
- **允许**：合并短句、调整语序以符合中文阅读习惯；必要处加连接词使逻辑显式化。
- **禁止**：添加 SRT 中不存在的因果/评价/结论；编造未出现的专名或数据。

## 段落与时间锚

- 形成**可读中文正文**，不是字幕清单体（避免逐条短语罗列，除非内容本身就是列表教学）。
- **每个段落开头**注明该段内容对应的视频时间范围，便于回看：

```markdown
[09:28-09:52] 这部分内容……
```

- 时码格式与 SRT 一致取整到秒；跨段合并时取**首条字幕开始**至**末条字幕结束**。

## 附图占位（供阶段四替换为 CDN）

若某时刻**强烈依赖画面**才能理解，在对应位置插入标记：

```markdown
[🖼️ 附图描述 09:40]
```

- **描述**：说明需要什么画面（动作、示意、板书等），便于选帧或写 alt 文本。
- **时间**：与 `yt-shot` 截帧或已有截图时间点对齐（可近似到最近关键帧）。

## 注意事项类内容的结构化表达

口播里的**提醒、限制、禁忌、常见错误、纠错口令**（含 “don’t / avoid / common mistake / if you…”），译文在**不删条件、不弱化否定、不编造对策**的前提下，宜从「散文堆砌」改为**可扫读的结构**，便于对照执行与回看：

- **识别**：同一段或相邻时码内出现「不要做／很多人会／否则容易／应该／确保」等**规范句**或**对错对照**，即按本节处理。
- **呈现（择一或组合，以贴合原文密度为准）**
  - **小节加粗 + 列表**：例如小节名 **常见错误**、**对照检查**、**禁忌与边界**，下列 `-` 逐条对应字幕中的独立要点。
  - **现象 / 对策**：适用讲解者明确说「问题是什么、怎么改」时，写成 **现象：** … **对策：** …（可各占一行或多行列表），**对策句须能在 SRT 中找到依据**，勿凭空补「医学上更优」而字幕未提及的建议。
  - **不要 / 宜**：适用强对比口播时，用 **不要：** … **宜：** … 保留指令感，数字与身体部位名保持与字幕一致。
- **忠实度**：结构化是**版式层**操作；**不得**因合并条目而丢掉某一限制条件，也未出现过的警告**禁止**添加。
- **与列表口播的关系**：若原文本来就是「第一、第二」式枚举，可**保留列表体**，并用加粗小节标题收敛主题（见上），无需强行改段落散文。

## 产出形态

- 默认输出文件：`article.zh.md`（与 Skill 主文件路径约定一致）。
- 若用户要求写回「原文件」且上下文指 **SRT**，仅在用户明确要求时覆盖 SRT；**本 Skill 默认新建/更新 Markdown**，不覆盖 `source.*.srt`。
