# 阶段三：子工作流 — 字幕转中文可读文章

## 输入信息

**硬依赖**：

- `$WORK_ROOT/$VIDEO_ID/source.$LANG.srt` 全文（或等价路径）。

**依据**：[references/srt-transform-rules.md](../references/srt-transform-rules.md)

## 目标要求（四要素摘要）

| 维度 | 内容 |
|------|------|
| **Input** | SRT 时间码 + 原文行；可选：视频标题、章节结构提示。 |
| **Goal** | 一篇 **中文** Markdown：**不删论点、不扭曲因果、不编造事实**；读者可不打开视频理解主线；段落带时间锚点；**注意事项 / 常见错误 / 禁忌**等规范句按规则文件做**结构化**呈现（非散文堆砌）。 |
| **Reference** | 段落格式、`[🖼️]` 占位、注意事项结构化规则见 `references/srt-transform-rules.md`。 |
| **Execution** | 通读 SRT → 按语义合并短句为段 → 翻译为自然中文 → 为每段标注时间范围（取该段首尾 cue 时间）→ 对必须看图理解的点插入附图占位 → **将提醒类口播收敛为加粗小节 + 列表或 现象/对策 等对偶块**。 |

## 核心动作

1. **解析 SRT**：保留时间顺序；合并同一主题的连续字幕，避免一句话一段。
2. **翻译**：英文（或其它源语言）→ 中文；专名、人名、机构名首次可保留英文括注。
3. **段落头**：每段开头使用 `[MM:SS-MM:SS] ` 或 `[HH:MM:SS-HH:MM:SS] `（与规则文件一致）。
4. **附图占位**：仅在「无图极难理解」处插入：  
   `[🖼️ 简述需要的画面  MM:SS]`  
   时刻尽量与后文阶段四截帧或已有 `yt-shot` 对齐。
5. **注意事项结构化**：对字幕中的警告、常见错误、禁忌与纠错口令，按 `references/srt-transform-rules.md` **「注意事项类内容的结构化表达」** 处理；**不增删语义**，只提升可扫读性。
6. **落盘**：将终稿写入 `$WORK_ROOT/$VIDEO_ID/article.zh.md`（或与用户约定的绝对路径）。

## 完成判定

- 随机抽取 3 处段落，能在 SRT 中找到对应时间范围内的原文支撑。
- 无整段原文在 SRT 中毫无依据（幻觉检测）。
- 未删除反驳、限制条件、数字、否定句等「容易被滑过去」的信息。

## 委派 Subagent 时（可选）

派发时附带：

- SRT 文件路径（或全文分块）。
- 强调：**禁止**摘要成条目而丢失论证链；**必须**保留时间锚与 🖼️ 占位格式；**提醒 / 纠错类句子**须按规则文件做结构化（列表、现象/对策等），**不得**省略字幕中的限制语与否定句。
