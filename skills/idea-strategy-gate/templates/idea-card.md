# Idea Card 模板（物种适配版）

用于把不同形态的想法归一化，方便比较。
实际对话中不必要求用户完整填写，可由主控根据用户回答补齐，并标注不明项。

```markdown
## ${idea_name}

**一句话定义**：${这个 idea 实际要解决什么问题，而不是功能名称}

### 物种识别
- 交付物形态（Form）：${software/service/methodology/content/asset/state/organization}
- 使用者（User）：${self/others/mixed}
- 价值实现路径（Value Path）：${direct/infrastructure/compound/option}
- 成熟度起点（Maturity）：${concept/hypothesis/prototype/validated}
- 成功定义（Success Criterion）：${used/understood/replicated/monetized/compounded/experienced}
- **主物种判断**：${对照速查表选择}
- **是否多物种**：${是/否，若是说明核心物种 + 外壳物种}
- **不适用的常规问题**：${列出该物种下不适用的标准维度}

### 概念清晰度
- 核心概念列表：
  - ${concept_1}：L0/L1/L2/L3，证据：
  - ${concept_2}：L0/L1/L2/L3，证据：
- **路由判定**：
  - [ ] 直接进入战略压测
  - [ ] 调整问题集后进入战略压测（研究/方法论物种 + 低级别概念）
  - [ ] 触发前置概念重构（产品物种 + L0/L1 概念）
  - [ ] 给出概念未定义预警

### 通用字段
**服务对象 / 使用者**：${自己 / 某类创作者 / 某类客户 / 某类项目 / 不明}

**根问题**：${它命中的底层问题}

**为什么现在想到它**：${触发来源，区分长期痛点和新鲜感}

**已有资产**：${笔记、案例、脚本、渠道、经验、数据、社群、材料}

**长期目标贴合**：${它如何服务用户 1-3 年主线}

**复利资产**：${按物种特定形态：方法论案例库 / 研究阶段成果 / 内容资产 / 用户基础}

**下游复用**：${它是否会成为其他项目/工具/内容/决策的输入}

**机会成本**：${推进它会挤掉什么}

**最大反方风险**：${物种特定的根本前提风险}

**当前判断**：${主推 / 观察 / 暂缓 / 否决 / 前置概念重构 / 前置研究规划 / 不明}

**推荐下一步路径**（按物种映射）：${方法论 → 手动跑通；研究 → 研究路径设计；产品 → 用户访谈/原型；等}

**需要澄清的问题**：
- ${question_1}
- ${question_2}
```

## 用户快速输入版

当候选 idea 很多时，只让用户先提供这个轻量版：

```markdown
1. idea 名称：
   一句话描述：
   它最终的形态是什么（软件 / 方法论 / 内容 / 研究 / 习惯 / 资产 / 其他）：
   它的最终使用者是谁（自己 / 他人 / 混合）：
   为什么会想做它：
   已经有什么积累：
```

再补充四个全局背景：

```markdown
- 未来 1-3 年最想积累的主线：
- 现在最强的 3 个技能/资产：
- 每周真实可投入时间：
- 最看重的价值排序：赚钱 / 能力复利 / 自用价值 / 影响力 / 长期自由度 / 组织能力
```

## 单 idea 深度输入版

```markdown
本轮我会把它当成「需要校验的方向」来评估，不是默认应该做。

### 形态信息
- 它最终的形态是什么？（软件 / 方法论 / 内容 / 研究 / 习惯 / 资产 / 状态 / 组织能力）
- 它的最终使用者是谁？（自己 / 他人 / 混合）
- 你认为「这件事做成」的判定标准是什么？

### 痛点信息
- 它解决的底层问题是什么？
- 过去 3-6 个月你自己遇到过几次类似问题？
- 现有工具/方法具体在哪一步失效？

### 概念信息
- 它包含的核心概念有哪些？
- 你能用一句话给出这些概念的输入和输出吗？
- 这些概念与已有的近邻概念边界是什么？

### 成本信息
- 如果推进它，会挤掉哪个你本来也想做的方向？
```
