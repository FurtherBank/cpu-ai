# 阶段三：战略五问门（物种适配版）

## 输入信息

**必须入参**：
- `归一化候选`：阶段二输出的候选定义和分组。
- `物种识别记录`：阶段一对每个候选的物种判断和概念清晰度评估。
- `目标函数初稿`：用户最看重的长期结果。

**上下文参考**：
- [战略五问门参考（元定义版）](../references/heuristic-gates.md)
- [Idea 物种识别参考](../references/idea-species.md)
- [概念清晰度评估参考](../references/concept-clarity.md)
- [语气校准参考](../references/tone-calibration.md)

## 目标要求

**任务**：用五个方向级判断点压缩比较，**根据物种动态调整每问的具体标准**，不进入执行层。

**完成标准**：
- 每个强候选和关键观察候选都按其物种执行五问判断。
- 每个候选至少有一条强信号或明确标注「强信号不足」。
- 每个候选至少有一条反方风险（基于物种特定的反方点）。
- 关键不明项被写成可追问问题，而不是被主控脑补。
- 所有问题与措辞符合 [tone-calibration.md](../references/tone-calibration.md) 的中立语气标准。

## 工作依据

### 第一步：确认物种与问题集

进入压测前，主控必须先回顾阶段一的物种识别记录，并显式声明：

```markdown
本次压测使用的问题集：
- ${idea_name}：${物种} → 套用 [heuristic-gates.md] 中对应物种的具体追问
- 该物种**不适用**的常规问题：${列出，避免误用}
```

### 第二步：执行五问

按 [heuristic-gates.md](../references/heuristic-gates.md) 的元定义执行五问，每问的具体追问按物种选择。

**避免常见误用**：

| 误用 | 后果 | 修正 |
|---|---|---|
| 用「产品独特进入权」追问研究类 Idea | 把「需要研究」误判为「核心资产缺失」 | 改用研究物种追问：研究路径是否可设计、是否有现成基础 |
| 用「产品 PMF」追问方法论类 Idea | 把「自用方法论」误判为「市场不成立」 | 改用方法论物种追问：手动版本是否成立、可迁移性、复利度 |
| 用「商业护城河」追问内容类 Idea | 把「无明显壁垒」误判为「不可持续」 | 改用内容物种追问：表达独特性、持续创作动力 |
| 用「基础设施上游性」自动推高研究类 Idea | 用终局价值掩盖当前阶段缺陷 | 区分「长期价值高」与「当前阶段是否值得投入」 |

### 第三步：物种特定追问模板

#### 产品/工具物种

```markdown
对每个候选回答：
- 核心人群是谁？
- 他们现在的旧流程是什么？
- 你有什么独特进入权？
- 如果失败，最可能是哪句前提错了？
```

#### 方法论物种

```markdown
对每个候选回答：
- 这个方法论解决的认知问题是什么？请举 3 个不同领域的真实例子。
- 不依赖任何工具，纯手动版本是否成立？
- 你愿意用第一个真实场景手动跑一遍吗？
- 这个方法论与已有的近邻方法论（如形式逻辑、本体论、思维导图）边界是什么？
- 如果手动跑通后发现不通用，会调整方法论本身还是放弃？
```

#### 研究/探索物种

```markdown
对每个候选回答：
- 你打算基于什么现成基础（学科 / 已有方法 / 类似研究）来推导这个方向？
- 研究路径上的第一个可验证里程碑是什么？
- 你怎么判断这个研究「成功」或「失败」？（证伪标准）
- 即使研究失败，会留下什么可迁移的洞察？
- 研究投入多少时间后，你会做第一次成败评估？
```

#### 内容/作品物种

```markdown
对每个候选回答：
- 受众是谁？他们当前从哪里获取同类内容？
- 你的视角与现有内容的差异在哪？
- 你能持续产出的素材源是什么？
- 在没有任何外部反馈的前提下，你愿意持续多久？
```

#### 商业/项目物种

```markdown
对每个候选回答：
- 商业模式是什么？单位经济模型怎么算？
- 护城河是什么？多久能被复制？
- 获客渠道是什么？获客成本估算？
- 如果失败，最可能是哪句前提错了？
```

#### 生活方式/习惯/技能物种

```markdown
对每个候选回答：
- 这个状态变化的真实收益是什么？
- 时间/精力/金钱成本估算？
- 3 个月后还会坚持的概率？依赖什么外部条件？
- 失败后会留下什么副作用？
```

#### 基础设施/资产物种

```markdown
对每个候选回答：
- 它会被哪些任务反复调用？请列出未来 6 个月内的真实场景。
- 它的输出会不会成为其他工具/skill/流程的输入？
- 不做它会卡住哪些后续能力？
- 它依赖哪些已有资产？
- 它解决一次性痛点，还是改变之后做事的方式？
```

#### 组织能力物种

```markdown
对每个候选回答：
- 这种能力缺失在过去多次协作中如何暴露？
- 谁是核心承载者？脱离他/她是否仍能运行？
- 培训路径是否可设计？
- 激励机制是否成立？
```

### 第四步：候选很多时的快速追问

候选很多时，不逐项长问，先对强候选做选择题（保持物种意识，不混评不同物种的绝对分数）：

```markdown
只对候选区追问 5 个问题，你可以用 A/B/C 快速答：

1. 哪个 idea 已经有最多真实材料积累？
2. 哪个 idea 即使不公开发布，你自己也会每周使用？
3. 哪个 idea 会让你未来做其他项目也变快？
4. 哪个 idea 最不依赖外部平台或热点变化？
5. 哪个 idea 如果做成，会自然生成内容、案例、方法论或工具资产？
```

注意：这组快速追问对所有物种适用，但解读时需结合物种背景：
- 自用方法论物种在「自己每周使用」上得分高 = 强信号
- 商业项目物种在「自己每周使用」上得分高不一定是强信号

## 产出格式

```markdown
## 战略五问记录

### ${idea_name}

**物种与问题集**：
- 主物种：${物种}
- 使用的问题集：${heuristic-gates.md 中对应物种的具体追问}
- 不适用的常规问题：${列出}

**五问判断**：
- 根问题强度：强 / 中 / 弱 / 不明，证据：
- 独特进入权：强 / 中 / 弱 / 不明，证据：
- 复利资产：强 / 中 / 弱 / 不明，证据：
- 上游杠杆：强 / 中 / 弱 / 不明，证据：
- 反方否决：${物种特定的最大风险}

**待澄清假设**：
- ${必须追问用户的关键事实，不脑补}

**物种结论倾向**（不是最终结论，仅作压测后初判）：
- [ ] 通过 / 主推
- [ ] 观察
- [ ] 暂存
- [ ] 暂缓 / 否决
- [ ] 前置概念重构（仅产品物种 + L0/L1 概念）
- [ ] 前置研究规划（仅研究物种 + 路径不清）
```

## 自查清单

阶段三结束前，主控必须自查：

- [ ] 是否对每个候选显式声明了使用的物种问题集？
- [ ] 是否避免了用一种物种的标准评估另一种物种？
- [ ] 是否所有判断措辞都符合 tone-calibration.md（不含「虚荣心 / 心动 / 宏大叙事 / 自我感动」等动机攻击词）？
- [ ] 是否所有「不明」项都写成了可追问问题，而非脑补结论？
