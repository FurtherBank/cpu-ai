# 阶段四：反方压测与多轮澄清（物种适配 + 三类压测）

## 输入信息

**必须入参**：
- `Top 候选`：阶段三中强信号最多或用户最偏爱的候选。
- `物种识别记录`：阶段一对该候选的物种判断与概念清晰度评估。
- `战略五问记录`：每个候选的强信号、弱信号、反方风险和待澄清假设。

**上下文参考**：
- [Idea 物种识别参考](../references/idea-species.md)
- [概念清晰度评估参考](../references/concept-clarity.md)
- [语气校准参考](../references/tone-calibration.md)

## 目标要求

**任务**：专门攻击最可能误导判断的地方，确认 Top 候选不是由短期信号、模糊概念或成本误算推高。

**完成标准**：
- Top 候选已被对应物种的「决定性问题」压测。
- 至少完成「概念压测 / 成本压测 / 价值压测」三类中适用的子类。
- 已明确推进该方向会挤掉什么机会成本。
- 已判断 Top 候选之间是并列、主从、合并还是冲突。
- 所有问题与措辞符合 [tone-calibration.md](../references/tone-calibration.md) 的中立语气标准。
- 若用户回答「这是执行问题」「需要研究」等模糊回避，**不直接判定为逃避**，按本文档「常见回答处理」流程二次追问。

## 工作依据

### 三类压测

**关键变更**：本版本不再合并所有压测，而是显式区分三类，因为它们针对的失误模式不同。

#### 类别 A：概念压测

**目的**：确认 Idea 中的核心概念是否真的能成立，而不是黑箱愿景。

**适用条件**：
- 核心概念在阶段一被评为 L0/L1
- 或物种是研究/方法论，需要确认概念可形式化

**通用问题**：
```markdown
- 这个概念能用一个具体场景说明它的输入和输出吗？
- 这个概念与 ${近邻概念} 的边界在哪？为什么不能直接用 ${近邻概念} 替代？
- 如果你只能拿一张 A4 纸手动应用这个概念，能跑通一个真实例子吗？
- 如果应用一次后发现失效，你会调整这个概念本身，还是放弃方向？
```

**研究/方法论物种的特别问题**：
```markdown
- 你的概念会基于哪个现成基础（学科/方法/前人工作）展开？
- 你怎么判断这个概念被「研究清楚了」？（证伪标准）
- 第一个可验证里程碑是什么？多久能完成？
```

#### 类别 B：成本压测

**目的**：确认推进该方向的真实成本，避免「时间是无限的」幻觉。

**通用问题**：
```markdown
- 如果推进这个方向，未来 1-3 个月它会挤掉哪个你本来也想做的方向？
  请不要回答「不会挤掉」。只要做深，就一定会挤掉东西。

- 这个方向需要多少前置投入才能开始产生价值？这个投入期你能承受吗？

- 推进路径上的最大成本类别是什么：时间 / 学习 / 试错 / 关系 / 注意力？
```

**ROI 反方问题**（针对方法论/基础设施物种）：
```markdown
- 用这套方法/工具去解决一次新问题的成本，是否大于直接用传统方式的成本？
- 在什么前提下 ROI 会倒挂？这些前提是否常见？
```

#### 类别 C：价值压测

**目的**：确认 Idea 的真实价值不依赖未被验证的外部条件。

**形态退化问题**（核心物种价值确认）：

适配不同物种的「形态退化」版本：

| 物种 | 形态退化问题 |
|---|---|
| 产品/工具 | 如果它永远做不成完整软件，只能是个内部脚本，你还会做吗？ |
| 方法论 | 如果它永远做不成软件，只能是张 A4 纸的判断清单，每次手动套用，你还会做吗？ |
| 研究 | 如果研究最终结论无法发表/分享，只是你自己的笔记，你还会做吗？ |
| 内容 | 如果你的内容永远没有任何外部反馈，你还会持续产出吗？ |
| 商业 | 如果短期盈利做不到，但能积累某项能力/资源，你还会做吗？ |
| 生活方式 | 如果状态改善的客观指标无法量化，只能凭主观感受，你还会做吗？ |
| 基础设施 | 如果它的下游调用次数低于你预期的 30%，你还会维护它吗？ |
| 组织能力 | 如果它无法被复制传承，只能在你手上运行，你还会建吗？ |

**判断**：
- 答「会」：底层方向价值成立，外壳形态可暂缓决定。
- 答「不会」：当前 Idea 的价值更多依赖外壳/工具/外部反馈，需要重新审视核心物种。
- 答「不确定」：暂存，先澄清成功定义。

**热点与外部依赖压测**：
```markdown
- 如果市场热度下降、平台流量消失、工具版本变化，这个方向还会因为你的长期主线而值得做吗？
- 这个方向有多少价值依赖某个特定外部条件（平台 / 渠道 / 工具 / 政策 / 关系）？
```

**基础设施 vs 终局价值**（针对基础设施物种）：
```markdown
- 它是输入层、方法论层、垂直工具链，还是上层集成器？
- 如果它消费多个前置资产，不要因为终局价值高就自动排第一。
- 它的下游真实调用次数预估？这些下游本身值不值得做？
```

### 关系判断

```markdown
这两个强候选是两个项目，还是一个项目的两个层级？
- 哪个解决「做什么」的上游问题？
- 哪个解决「怎么沉淀/承载」的载体问题？
- 如果只能选一个做 3 个月，你每天更可能打开哪个？
```

## 常见回答处理（关键章节）

用户在压测中常出现以下回答，**不要直接判定为逃避或缺陷**，按下方流程二次追问。

### 用户回答「需要研究」

可能的真实情况：
- A. 这是研究类物种的正常起点
- B. 这是产品类物种的核心资产缺失
- C. 这是用户回避追问的模糊回答

二次追问流程：
```markdown
1. 你能描述一下这个「研究」的可能路径吗？
   - 基于什么基础？预期产出什么形态？
2. 你判断它「研究成功」的标志是什么？
3. 你愿意先用多少时间做第一个最小试验？
```

判定：
- 能给出研究路径 + 证伪标准 → 研究物种正常起点（不构成否决）
- 能给出研究路径但缺证伪标准 → 标记为「前置研究规划」结论
- 无法给出任何路径 → 触发概念压测（类别 A）

### 用户回答「这是执行问题」

可能的真实情况：
- A. 该问题确实属于执行层（接受）
- B. 该问题底层有未澄清的战略假设（追问）
- C. 用户用执行层包装战略层（追问）

二次追问流程：
```markdown
1. 这个「执行问题」的解决依赖哪个前提？
2. 这个前提是否已经验证？
3. 如果该前提不成立，方向是否仍然成立？
```

判定：
- 前提已验证且独立于方向价值 → 接受为执行问题（不影响战略判断）
- 前提未验证但方向不依赖它 → 标记为风险但不阻断
- 前提未验证且方向依赖它 → 升级为战略问题，必须压测

### 用户回答「需要能够 X」（用愿景回应反方）

可能的真实情况：
- A. 用户在描述目标状态，未回应反方
- B. 用户已有该能力的初步证据，只是表达简化
- C. 用户用愿景回避现实可行性

二次追问流程：
```markdown
1. 你预期通过什么具体路径实现「X」？
2. 当前是否有任何已知证据/案例/方法表明「X」是可达成的？
3. 如果最终发现「X」不可达成，方向价值是否仍然成立？
```

### 用户回答「都重要」「都想做」

按 intake-context.md 的处理：改问「最不能接受牺牲哪一个」。

### 用户对反方表现强防御

可能的真实情况：
- A. 用户有未披露的关键资产/案例支撑
- B. 用户对该 Idea 有强情感投资
- C. 用户已经做出决定，只想要验证

二次追问流程：
```markdown
1. 是否存在未披露的资产/经验/案例？
2. 即使我们暂时不评估它，你是否会在接下来 1 个月内推进它？
3. 如果答案是「会」，那本次校验的真实目的是什么——决策、还是寻求支持？
```

判定：
- 用户已决定推进 → 战略门转换为「补强建议」模式（指出风险点 + 建议监测信号）
- 用户仍在决策 → 继续中立压测

## 异常处理（保留原有）

- 用户只表达喜欢：要求过去 3-6 个月真实样本。
- 用户证明「很多人需要」：追问具体服务对象和旧流程。
- 用户每个都想做：强制输出主轴、旁支、停止分散三层。
- 用户执着被否决项：问是否存在未披露的隐藏资产，如渠道、素材、案例、用户、脚本。
