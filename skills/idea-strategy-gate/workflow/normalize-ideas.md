# 阶段二：Idea 归一化与候选分组（物种感知版）

## 输入信息

**必须入参**：
- `候选 idea 原始信息`：阶段一得到的名称、描述、动机和已有积累。
- `物种识别记录`：阶段一对每个候选的物种判断与概念清晰度评估。
- `目标函数初稿`：长期主线、价值排序、资源约束和已有资产。

## 目标要求

**任务**：把不同粒度、不同形态的 idea 转成可比较的候选卡片，并先做粗分组。归一化时**保留并显式标注物种**，避免后续阶段误用统一标准。

**完成标准**：
- 每个候选都有一句话定义，不能只保留模糊名称。
- 每个候选都标注了物种（沿用阶段一的判断）。
- 已识别明显重复、包含、主从或互相冲突的 idea。
- 已输出 `强候选 / 观察候选 / 暂缓或偏题候选` 的初步分组。
- 未用「容易实现」或「短期热门」作为进入强候选的唯一理由。
- 跨物种比较时不直接套统一分数（参见 heuristic-gates.md「物种约束」）。

## 工作依据

### 归一化原则

- 把「产品外壳」翻译成「它解决的根问题」。
- 把「我想做」翻译成「它服务哪个长期目标」。
- 把「工具形态」翻译成「它是否沉淀资产或成为下游输入」。
- **保留物种维度**：归一化不是抹平形态差异，而是用统一字段呈现差异。

### 多物种 Idea 的归一化处理

当 Idea 是多物种（核心 + 外壳）时，归一化必须显式拆分：

```markdown
示例：「想做一个降低语言熵的工具」

归一化拆分：
- 核心方向（方法论物种）：建立一套降低语言熵的认知方法论
- 外壳方向（产品物种）：把方法论封装为软件工具
- 关系：外壳依赖核心成立；核心不依赖外壳
- 评估顺序：先评核心，外壳暂缓
```

### 常见关系判断

- 两个 idea 都强，但一个是主方向、一个是载体。例如「方向判断工具」可以作为「Agent Skill 库」的第一个 Skill。
- 一个 idea 看似独立，其实只是另一个方向的模块。例如「个人效率系统」可能只是「执行前筛选工具」的后续模块。
- 一个 idea 战略终局很高，但当前依赖前置资产。例如「项目 AI 赋能工具」可能依赖 SOP 与规则资产。
- **跨物种关系**：一个研究类 idea 与一个产品类 idea 可能是「研究产出 → 产品化」的上下游关系，归类时应保留物种差异，不强行合并。

### 粗分组信号（物种适配）

- **强候选**：贴合长期主线、有已有资产、有复利、有上游杠杆，且物种识别明确。
- **观察候选**：有价值但缺关键事实，或需要某个条件先成立，或物种识别存在不确定性。
- **暂缓/偏题候选**：主要来自热点、新鲜感、容易实现、短期变现想象，或与主线弱相关，或核心概念在 L0 且用户无意愿研究。

**特殊处理**：
- 物种是「研究」且核心概念在 L0 → 仍可进入强候选，但标注「需配合研究路径设计」
- 物种是「产品」且核心概念在 L0/L1 → 必须降级为观察候选并标注「需前置概念重构」
