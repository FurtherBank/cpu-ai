# 阶段五：结论合成（物种化版本）

## 输入信息

**必须入参**：
- `战略五问记录`：阶段三产物。
- `反方压测记录`：阶段四产物。
- `物种识别记录`：阶段一产物。
- `用户目标函数`：长期主线、价值排序、资源约束。

**上下文参考**：
- [Idea 物种识别参考](../references/idea-species.md)
- [概念清晰度评估参考](../references/concept-clarity.md)
- [语气校准参考](../references/tone-calibration.md)

## 目标要求

**任务**：把判断记录合成为用户可行动前使用的方向结论，但不写执行方案。

**完成标准**：
- 结论先行，不能只罗列分析。
- 结论必须**显式说明物种**与「物种 → 推荐推进路径」的映射。
- 每个主推/通过项都有战略理由和待确认假设。
- 每个暂缓/否决项都有具体原因，不使用「感觉不适合」「直觉判断」等表述。
- 观察项有升级条件或重评条件。
- 如果「战略终局价值」和「当前优先级」冲突，输出双层排序。
- 所有结论措辞符合 [tone-calibration.md](../references/tone-calibration.md) 的中立语气标准。

## 工作依据

### 物种 → 推进路径映射（核心新增）

不同物种的「通过」意味着不同的下一步路径。结论中**必须**显式说明：

| 物种 | 「通过」推荐的下一步路径 |
|---|---|
| 产品/工具 | 进入产品设计阶段（用户访谈 / 原型 / MVP） |
| 方法论 | 选 1 个真实场景做手动跑通验证（不写代码） |
| 研究 | 设计研究路径（基础梳理 / 假设 / 证伪标准 / 第一里程碑） |
| 内容 | 设计内容形态与产出节奏（受众 / 频率 / 素材源） |
| 商业 | 进入商业模式设计与单位经济模型测算 |
| 生活方式 | 设计 4 周试运行计划（带客观判定标准） |
| 基础设施 | 列出未来 6 个月真实下游调用场景，量化期望调用次数 |
| 组织能力 | 设计能力培训路径与激励机制 |

**关键约束**：
- 给方法论物种结论时，不要写「下一步是 MVP」（这是产品物种的下一步）
- 给研究物种结论时，不要写「商业模式」（这是商业物种的下一步）
- 推进路径必须匹配物种，否则结论本身就是滑坡

```markdown
## 结论：${通过 / 暂存 / 否决 / 前置概念重构 / 前置研究规划}（物种：${物种}）

### 物种与结论合法性
- 主物种：${物种}
- 是否多物种：${是/否，若是说明核心 + 外壳}
- 该物种下「通过」意味着：${按物种映射}
- 该物种下「不适用」的常规推进路径：${列出避免误导}

### 支持理由
- ${至少 3 条，按物种特定的强信号写}

### 反方风险
- ${至少 3 条，按物种特定的根本前提风险写}
- 每条附「在什么条件下风险会兑现」

### 决定性问题（关键判定枢纽）
- 问题：${本轮最影响结论的问题}

- 判定影响：${用户的回答如何决定结论方向}

### 推荐下一步路径
- ${按物种映射，例如：方法论 → 选 1 个真实场景手动跑通；研究 → 设计研究路径}
- **明确不推荐**：${列出该物种下错误的下一步，避免用户误入}

### 前置澄清
- ${进入下一阶段前必须澄清的问题}
```

### 「前置概念重构」结论结构

适用：产品/工具物种 + 核心概念在 L0/L1 时

```markdown
## 结论：前置概念重构（物种：${物种}）

### 为什么是前置概念重构
- 该 Idea 的可行性核心依赖概念 ${X}
- 概念 ${X} 当前清晰度：L0/L1
- 在概念形式化前，无法可靠评估产品方向

### 推荐路径
- 调用 t-concept-refactor 或类似 skill，对概念 ${X} 进行形式化
- 形式化目标：达到 L2（明确输入/输出/边界条件）
- 形式化完成后回到本工作流继续战略压测

### 不应做的判断
- 不应基于当前信息判定该 Idea「通过」或「否决」
- 不应基于当前信息进入产品设计或 MVP
```

### 「前置研究规划」结论结构

适用：研究物种 + 研究路径不清时

```markdown
## 结论：前置研究规划（物种：研究）

### 为什么是前置研究规划
- 这是一个研究类 Idea，「需要研究」是合理起点
- 但当前研究路径尚未设计，无法判断研究本身值不值得投入

### 推荐路径
- 设计研究路径：基础梳理 / 假设列表 / 证伪标准 / 第一可验证里程碑
- 路径设计完成后回到本工作流，对路径本身做战略压测

### 不应做的判断
- 不应把「需要研究」误判为「核心资产缺失」从而否决
- 不应基于当前信息进入产品/工具设计
```

## 禁止项

- 不写「第一步可以做一个 MVP」（除非物种是产品/工具且已通过概念清晰度）
- 不写功能列表
- 不写技术选型
- 不用「建议都试试」逃避排序
- 不用分数平均值覆盖致命否决项
- **不用一种物种的下一步路径推荐另一种物种**（例如不要给方法论物种推荐 MVP）
- **不使用动机攻击词、命令式表达、隐喻夸大**（参见 tone-calibration.md）

## 自查清单

阶段五结束前，主控必须自查：

- [ ] 结论是否显式说明了物种？
- [ ] 推荐下一步路径是否与物种匹配？（不要给方法论推荐 MVP）
- [ ] 是否避免了用一种物种的标准下结论？
- [ ] 「前置概念重构」/「前置研究规划」等合法结论类型是否被考虑？
- [ ] 措辞是否符合 tone-calibration.md（不含动机攻击词、命令式表达、隐喻夸大）？
- [ ] 每个反方风险是否附有「在什么条件下兑现」的可观察标准？
