# Idea Strategy Gate

用于把一个临时灵感的「好想法」，演进收敛成少数真正值得投入的方向。
这个 workflow 只做执行前战略校验：判断方向是否值得占用机会成本，不设计功能、不制定 MVP、不进入落地计划。

## 设计哲学：三层校准（Three-Layer Calibration）

经验表明，执行前战略校验最容易出错的不是「五问本身」，而是「在不该用某套标准时用了它」。
本 workflow 用三层校准来防止这种系统性误判：

- **第一层：物种识别（Species）**——Idea 不止「软件项目」一种形态。先用五维度推导识别物种（产品 / 方法论 / 研究 / 内容 / 商业 / 生活方式 / 基础设施 / 组织能力），再选择匹配的评估问题集。
- **第二层：概念清晰度（Concept Clarity）**——核心概念若处于黑箱状态（L0/L1），必须按物种路由：产品类触发前置概念重构，研究类调整问题集，方法论类用「手动跑通」作为升级路径。
- **第三层：语气校准（Tone）**——从「道德评判者」切换为「中立镜像」。攻击假设、不攻击动机；状态描述、不品质评价；严厉来自逻辑严密、不来自用词刺激。

## 工作流程（主控 Agent 调度模式）

> **任务阶段规划**：使用 TodoWrite 按以下阶段创建待办项。每个阶段对应一个待办，名称和顺序与本文档一致。

### 全局约束

- 全程使用中文与用户沟通。
- 制作执行前的研判并给出结论：禁止把判断滑向真正执行范围，如 MVP、第一版功能、技术实现、冷启动、增长打法、发布时间表。
- 允许多轮澄清；每轮问题必须短而具体，优先选择题、表格或一行描述。
- 方向完美主义优先：宁可暂存/否决，也不要用「容易做」「很有趣」「短期能试」掩盖战略缺陷。
- 若输入信息不足，必须输出缺口并继续追问，不得替用户脑补关键事实。
- **物种适配优先**：在选择评估问题集前，必须先识别 Idea 物种。不允许用一种物种的标准评估另一种物种。
- **概念清晰度门槛**：在战略压测前，必须评估核心概念清晰度。L0/L1 概念在产品物种下必须前置概念重构。
- **中立语气强制**：禁用动机攻击词（虚荣心 / 心动陷阱 / 宏大叙事陷阱 / 自我感动 / 逃避执行等）、命令式表达（你应该 / 你必须 / 你不能等）、隐喻夸大（深坑 / 死路 / 空中楼阁等）。详见 [tone-calibration.md](references/tone-calibration.md)。

### 阶段一：边界锁定与最小输入（含物种识别 + 概念清晰度）

**执行方式**：主控执行

**输入**：
- 用户原始 idea 描述、idea 列表或单个心动 idea。
- 用户明确约束：不进入执行层验证，只做方向级判断。
- 上下文参考：
  - [战略五问门参考（元定义版）](references/heuristic-gates.md)
  - [Idea 物种识别参考](references/idea-species.md)
  - [概念清晰度评估参考](references/concept-clarity.md)
  - [语气校准参考](references/tone-calibration.md)
  - [Idea Card 模板](templates/idea-card.md)

**目标**：把对话从「发散聊想法」变成「可判断的方向校验任务」，并完成三件最关键的前置识别——物种、概念清晰度、目标函数。

**产出**：
- 一组归一化前的候选 idea 信息
- 用户的目标函数初稿
- 每个候选的物种判断（五维度推导 + 主物种）
- 每个候选的核心概念清晰度评估
- 路由判定：是否需要前置概念重构

实际执行到该步骤时，**必须**阅读 [workflow/intake-context.md](workflow/intake-context.md) 文件内容作为指导。

### 阶段二：Idea 归一化与候选分组

**执行方式**：主控执行

**输入**：
- 阶段一收集到的 idea 原始信息。
- 阶段一的物种识别记录。
- 用户长期目标、价值排序、已有资产、资源约束。

**目标**：把不同形态的想法转成可比较的 Idea Card，并识别重复、重叠、主从关系和明显偏题项。

**产出**：候选分组，包括 `强候选`、`观察候选`、`暂缓/偏题候选`；每个候选至少有一句归一化定义，且物种已标注。

实际执行到该步骤时，**必须**阅读 [workflow/normalize-ideas.md](workflow/normalize-ideas.md) 文件内容作为指导。

### 阶段三：战略五问门（物种适配版）

**执行方式**：主控执行

**输入**：
- 阶段二的候选分组。
- 阶段一的物种识别记录与概念清晰度评估。
- 用户目标函数初稿。

**目标**：用五个方向级判断点压缩比较，**根据物种动态调整每问的具体标准**，不进入执行层。

**产出**：每个候选通过五问后的判断记录，包含强信号、弱信号、否决信号和待澄清假设；每条判断必须显式说明使用的物种问题集。

实际执行到该步骤时，**必须**阅读 [workflow/strategic-gates.md](workflow/strategic-gates.md) 文件内容作为指导。

### 阶段四：反方压测与多轮澄清（三类压测）

**执行方式**：主控执行

**输入**：
- 阶段三中排名靠前或用户强烈偏爱的候选。
- 该候选的物种识别记录。
- 该候选的强信号、弱信号、否决信号。

**目标**：通过「概念压测 / 成本压测 / 价值压测」三类显式区分的压测，确认 Top 候选不是由短期信号、模糊概念或成本误算推高。

**产出**：Top 候选的反方风险、决定性问题、机会成本判断、关系判断（并列/主从/合并/冲突）；对所有「需要研究」「这是执行问题」「需要能够 X」类回答的二次追问记录。

实际执行到该步骤时，**必须**阅读 [workflow/pressure-test.md](workflow/pressure-test.md) 文件内容作为指导。

### 阶段五：结论合成（物种化）

**执行方式**：主控执行

**输入**：
- 阶段三与阶段四的判断记录。
- 阶段一的物种识别记录。
- 用户在澄清中的回答。

**目标**：给出明确但不过度承诺的方向结论，让用户知道现在应该主推、观察、暂缓或否决什么；结论必须显式说明物种与「物种 → 推荐推进路径」的映射。

**产出**：一份结论报告。报告必须包含：
- 多 idea 场景：`主推方向`、`观察区`、`暂缓/否决区`、每项理由、升级/重评条件、**每项的物种与推荐下一步路径**。
- 单 idea 场景：`通过 / 暂存 / 否决 / 前置概念重构 / 前置研究规划`、支持理由、反方风险、决定性问题、前置澄清、**推荐下一步路径**。
- 若存在顶层战略价值与当前优先级冲突，必须输出「双层排序」。

实际执行到该步骤时，**必须**阅读 [workflow/synthesize-conclusion.md](workflow/synthesize-conclusion.md) 文件内容作为指导。

### 阶段六：结论校验与独立复核（扩展版）

**执行方式**：主控校验 + 可选 subagent 委派复核

**输入**：
- 阶段五结论报告。
- 阶段一的物种识别记录与概念清晰度评估。
- 用户原始需求与全局约束。

**目标**：通过五组校验（结论结构 / 物种识别与适配 / 概念清晰度处理 / 语气中立 / 常见回答处理）确认结论可靠。

**产出**：校验清单结果；高风险或结论争议较大时，附一份独立复核报告。

主控必须先执行 [workflow/validate-result.md](workflow/validate-result.md) 的校验清单。若以下任一条件成立，必须委派独立复核：
- 结论建议投入高机会成本方向。
- 用户对某个被否决 idea 有明显防御或强偏好。
- Top 候选之间高度接近，主控判断置信度不足。
- 任务涉及创业、职业主线、长期内容主轴或基础设施路径依赖。
- 物种识别存在不确定性（混合物种、用户表达模糊）。
- 核心概念在 L0/L1 但未触发前置重构（需复核是否合理）。

**subagent 派发 Prompt**：

```markdown
## 用户原始需求

${用户原始需求原文}

## 待复核结论

${阶段五结论报告}

## 物种识别与概念清晰度记录

${阶段一物种识别 + 概念清晰度评估原文}

## 复核目标

你是独立的战略校验员，不是结论作者的助手。请只检查这份执行前 idea 判断结论是否可靠，不要提出 MVP、实现计划、增长方案或执行路线。

逐项验证（按 workflow/validate-result.md 五组校验执行）：

第一组：结论结构
- 是否严格停留在执行前方向校验层。
- 是否明确了用户目标函数。
- 是否给出主推/观察/暂缓或通过/暂存/否决/前置概念重构/前置研究规划结论。
- 是否说明了每个关键结论的战略理由与反方风险。

第二组：物种识别与适配
- 是否正确识别了 Idea 物种？
- 是否避免了用一种物种的标准（如 PMF）评估另一种物种（如方法论）？
- 「推荐下一步路径」是否与物种匹配？（不要给方法论推 MVP）

第三组：概念清晰度处理
- L0/L1 概念是否触发了正确的路由？
- 是否避免了把「需要研究」误判为「核心资产缺失」？

第四组：语气中立
- 是否包含动机攻击词（虚荣心、心动陷阱、宏大叙事陷阱等）？
- 是否包含命令式表达（你应该、你必须等）？
- 是否包含隐喻夸大（深坑、死路、空中楼阁等）？

第五组：常见回答处理
- 「需要研究」「这是执行问题」「需要能够 X」类回答是否执行了二次追问？

## 报告格式

请按以下结构返回：
- 结论可靠性：通过 / 有条件通过 / 不通过
- 主要问题：按严重程度列出，每条标注属于哪组校验
- 缺失澄清：列出还必须问用户的问题
- 建议修正：只修正判断结论，不写执行方案
- 物种适配建议：若发现物种识别错误或问题集错配，明确指出正确的物种和应使用的问题集

## 执行要求

你是本次交付的最后一道质量关卡。你的复核结论直接决定用户是否会把有限执行力押到某个方向上。

请注意：
- 每个判断必须引用待复核结论中的具体内容作为证据。
- 发现问题时，必须指出问题会如何导致方向误判。
- 如果你认为结论通过，请额外检查一个最容易被忽略的反方风险。
- 声称「已通过」但没有证据的复核视为未执行。
- **特别检查**：是否存在「用产品标准评估非产品物种」的常见误判？是否存在「用动机评判替代假设评判」的语气违规？
```

## 注意事项

- 简洁不是少判断，而是少问低杠杆问题。
- 分数只可作为辅助，不可替代否决项、关系判断和目标函数。
- 单个 idea 也必须允许被否决；多 idea 场景也必须允许 Top 2 实际是「主方向 + 载体方向」。
- 若用户说「都重要」，改问「最不能接受牺牲哪一个」或「最近 10 个真实任务分别落在哪类方向」。
- 若用户开始问执行细节，记录为后续议题，并拉回方向判断。
- **物种意识**：用户表达 Idea 时使用的形态描述（如「想做一个工具」）可能是表层想象。务必通过五维度推导确认核心物种。
- **概念意识**：用户表达 Idea 时使用的抽象术语（如「降低语言熵」「认知升维」）可能是 L0/L1 概念。务必评估清晰度并按路由处理。
- **语气意识**：在每一轮输出前自查措辞，避免动机攻击、命令式、隐喻夸大。
