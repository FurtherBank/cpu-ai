# 战略五问门参考（元定义版）

这个参考用于执行前判断 idea 的方向价值。
它不评估实现难度，不替代用户执行验证，不讨论 MVP。

**关键变更**：本版本不再提供物种无关的「统一五问」。
每一问的具体标准和判定信号会**根据 Idea 物种动态调整**（参见 [idea-species.md](./idea-species.md)）。

## 核心区分

- **方向价值**：这个方向是否值得占用长期机会成本。
- **方案质量**：具体产品、功能、内容形式或实现路径是否设计得好。
- **执行便利**：现在是否容易开始做。

本 workflow 只判断方向价值。方案质量和执行便利最多作为风险备注，不作为主排序依据。

## 五问的元定义

五问的抽象层在所有物种下保持稳定，但**每个问题的具体含义**随物种变化。

### 元问题 1：根问题强度

**抽象定义**：这个 Idea 命中的是长期反复出现的底层问题，还是一次性兴趣？

**物种适配**：

| 物种 | 具体追问 | 强信号 |
|---|---|---|
| 产品/工具 | 目标用户的痛点是否长期反复出现？ | 用户/自己过去 3-6 个月反复遇到 |
| 方法论 | 这个方法论解决的认知问题是否长期反复出现？ | 用户在多个不同领域中反复遇到同一类困惑 |
| 研究 | 这个研究问题是否长期被领域内反复讨论但未解决？ | 多个学科/作者讨论但无统一答案 |
| 内容 | 这类内容回应的需求是否长期存在？ | 受众的同类问题持续出现 |
| 商业 | 市场需求是否长期存在且未被充分满足？ | 用户当前用替代品凑合 |
| 生活方式 | 这个状态变化是否长期是用户的核心追求？ | 用户多次表达过、多次尝试过 |
| 基础设施 | 下游任务是否长期反复需要它？ | 用户列出 3+ 个未来真实场景会调用 |
| 组织能力 | 这种能力缺失是否长期是组织瓶颈？ | 多次在协作中暴露同类问题 |

### 元问题 2：独特进入权

**抽象定义**：用户是否拥有别人短期难复制的理解、素材、渠道、数据、身份、工作流或亲身场景？

**物种适配**：

| 物种 | 具体追问 |
|---|---|
| 产品/工具 | 你凭什么比普通进入者更适合做这个产品？（场景洞察 / 渠道 / 技术 / 用户关系） |
| 方法论 | 你的认知或经验背景使你比普通学习者更容易抽象出这个方法论吗？ |
| 研究 | 你的学科基础、信息源或思考积累，是否构成研究路径的有效起点？ |
| 内容 | 你的视角、素材库、亲历经验是否他人难以复制？ |
| 商业 | 你的资源、人脉、技术或场景是否构成可持续护城河？ |
| 生活方式 | 你的当前条件（时间 / 健康 / 关系）是否支撑这个改变？ |
| 基础设施 | 你比谁更清楚下游任务的真实需求？是否有跑通其他基础设施的经验？ |
| 组织能力 | 你的角色、信任度、协作基础是否支撑这种能力构建？ |

**研究物种的特别提示**：
- 「需要研究」是合理起点，不应被判定为缺乏独特进入权。
- 应追问：「研究路径是否可设计？」「是否有可借鉴的现成基础？」

### 元问题 3：复利资产

**抽象定义**：做这个方向一年后，资产是否越来越厚，而不是每次都从零开始？

**物种适配**：

| 物种 | 复利资产形态 |
|---|---|
| 产品/工具 | 用户基础、数据、产品成熟度、技术沉淀 |
| 方法论 | 应用案例库、方法论自身的迭代版本、可教学化材料 |
| 研究 | 阶段性结论、可发表/分享材料、研究基础设施 |
| 内容 | 内容库、受众、风格资产、再创作素材 |
| 商业 | 收入、复购、品牌、运营 SOP |
| 生活方式 | 习惯沉淀、状态变化、可示范效果 |
| 基础设施 | 被调用次数、调用方多样性、API/接口稳定性 |
| 组织能力 | 流程文档、训练好的成员、复制方法 |

### 元问题 4：上游杠杆

**抽象定义**：它是否能改善后续许多决策或任务，而不仅是局部省时间？

**物种适配**：

| 物种 | 上游杠杆判定 |
|---|---|
| 产品/工具 | 它是否成为其他工具/工作流的输入层？ |
| 方法论 | 它是否影响用户后续多个决策类型？ |
| 研究 | 它是否会改变用户后续看待相关领域的方式？ |
| 内容 | 它是否会成为其他内容/课程/产品的素材源？ |
| 商业 | 它的收入是否能反哺其他方向？ |
| 生活方式 | 它的状态改善是否会赋能其他方向？ |
| 基础设施 | 它能让多少类下游任务变得更快/更稳？ |
| 组织能力 | 它能让组织中多少类协作变得更顺？ |

### 元问题 5：反方否决

**抽象定义**：这个方向最可能因为哪条根本前提错误而不值得做？

**物种适配的核心反方点**：

| 物种 | 最常见的根本前提风险 |
|---|---|
| 产品/工具 | 目标用户群不清；痛点真实但不愿付费；个人痛感被误判为群体需求 |
| 方法论 | 方法论无法手动跑通；方法论只在特定领域成立；用户无意愿反复使用 |
| 研究 | 研究路径不存在；缺乏证伪标准；问题定义本身循环 |
| 内容 | 受众不清晰；持续创作动力不足；表达独特性弱 |
| 商业 | 商业模式不成立；护城河浅；获客成本超过 LTV |
| 生活方式 | 收益/成本不匹配；可持续性弱；依赖外部条件 |
| 基础设施 | 下游真实调用次数为零；上游成本超过下游收益总和 |
| 组织能力 | 脱离创始人无法运行；激励机制不成立 |

**通用反方点**（所有物种通用）：
- 概念假设未被验证（参见 concept-clarity.md）
- 短期热点/新鲜感被误当成长期价值
- 容易实现/技术顺手被误当成战略价值
- 方向本身会制造新的分散注意力

## 简洁评分规则

需要快速比较多个 idea 时，每个判断点只标 `强 / 中 / 弱 / 不明`。不要默认使用精细数字。

只有在用户明确需要排序表时，才可用 0-2 辅助分：
- 0：缺失或明显弱。
- 1：有信号但不稳定。
- 2：强信号且有事实支撑。

**关键约束**：若出现致命否决项，即使总分高，也必须降级或暂缓。
**物种约束**：评分时必须使用对应物种的判定标准，不能跨物种比较绝对分数。

## 结论词

### 多 idea 场景

- **主推**：战略理由强，反方风险可被前置澄清，且与用户目标函数高度贴合。
- **观察**：有价值但缺关键事实，或需要等待某个条件成立。
- **暂缓**：方向不坏，但当前不应占用主要注意力。
- **否决**：核心前提不成立，或会显著放大分散、换方向、机会成本浪费。

### 单 idea 场景

- **通过**：底层方向值得进入下一阶段设计。
- **暂存**：动机或部分方向有价值，但当前表达混杂，需要改写或澄清。
- **否决**：方向级价值不足，或主要由短期信号驱动。
- **前置概念重构**：方向可能成立，但核心概念尚未形式化，需要先做概念重构再回到本工作流。
- **前置研究规划**：方向是研究类，研究路径需要先设计才能评估值不值得投入。

## 物种与结论的合法组合

不是所有物种都能给出所有结论。常见的合法路径：

| 物种 | 常见合法结论 |
|---|---|
| 产品/工具 | 通过 / 暂存 / 否决 / 前置概念重构 |
| 方法论 | 通过（手动版本起步）/ 暂存 / 否决 |
| 研究 | 通过（带研究路径）/ 暂存 / 否决 / 前置研究规划 |
| 内容 | 通过 / 暂存 / 否决 |
| 商业 | 通过 / 暂存 / 否决 |
| 生活方式 | 通过 / 暂存 / 否决 |
| 基础设施 | 通过 / 暂存 / 否决（基于下游真实调用次数判断） |
| 组织能力 | 通过 / 暂存 / 否决 |

**关键提示**：
- 给方法论物种结论时，「通过」常常意味着「手动版本起步」，而非「立即工程化」
- 给研究物种结论时，「通过」常常意味着「按研究路径推进」，而非「按产品路径推进」
- 给基础设施物种结论时，必须基于「下游未来真实调用次数」判断，不能用「终局价值」自动推高
