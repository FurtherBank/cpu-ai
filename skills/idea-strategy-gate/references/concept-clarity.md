# 概念清晰度评估

战略校验的隐含前提是：**Idea 中使用的核心概念是清晰的**。
但很多 Idea 里包含尚未形式化的抽象概念（例如「降低语言熵」「提纯 DSL」「认知升维」「构建心智模型」「打造个人 IP」）。在这些概念被精确定义之前进行战略压测，会得出不可靠的结论。

本文档提供概念清晰度的分级、识别方法、以及「直接进入战略压测」与「前置概念重构」的路由规则。

## 核心原则

- **战略压测有前提条件**：前提是 Idea 的关键概念至少处于「可操作」级别。
- **不要把「概念未定义」误判为「方向不可行」**：很多有价值的研究方向起点就是模糊概念。
- **不要把「概念未定义」误判为「核心资产缺失」**：研究类 Idea 允许概念尚未形式化，但需要有「可设计的研究路径」。
- **路由不是结论**：触发前置概念重构 ≠ 否决该 Idea，而是延迟战略压测，先把概念形式化。

## 概念清晰度分级

对 Idea 中**每一个核心概念**单独评估清晰度。

### L0：黑箱（Black Box）

只有名称，没有定义。

特征：
- 用户无法用一句话给出输入/输出/边界
- 用其他同样模糊的概念解释它（循环定义）
- 只能用比喻或愿景表达

示例：
- 「降低语言熵」（什么算「熵高」？怎么测量？降到什么程度算成功？）
- 「打造个人 IP」（IP 是什么形态？凭什么属性 IP 化？验证标准是什么？）
- 「认知升维」（升到哪一维？维度怎么定义？升维的判断信号是什么？）

### L1：轮廓（Sketch）

有直觉描述，但输入/输出/边界仍不清。

特征：
- 用户能描述这个概念在做什么
- 但无法回答「输入是什么 / 输出是什么 / 边界在哪 / 怎么判定成功」中至少一项
- 与近邻概念（容易混淆的概念）边界不清

示例：
- 「优化通用自然语言用于建模」（输入和输出是什么？什么样的语言被认为是「优化后的」？）
- 「提纯领域 DSL」（什么算 DSL？怎么算「提纯成功」？）

### L2：可操作（Operationalized）

已有明确的输入、输出、判定标准、边界条件。

特征：
- 能给出该概念的「函数签名」（输入类型 → 输出类型）
- 能列出至少 3 个反例（什么不属于这个概念）
- 能与近邻概念清晰区分

示例：
- 「提取领域核心术语并按 X 公式筛选」其中 X 公式有明确步骤
- 「降低语言熵」具体定义为「在领域内识别同义概念并合并、识别歧义概念并消歧、识别冗余概念并删除」

### L3：已验证（Validated）

L2 基础上，已在至少 1 个真实场景跑通。

特征：
- 至少有一个完整的应用案例
- 案例中能看到该概念产生了真实价值
- 案例中暴露的问题已被记录

## 评估流程

阶段一收集到 Idea 描述后，按以下流程评估：

```markdown
## 概念清晰度评估：${idea_name}

### 步骤 1：识别核心概念
列出 Idea 描述中**决定可行性**的关键术语（通常 1-3 个）。
判断标准：如果这个概念被证伪或被证明无法形式化，整个 Idea 就站不住。

示例（用户的「自然语言编译计划」）：
- 核心概念 1：「降低语言熵」
- 核心概念 2：「领域 DSL 提纯」
- 核心概念 3：「专门研究过的筛选公式」

### 步骤 2：对每个概念评分

| 核心概念 | 当前级别 | 用户能给出的最高级别证据 |
|---|---|---|
| ${概念} | L0/L1/L2/L3 | ${用户实际能说出的定义/示例/案例} |

### 步骤 3：路由判定
按下文「路由规则」决定下一步。
```

## 路由规则

根据核心概念的清晰度组合，决定流向：

### 规则 A：所有核心概念 ≥ L2 → 直接进入战略压测

可以正常执行阶段三战略压测。
保留：「概念假设风险」作为反方信号之一。

### 规则 B：至少一个核心概念在 L0/L1，但 Idea 物种是「研究/探索（Research）」 → 直接进入战略压测，但调整问题集

研究类 Idea 允许从黑箱概念起步。
**不应**追问「这个公式是什么」，**应**追问：
- 你打算基于什么现成基础（学科 / 已有方法 / 类似研究）来推导这个概念？
- 你怎么判断这个概念被「研究清楚了」？（证伪标准）
- 研究路径上的第一个可验证里程碑是什么？

### 规则 C：至少一个核心概念在 L0/L1，且 Idea 物种是「产品/工具/商业项目」 → 触发前置概念重构

不能跳过概念形式化直接评估「产品该不该做」。
向用户提示：
- 「在评估这个 Idea 的产品形态前，需要先把核心概念 X 形式化」
- 「建议先用概念重构（t-concept-refactor）把 X 的输入/输出/边界定义清楚」
- 「概念形式化后再回到本工作流进行战略压测」

战略门此时**暂停**输出最终结论，给出「前置概念重构」作为合法结论。

### 规则 D：所有核心概念在 L0，且用户没有意愿研究它们 → 直接给出「概念未定义」预警

这种情况下 Idea 通常处于「灵感闪现」阶段，没有进入战略压测的最低条件。
战略门给出：
- 当前不进行通过/否决判断
- 列出必须澄清的最少 3 个问题
- 用户回答后再回到本工作流

## 与战略压测的协作

概念清晰度评估的结果会影响阶段三/四：

| 概念清晰度 | 战略压测的核心问题调整 |
|---|---|
| 全部 L2+ | 正常执行五问，保留「概念假设风险」 |
| 关键概念 L1（研究物种） | 五问中「独特进入权」改为「研究路径有效性」，「反方否决」改为「证伪标准」 |
| 关键概念 L0（研究物种） | 增加问题：「在哪 1 个真实领域可以做最小规模手动试跑？」 |
| 关键概念 L0/L1（产品物种） | 暂停战略压测，前置触发概念重构 |

## 与「方法论」物种的特殊关系

方法论类 Idea（参见 idea-species.md 物种 B）有一个常见模式：

- 用户提出一个看似抽象的方法论
- 核心概念暂处于 L1（轮廓级）
- 用户能确认「即使是手动 A4 纸版本也愿意用」

这种情况下：
- **不要**触发前置概念重构（因为方法论的成功定义是「自己反复用」，不是「概念被精确化」）
- **应**追问：「你愿意用第一个真实场景手动跑一遍这套方法吗？」
- 用「跑通一次手动版本」作为概念清晰度从 L1 升到 L2 的最小验证

这是研究类与方法论类共享的关键路径：**手动跑通 1 次 = 概念升级**。

## 反例：错误路由的后果

| 错误路由 | 错误后果 | 修正 |
|---|---|---|
| L0 概念直接进入产品物种战略压测 | 用「产品独特进入权」标准追问，得到「核心资产缺失」结论 | 触发前置概念重构 |
| L0 研究类 Idea 被追问 PMF | 误判为「需要研究 = 不可行」 | 改用研究物种问题：研究路径、证伪标准 |
| L1 方法论被强制要求形式化到 L2 才能继续 | 阻断方法论自然演进路径 | 用「手动跑通 1 次」作为升级路径，允许边用边升级 |
| 把「需要研究」字面理解为「资产缺失」 | 把研究的正常起点判定为缺陷 | 区分「研究路径不存在」（缺陷）与「研究尚未完成」（正常） |
