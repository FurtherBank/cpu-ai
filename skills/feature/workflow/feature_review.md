# Feature Review 审查执行规范

> 本文件是 Coder Subagent 进行代码风险自审的执行规范。请严格遵守以下要求。

## 审查标准

在完成代码修改后，需要识别在修改代码过程中看到的项目代码中的风险信号。

**执行步骤**：

1. **回顾修改过程**：
   - 回顾在修改代码过程中接触到的所有代码文件
   - 识别这些代码中可能存在的风险信号

2. **参照风险信号标准**：
   - 参照 `../references/代码风险信号.md` 中的六大核心信号进行识别
   - 六大核心信号包括：
     - 隐性认知过多 (Excessive Implicit Cognition)
     - 职责边界错位 (Misplaced Responsibility)
     - 修改扩散效应 (Modification Ripple Effect)
     - 架构范式不一致 (Inconsistent Architectural Paradigms)
     - 防御性缺失 (Defensive Deficiency)
     - 过度设计与僵尸化 (Over-Engineering & Zombie Code)

3. **记录风险信号**：
   - 对于识别到的风险信号，记录具体的代码位置和问题描述
   - 说明该风险信号可能带来的影响
   - 如果风险信号较为严重，建议在记录中标注优先级

## 输出格式要求

请严格按照以下 Markdown 格式输出您的审查报告：

```markdown
## 代码风险信号

### 识别到的风险信号

#### 1. [风险信号类型] - [具体位置]

- **问题描述**：{{具体问题描述}}
- **代码位置**：{{文件路径和具体位置}}
- **影响评估**：{{可能带来的影响}}
- **优先级**：{{高/中/低}}

#### 2. [风险信号类型] - [具体位置]

- **问题描述**：{{具体问题描述}}
- **代码位置**：{{文件路径和具体位置}}
- **影响评估**：{{可能带来的影响}}
- **优先级**：{{高/中/低}}

### 无风险信号

（如果未识别到风险信号，可标注：本次修改过程中未发现明显的代码风险信号）
```
