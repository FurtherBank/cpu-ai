# 阶段一：规范解析 执行指导

## 输入信息

**必须入参**：

- 规范文档（完整文件路径）— 来自 spec-design 产出，必须完整阅读，不能只读部分
- 改造目标描述（一句话）— 来自用户或上游任务说明

## 目标要求

**任务**：将规范文档的每条语句原子化，建立可追溯的条款索引表

**目标**：每条规范语句有唯一 ID；每条有类型和优先级标注；有技术决策依赖标记

**特性要求**：

- 每条规范语句有唯一 ID（建议格式：S1/S2/S3…，或按章节 A1/A2/B1/B2…）
- 每条有类型标注（领域概念 / 存储约束 / 行为规则 / 集成要求）
- 每条有优先级（MUST / SHOULD / MAY）
- 每条有技术决策依赖标记（是/否）
- 无「笼统理解整段规范」的模糊条目

## 工作依据

**规范内容的三类典型结构**：

- 「是什么（概念定义）」→ 类型标注为「领域概念」，后续映射到具体 DB 表/实体
- 「满足什么条件（约束规则）」→ 类型标注为「存储约束」或「行为规则」，后续驱动四态差距判断
- 「如何交互（集成要求）」→ 类型标注为「集成要求」，后续映射到接口或配置

**技术决策依赖的识别方法**：

- 若条款只描述概念/规则而不说明「存在哪里/怎么实现」，且不同实现方案会导致不同的表/接口设计 → 标「是」
- 若条款明确了「存在数据库」「通过 HTTP 接口暴露」等落点形式 → 标「否」

**实现设计类的识别方法（新增标注维度）**：

「实现设计类」用于标识规范条款中包含「执行某个自动化流程/agent」的粗粒度描述，这类条款内部隐含的实现方案（prompt 框架、触发时机、交互方式）需要专项设计才能确定，仅靠技术决策无法收敛。

识别信号（满足任一即标「是」）：

- 条款含「运行 agent」「触发 agent」「agent 任务」「自动生成」「自动更新」等描述
- 条款描述了一个「如果 X 则执行 Y」的触发机制，但 Y 的内部实现在代码库中不存在
- 条款要求「调用大模型/AI 完成 X 任务」，但 prompt 设计、输入输出格式、质量校验方式均未在规范中说明

**集合运算类条款的处理（重要）**：

- 「代理权限集 = A ∩ B ∩ C ∩ D」这类公式，把每个集合元素（A/B/C/D）**分别拆成独立条款**
- 集合运算本身也是一条条款（描述计算规则）
- 这是最容易被漏掉的条款类型

**优先级标注规则**：

- 规范写「必须」「require」「shall」→ MUST
- 规范写「应该」「建议」「should」→ SHOULD
- 规范写「可以」「可选」「may」「optional」→ MAY
- 措辞模糊时默认 MUST，并在备注中写「待确认」

## 产出格式

```markdown
## 条款索引表

| ID  | 描述                                               | 类型     | 优先级 | 技术决策依赖 | 实现设计类 | 备注                               |
| --- | -------------------------------------------------- | -------- | ------ | ------------ | ---------- | ---------------------------------- |
| S1  | 开放资产必须携带 type=open 标记                    | 存储约束 | MUST   | 否           | 否         |                                    |
| S2  | 开放资产必须携带 tenant=租户id                     | 存储约束 | MUST   | 否           | 否         |                                    |
| S3  | 引入「租户」领域概念，作为开放平台的调用主体       | 领域概念 | MUST   | 是           | 否         | 存储方案待技术决策                 |
| S4  | 租户代行鉴权：凭证 = 租户 token + auth-workid      | 行为规则 | MUST   | 否           | 否         |                                    |
| S9  | 开启 OpenAPI 功能时运行 agent 全量生成 openapi 文档 | 行为规则 | MUST   | 是           | **是**     | agent prompt/触发机制/输出格式未定 |
| S5  | 代理权限集 = 开发者自管权限集 ∩ 租户业务开通范围集 | 行为规则 | MUST   | 是           | 否         | 「开发者自管权限集」存储形态待决策 |
```

**完成判定自检**：

- [ ] 条款总数量与规范段落数大致匹配（不应比段落少超过 20%）
- [ ] 集合运算类条款已逐个拆解
- [ ] 无「笼统理解」的整段描述
- [ ] 技术决策依赖标记已填写
- [ ] 实现设计类标注已填写；「实现设计类=是」的条款已在备注中写清「缺少哪方面的设计」
