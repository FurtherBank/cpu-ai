# 变更清单产出模板

## 条款索引表

| ID  | 描述                     | 类型                                | 优先级          | 技术决策依赖 | 实现设计类 | 备注 |
| --- | ------------------------ | ----------------------------------- | --------------- | ------------ | ---------- | ---- |
| S1  | [规范语句原文或精炼描述] | 领域概念/存储约束/行为规则/集成要求 | MUST/SHOULD/MAY | 是/否        | 是/否      |      |

_类型说明_：

- **领域概念**：定义了一个新的业务实体（如「租户」「开发者 App」）
- **存储约束**：规定了数据存储必须满足的条件（如「资产必须携带 tenant 字段」）
- **行为规则**：规定了系统运行时的行为逻辑（如「代理权限集 = A ∩ B ∩ C」）
- **集成要求**：规定了对外接口或跨系统交互的要求

_实现设计类说明_：

- **是**：条款包含「运行 agent」「触发自动化流程」「调用大模型完成 X」等描述，其内部实现设计（prompt 框架、触发条件逻辑、与现有系统的集成方式）需要专项设计。此类条款在阶段三需产出「需设计型缺口描述」，并在阶段三点五完成专项设计
- **否**：常规条款，无需专项设计
